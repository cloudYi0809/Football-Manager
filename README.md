# 绿茵王朝：2002

`Green Dynasty 2002` 是一款 Android 单机足球经理模拟游戏项目，目标是做出一个可长期游玩、可持续演化、具备真实经营压力和历史变动感的足球经理体验。

玩家将扮演俱乐部经理，从 2002 年代风格的足球世界出发，管理阵容、制定战术、推进赛季、处理转会、培养青训、应对伤病、经营财政，并在多年模拟中见证一个不断变化的足球世界。

## 项目定位

本项目不是简单的比分模拟器，而是一套完整的足球经理经营模拟框架。

核心目标包括：

* 构建可校准的比赛模拟引擎
* 支持长期赛季循环与存档演进
* 提供阵容、战术、合同、转会、青训等经营系统
* 通过 AI 俱乐部画像、经济通胀、蝴蝶效应等机制制造世界变化
* 支持虚构数据与本地数据包导入，降低版权风险
* 以 Android 单机体验为主，优先保证稳定性、可玩性和长程模拟性能

## 核心玩法模块

### 1. 比赛模拟引擎

比赛引擎采用分层模拟思路，避免纯随机比分。

计划包含：

* 阵容与球员状态输入
* 进攻、防守、控制力评分
* xG 预期进球生成
* 泊松分布比分生成
* 事件流生成
* 球员评分
* 射门、控球、角球、犯规、红黄牌、伤病等统计
* 战术克制与球星模板影响

### 2. 阵容与战术系统

玩家可以围绕球队阵容做日常管理。

计划包含：

* 首发与替补选择
* 球员位置适配
* 阵型配置
* 战术风格选择
* 压迫强度、防线高度、节奏、传球方式、比赛心态
* 战术克制关系
* 不同球员角色对比赛表现的影响

### 3. 赛季推进系统

游戏以日期为核心推进世界状态。

计划包含：

* 每日推进
* 赛程生成
* 联赛积分榜
* 杯赛推进
* 球员状态结算
* 伤病恢复
* 新闻事件
* 转会窗口
* 赛季归档
* 多年长程模拟

### 4. 转会、合同与谈判

转会系统是经营玩法的重要组成部分。

计划包含：

* 球员搜索
* 球探任务
* 转会报价
* 租借
* 自由签约
* 短期临时合同
* 续约谈判
* 薪资预算
* 市场价值变化
* AI 俱乐部转会决策

### 5. 青训与历史新星

青训系统用于增强长期培养体验。

计划包含：

* 青训学院
* 青训等级
* 潜力新星池
* 球员成长曲线
* 天赋发现
* 地区球探
* 训练影响
* 历史新星的非线性发展

### 6. 伤病与成长系统

球员不会只是一组静态数值。

计划包含：

* 伤病严重程度
* 复发风险

* 士气变化
* 年龄曲线
* 当前能力成长
* 潜力兑现
* 训练与比赛时间影响

### 7. AI 俱乐部与世界演化

世界中的其他俱乐部也会持续变化。

计划包含：

* AI 俱乐部画像
* 财政能力
* 青训偏好
* 明星偏好
* 风险偏好
* 工资控制
* 经理耐心
* 转会策略
* 联赛经济通胀
* 长期竞争格局变化

### 8. 蝴蝶效应系统

玩家的选择会影响世界走向。

计划包含：

* 球员成长路径变化
* 俱乐部战略变化
* 历史新星命运变化
* 事件影响扩散
* 多层级世界状态记录

## 数据库设计方向

项目采用三库结构：

```text
history.db
只读历史基础数据，包含球员、俱乐部、赛季、比赛、赛事、历史新星池等。

save_xxx.db
玩家存档数据，记录当前世界状态、球员状态、俱乐部状态、转会、伤病、新闻、AI 决策、蝴蝶事件等。

cache.db
可重建缓存数据，用于搜索索引、榜单缓存、统计缓存等。
```

这种结构的目标是：

* 降低存档污染基础数据的风险
* 支持多个独立存档
* 便于后续版本迁移
* 提升搜索与统计性能
* 支持长程模拟后的数据归档

## 设计原则

### 可玩性优先

系统复杂度必须服务于玩家体验。任何算法、数据表和模拟机制，最终都要让玩家感受到明确的经营反馈。

### 可校准优先

比赛结果、球员成长、经济变化、AI 转会等系统都需要可以调参、可以测试、可以回归，不依赖不可控随机。

### 长程稳定优先

足球经理游戏的核心体验来自多年经营，因此存档体积、性能、版本迁移和长期世界稳定性是项目重点。

### 版权合规优先

公开版本默认使用虚构数据，不内置真实队徽、球衣、头像或未经授权的真实赛事资产。真实数据仅作为用户本地导入包处理。
