> 任务编号：T01
> 周期：W4-W7（4 周）
> 优先级：P0（A 轨数据基础，游戏启动前提）
> 依赖：T00 三库结构（上游，已定义 history.db 全部 Entity）
> 下游：T02 比赛引擎（球员属性）/ T06 赛程积分榜（赛事俱乐部）/ T11 转会（经纪人）/ T14 球探（球探数据）/ T15 历史新星池（新星配置）/ T17 经济模型（经济配置）/ T20 蝴蝶效应（历史转会路径）

## 一、业务目标与范围

### 1. 业务目标

将游戏初始数据从 CSV/JSON 数据包导入到 history.db（只读库），为后续所有游戏系统提供基础数据支撑。导入内容覆盖球员、俱乐部、联赛、赛程、历史新星、经纪人、球探、蝴蝶效应配置、经济模型配置 9 大类。

### 2. V1 做的范围

| 类别 | 数据内容 | 数据量级 | 目标表 |
|------|----------|----------|--------|
| 球员基础 | 姓名/国籍/位置/生日/惯用脚/身高体重 | 10000+ | player |
| 球员属性 | CA/PA + 30+ 属性（按赛季） | 10000+ × N 赛季 | player_attributes |
| 俱乐部 | 名称/国家/城市/声望/财政/设施 | 500+ | club |
| 联赛/赛事 | 联赛结构/规则/级别 | 100+ | competition + season + club_competition_season |
| 赛程模板 | 历史赛季比赛（真实赛程） | 5000+ | match |
| 历史阵容 | 球员-俱乐部-赛季关系 + 号码 + 合同 | 30000+ | squad_membership |
| 历史转会 | 球员转会路径（蝴蝶基线） | 5000+ | transfer_history |
| 历史新星 | 100+ 巨星可发现配置 | 100+ | historical_prospect_pool |
| 球探模板 | 球探属性 + 区域知识 | 200+ | scout + scout_region_knowledge |
| 经纪人 | 经纪人属性 + 球员代理关系 | 100+ | agent + player_agent |
| 教练/员工 | 教练/队医属性 | 500+ | staff |
| 青训配置 | 俱乐部青训学院配置 | 500+ | youth_academy |
| 历史事件 | 剧本事件（挑战模式用） | 50+ | historical_event |
| 蝴蝶配置 | 历史关键转会路径标记 | 在 transfer_history 标记 | transfer_history.was_interrupted |
| 经济配置 | 通胀指数 + 联赛商业系数 | 100+ 年 + 9 联赛 | economy_index + league_economy_profile |
| 数据包清单 | manifest 元信息 | 1 | data_pack_manifest |

### 3. V1 不做（推迟）

- **云端数据包下载**：V1 仅支持 assets 内置 + 本地文件导入，不联网
- **数据编辑器**：T25 任务范围，V1 只读导入
- **增量数据包合并**：V1 仅支持「全量替换」，不支持多数据包合并
- **真实赛程自动回放**：V1 仅导入历史赛季真实比赛到 history.match，未来赛季由 T06 生成器生成
- **数据问题反馈 UI**：local_data_issue 表 V1 仅建表，编辑器在 T25
- **多语言译名导入**：V1 仅导入单一语言字段，多语言在 T0F-4
- **图片资源校验**：V1 仅校验路径字符串非空，不校验文件实际存在（图片懒加载由 T0F-4 处理）

## 四、核心类设计

### 1. DataImportManager（导入管理器）

```kotlin
// api/DataImportManager.kt
class DataImportManager(
    private val context: Context,
    private val historyDb: HistoryDatabase,
    private val config: ImportConfig,
    private val ioDispatcher: CoroutineDispatcher = Dispatchers.IO
) {

    /**
     * 入口：执行完整数据导入
     * @param packPath 数据包目录（null 表示用 assets 内置虚构包）
     * @param callback 进度回调（UI 订阅）
     * @return ImportReport
     */
    suspend fun importPack(
        packPath: String?,
        callback: ImportProgressCallback? = null
    ): ImportReport = withContext(ioDispatcher) {
        val startMs = System.currentTimeMillis()
        val report = ImportReport()

        // 1. 解析数据包路径（assets 或本地目录）
        val packDir = resolvePackDir(packPath)
        callback?.onStage(ImportStage.LOADING_MANIFEST)

        // 2. 加载并校验 manifest.json
        val manifest = ManifestLoader(packDir).loadAndValidate()
        if (!manifest.isSchemaCompatible(SchemaVersion.CURRENT)) {
            return@withContext report.fail("schema_version 不兼容：${manifest.schemaVersion}")
        }
        if (!manifest.verifyChecksum(packDir, config.enableChecksumVerify)) {
            return@withContext report.fail("数据包 checksum 校验失败")
        }

        // 3. 准备 history.db（关闭只读约束以写入）
        HistoryDbWriter(historyDb).openForWrite()

        // 4. 按依赖顺序编排 Importer 链
        val importers = buildImporterChain(packDir, manifest)
        val totalCount = importers.size
        var index = 0

        // 5. 写入 manifest 元信息（先写，便于断点识别）
        writeManifest(manifest)

        // 6. 逐 Importer 执行
        for (importer in importers) {
            index++
            callback?.onStage(importer.stage, index, totalCount)
            try {
                val subReport = importer.execute()
                report.merge(subReport)
                callback?.onProgress(index, totalCount, subReport.summary())
            } catch (e: Exception) {
                report.addError(ImportError(
                    stage = importer.stage.name,
                    message = "Importer 异常：${e.message}",
                    throwable = e
                ))
                if (config.abortOnError) {
                    HistoryDbWriter(historyDb).closeAndRestoreReadOnly()
                    return@withContext report.fail("导入中止（abortOnError=true）：${e.message}")
                }
            }
        }

        // 7. 跨表引用完整性最终校验
        callback?.onStage(ImportStage.FINAL_VALIDATION)
        val crossValidation = CrossTableValidator(historyDb).validate()
        report.merge(crossValidation)

        // 8. 关闭写入模式，恢复只读
        HistoryDbWriter(historyDb).closeAndRestoreReadOnly()

        // 9. VACUUM 优化（异步，不阻塞）
        if (config.enableVacuumAfterImport) {
            historyDb.openHelper.writableDatabase.execSQL("VACUUM")
        }

        report.durationMs = System.currentTimeMillis() - startMs
        callback?.onComplete(report)
        report
    }

    /**
     * 依赖顺序：基础表 → 关系表 → 派生表
     * 顺序错误会导致外键校验失败
     */
    private fun buildImporterChain(packDir: File, manifest: DataPackManifest): List<BaseImporter> {
        val csv = { name: String -> File(packDir, name) }
        val validator = DataValidator(config.validation)
        val writer = BatchWriter(historyDb, config.batchSize)

        return listOf(
            EconomyConfigImporter(csv("economy_index.csv"), csv("league_economy_profile.csv"), validator, writer),
            LeagueImporter(csv("seasons.csv"), csv("competitions.csv"), csv("club_competition_season.csv"), validator, writer),
            ClubImporter(csv("clubs.csv"), csv("youth_academy.csv"), validator, writer),
            PlayerImporter(csv("players.csv"), csv("player_attributes.csv"), validator, writer, config.playerStreamBatchSize),
            SquadImporter(csv("squads.csv"), validator, writer),
            TransferImporter(csv("transfers.csv"), validator, writer),
            ButterflyConfigImporter(csv("butterfly_transfers.csv"), historyDb, validator, writer),
            ScheduleImporter(csv("matches.csv"), validator, writer),
            ProspectImporter(csv("historical_prospects.csv"), validator, writer),
            ScoutImporter(csv("scouts.csv"), csv("scout_region_knowledge.csv"), validator, writer),
            AgentImporter(csv("agents.csv"), csv("player_agent.csv"), validator, writer),
            StaffImporter(csv("staff.csv"), validator, writer),
            HistoricalEventImporter(csv("historical_events.csv"), validator, writer)
        )
    }

    private fun resolvePackDir(packPath: String?): File {
        return if (packPath != null) {
            File(packPath).also { require(it.exists() && it.isDirectory) { "数据包目录不存在：$packPath" } }
        } else {
            // 从 assets 解压到 filesDir/packs/fictional_default
            val target = File(context.filesDir, "packs/fictional_default")
            if (!target.exists() || config.alwaysReextractAssets) {
                target.mkdirs()
                extractAssetsPack("packs/fictional_default", target)
            }
            target
        }
    }

    private fun extractAssetsPack(assetPath: String, target: File) {
        val assetFiles = context.assets.list(assetPath) ?: emptyArray()
        for (name in assetFiles) {
            context.assets.open("$assetPath/$name").use { input ->
                File(target, name).outputStream().use { output -> input.copyTo(output) }
            }
        }
```

```kotlin
// parser/CsvParser.kt
class CsvParser(
    private val file: File,
    private val charset: Charset = Charsets.UTF_8,
    private val delimiter: Char = ',',
    private val quote: Char = '"'
) : Closeable {

    /**
     * 流式逐行解析，避免一次性加载 10000+ 行到内存
     * @param action 每行回调（rowNumber 从 1 开始，1 = 表头）
     */
    fun forEachRow(action: (rowNumber: Long, row: CsvRow) -> Unit) {
        file.bufferedReader(charset).use { reader ->
            // 1. 读取表头，建立列名 → 列索引映射
            val headerLine = reader.readLine() ?: error("CSV 文件为空：${file.name}")
            val headers = parseLine(headerLine)
            val headerIndex = headers.mapIndexed { i, name -> name.trim() to i }.toMap()

            var line: String?
            var rowNumber = 1L
            while (run { line = reader.readLine(); line != null }) {
                rowNumber++
                val values = parseLine(line!!)
                action(rowNumber, CsvRow(rowNumber, headers, values, headerIndex))
            }
        }
    }

    /**
     * CSV 行解析（支持引号、转义、嵌入换行）
     * 简化版：假设无嵌入换行，复杂场景可换用 opencsv / commons-csv
     */
    private fun parseLine(line: String): List<String> {
        val result = mutableListOf<String>()
        val current = StringBuilder()
        var inQuotes = false
        var i = 0
        while (i < line.length) {
            val c = line[i]
            when {
                c == quote -> {
                    if (inQuotes && i + 1 < line.length && line[i + 1] == quote) {
                        current.append(quote); i++  // 转义引号
                    } else {
                        inQuotes = !inQuotes
                    }
                }
                c == delimiter && !inQuotes -> {
                    result.add(current.toString()); current.clear()
                }
                else -> current.append(c)
            }
            i++
        }
        result.add(current.toString())
        return result
    }

    override fun close() { /* BufferedReader 已在 use 中关闭 */ }
}

// parser/CsvRow.kt
class CsvRow(
    val rowNumber: Long,
    val headers: List<String>,
    private val values: List<String>,
    private val headerIndex: Map<String, Int>
) {
    fun get(column: String): String? = headerIndex[column]?.let { values.getOrNull(it) }
    fun getOrEmpty(column: String): String = get(column) ?: ""
    fun getInt(column: String): Int? = get(column)?.trim()?.takeIf { it.isNotEmpty() }?.toIntOrNull()
    fun getIntOrDefault(column: String, default: Int): Int = getInt(column) ?: default
    fun getLong(column: String): Long? = get(column)?.trim()?.takeIf { it.isNotEmpty() }?.toLongOrNull()
    fun getBoolean(column: String, default: Boolean = false): Boolean =
        get(column)?.trim()?.lowercase()?.let { it == "1" || it == "true" } ?: default
}
```

```kotlin
// validator/RequiredFieldRule.kt
class RequiredFieldRule(vararg private val fields: String) : ValidationRule {
    override val field: String = fields.first()
    override fun check(row: CsvRow, context: ValidationContext): RuleOutcome {
        for (f in fields) {
            val v = row.get(f)
            if (v == null || v.isBlank()) {
                return RuleOutcome(Severity.ERROR, "必填字段缺失：$f", "REQUIRED_MISSING")
            }
        }
        return RuleOutcome(Severity.OK, "")
    }
}

// validator/RangeCheckRule.kt
class RangeCheckRule(
    override val field: String,
    private val min: Int? = null,
    private val max: Int? = null,
    private val default: Int? = null,
    private val nullable: Boolean = false
) : ValidationRule {
    override fun check(row: CsvRow, context: ValidationContext): RuleOutcome {
        val raw = row.get(field)
        if (raw.isNullOrBlank()) {
            return if (nullable) RuleOutcome(Severity.OK, "")
            else if (default != null) RuleOutcome(Severity.WARNING, "字段为空，使用默认值 $default", "DEFAULT_APPLIED")
            else RuleOutcome(Severity.ERROR, "必填字段为空：$field", "REQUIRED_MISSING")
        }
        val value = raw.toIntOrNull()
            ?: return RuleOutcome(Severity.ERROR, "字段非整数：$field=$raw", "TYPE_INVALID")
        if (min != null && value < min) return RuleOutcome(Severity.ERROR, "值 $value < 最小 $min：$field", "RANGE_UNDER")
        if (max != null && value > max) return RuleOutcome(Severity.ERROR, "值 $value > 最大 $max：$field", "RANGE_OVER")
        return RuleOutcome(Severity.OK, "")
    }
}

// validator/ForeignKeyRule.kt
class ForeignKeyRule(
    override val field: String,
    private val references: (ValidationContext) -> Set<Int>
) : ValidationRule {
    override fun check(row: CsvRow, context: ValidationContext): RuleOutcome {
        val raw = row.get(field)
        if (raw.isNullOrBlank()) return RuleOutcome(Severity.OK, "")  // 允许 null 外键
        val value = raw.toIntOrNull()
            ?: return RuleOutcome(Severity.ERROR, "外键非整数：$field=$raw", "FK_TYPE_INVALID")
        return if (references(context).contains(value)) RuleOutcome(Severity.OK, "")
        else RuleOutcome(Severity.ERROR, "外键不存在：$field=$value", "FK_NOT_FOUND")
    }
}

// validator/UniqueKeyRule.kt
class UniqueKeyRule(override val field: String) : ValidationRule {
    private val seen = mutableSetOf<String>()
    override fun check(row: CsvRow, context: ValidationContext): RuleOutcome {
        val value = row.get(field) ?: return RuleOutcome(Severity.ERROR, "唯一键为空：$field", "UNIQUE_NULL")
        return if (seen.add(value)) RuleOutcome(Severity.OK, "")
        else RuleOutcome(Severity.ERROR, "唯一键重复：$field=$value", "UNIQUE_DUPLICATE")
    }
}
```

### 6. BatchWriter（批量写入器，事务）

```kotlin
// writer/BatchWriter.kt
class BatchWriter(
    private val historyDb: HistoryDatabase,
    private val batchSize: Int = 500
) {

    /**
     * 批量写入：单大事务 + 内部分批
     * @param entities 待写入 Entity 列表
     * @param inserter 单条插入函数（用 DAO）
     * @param batchInserter 批量插入函数（用 DAO @Insert List）
     */
    fun <T : Any> writeBatch(
        entities: List<T>,
        inserter: (T) -> Unit,
        batchInserter: (List<T>) -> Unit
    ): BatchResult {
        if (entities.isEmpty()) return BatchResult(0, 0, 0)
        var success = 0
        var failed = 0
        var skipped = 0

        historyDb.runInTransaction {
            val totalBatches = (entities.size + batchSize - 1) / batchSize
            for (batchIndex in 0 until totalBatches) {
                val from = batchIndex * batchSize
                val to = minOf(from + batchSize, entities.size)
                val batch = entities.subList(from, to)
                try {
                    batchInserter(batch)
                    success += batch.size
                } catch (e: Exception) {
                    // 批量失败：回退为逐条插入，定位错误行
                    historyDb.openHelper.writableDatabase.execSQL("SAVEPOINT batch_$batchIndex")
                    try {
                        for (entity in batch) {
                            try {
                                inserter(entity)
                                success++
                            } catch (e2: Exception) {
                                failed++
                                // 错误行跳过，继续
                            }
                        }
                        historyDb.openHelper.writableDatabase.execSQL("RELEASE SAVEPOINT batch_$batchIndex")
                    } catch (e3: Exception) {
                        historyDb.openHelper.writableDatabase.execSQL("ROLLBACK TO SAVEPOINT batch_$batchIndex")
                        failed += batch.size
                    }
                }
            }
        }
        return BatchResult(success, failed, skipped)
    }

    /**
     * 流式写入：避免全量加载到内存
     * 用于球员等大数据量表
     */
    fun <T : Any> writeStream(
        stream: Sequence<T>,
        batchInserter: (List<T>) -> Unit
    ): BatchResult {
        var success = 0; var failed = 0; var skipped = 0
        val buffer = mutableListOf<T>()

        historyDb.runInTransaction {
            for (entity in stream) {
                buffer.add(entity)
                if (buffer.size >= batchSize) {
                    try {
                        batchInserter(buffer)
                        success += buffer.size
                    } catch (e: Exception) {
                        failed += buffer.size
                    }
                    buffer.clear()
                }
            }
            // 写入剩余
            if (buffer.isNotEmpty()) {
                try {
                    batchInserter(buffer)
                    success += buffer.size
                } catch (e: Exception) {
                    failed += buffer.size
                }
            }
        }
        return BatchResult(success, failed, skipped)
    }
}

data class BatchResult(val success: Int, val failed: Int, val skipped: Int)
```

```kotlin
    private val streamBatchSize: Int = 1000

    override fun execute(): StageReport {

        val errors = mutableListOf<ImportError>()

        // 1. 第一遍：流式解析 + 校验 + 写入 player 表
        val playerSequence = sequence {

                parser.forEachRow { rowNumber, row ->

                    when {
                        result.skipped -> { skipped++; return@forEachRow }

                            failed++

                                yield(mapper.mapPlayer(row) to rowNumber)

                            } catch (e: Exception) {
                                failed++
                                errors.add(ImportError(stage, rowNumber, message = "映射失败：${e.message}", throwable = e))
                            }
                        }
                    }
                }
            }
        }

        val playerDao = historyDb.playerDao()
        val playerResult = writer.writeStream(playerSequence) { batch ->
            playerDao.insertAll(batch.map { it.first })
        }

        val playerIds = historyDb.playerDao().getAllIds().toSet()

        val attrSequence = sequence {
            CsvParser(attributesFile).use { parser ->

                            failed++

// importer/ClubImporter.kt（含青训配置）
class ClubImporter(
    private val clubsFile: File,
    private val youthAcademyFile: File,
    private val validator: DataValidator,
    private val writer: BatchWriter
) : BaseImporter(ImportStage.IMPORTING_CLUB) {

    override fun execute(): StageReport {
        val mapper = ClubMapper()
        val errors = mutableListOf<ImportError>()
        var success = 0; var failed = 0; var skipped = 0
        val clubs = mutableListOf<ClubEntity>()

        CsvParser(clubsFile).use { parser ->
            parser.forEachRow { rowNumber, row ->
                val result = validator.validate("club", row, ValidationContext())
                when {
                    result.skipped -> skipped++
                    !result.isValid -> {
                        failed++
                        errors.add(ImportError(stage, rowNumber, message = result.errors.joinToString { it.message }))
                    }
                    else -> {
                        try { clubs.add(mapper.mapClub(row)); success++ }
                        catch (e: Exception) { failed++; errors.add(ImportError(stage, rowNumber, throwable = e)) }
                    }
                }
            }
        }

        val clubResult = writer.writeBatch(clubs, { historyDb.clubDao().insert(it) }) { historyDb.clubDao().insertAll(it) }

        // 加载 club_ids 供 youth_academy 外键校验
        val clubIds = historyDb.clubDao().getAllIds().toSet()
        val ctx = ValidationContext(clubIds = clubIds)

        val academies = mutableListOf<YouthAcademyEntity>()
        CsvParser(youthAcademyFile).use { parser ->
            parser.forEachRow { rowNumber, row ->
                val result = validator.validate("youth_academy", row, ctx)
                if (result.isValid) {
                    try { academies.add(mapper.mapYouthAcademy(row)); success++ }
                    catch (e: Exception) { failed++ }
                } else if (!result.skipped) failed++
            }
        }
        val academyResult = writer.writeBatch(academies, { historyDb.youthAcademyDao().insert(it) }) { historyDb.youthAcademyDao().insertAll(it) }

        return StageReport(stage, clubResult.success + academyResult.success,
            clubResult.failed + academyResult.failed, skipped, errors.take(1000))
    }
}
```

## 五、核心算法伪代码

### 1. 数据解析流程

```
FUNCTION parseAndImport(csvFile, entityName, mapper, validator, writer):
    sequence = newSequence()
    CsvParser(csvFile).forEachRow { rowNumber, row ->
        result = validator.validate(entityName, row, ctx)
        IF result.skipped:
            skipped++; CONTINUE
        IF NOT result.isValid:
            failed++; recordError(rowNumber, result.errors); CONTINUE
        TRY:
            entity = mapper.map(row)
            sequence.yield(entity)
            success++
        CATCH e:
            failed++; recordError(rowNumber, e)
    }
    writer.writeStream(sequence, batchInserter)
    RETURN StageReport(success, failed, skipped, errors)
```

### 2. 数据校验规则

#### 2.1 必填字段校验

```
RULE RequiredField(fields[]):
    FOR f IN fields:
        IF row[f] IS NULL OR row[f].trim() == "":
            RETURN Error("REQUIRED_MISSING", f)
    RETURN OK
```

#### 2.2 范围检查

```
RULE RangeCheck(field, min, max, nullable, default):
    raw = row[field]
    IF raw IS NULL OR raw.isBlank():
        IF nullable: RETURN OK
        IF default != NULL:
            row[field] = default  // 应用默认值
            RETURN Warning("DEFAULT_APPLIED")
        RETURN Error("REQUIRED_MISSING")
    value = toInt(raw)
    IF value IS NULL: RETURN Error("TYPE_INVALID")
    IF min != NULL AND value < min: RETURN Error("RANGE_UNDER")
    IF max != NULL AND value > max: RETURN Error("RANGE_OVER")
    RETURN OK
```

#### 2.3 引用完整性校验

```
RULE ForeignKey(field, references):
    raw = row[field]
    IF raw IS NULL OR raw.isBlank(): RETURN OK  // 允许 null 外键
    value = toInt(raw)
    IF value IS NULL: RETURN Error("FK_TYPE_INVALID")
    IF value NOT IN references(ctx): RETURN Error("FK_NOT_FOUND")
    RETURN OK
```

#### 2.4 唯一性检查

```
RULE UniqueKey(field):
    value = row[field]
    IF value IS NULL: RETURN Error("UNIQUE_NULL")
    IF value IN seenSet: RETURN Error("UNIQUE_DUPLICATE")
    seenSet.add(value)
    RETURN OK
```

#### 2.5 跨表引用完整性最终校验

```
FUNCTION crossTableValidate():
    issues = []

    // 1. player_attributes.player_id 必须在 player.player_id 中
    orphanAttrs = db.query("SELECT DISTINCT player_id FROM player_attributes
                            WHERE player_id NOT IN (SELECT player_id FROM player)")
    IF orphanAttrs.notEmpty:
        issues.add(Error("ORPHAN_PLAYER_ATTRIBUTES", orphanAttrs.size))

    // 2. squad_membership 三外键
    orphanSquads = db.query("SELECT * FROM squad_membership sm
                             WHERE sm.player_id NOT IN (SELECT player_id FROM player)
                                OR sm.club_id NOT IN (SELECT club_id FROM club)
                                OR sm.season_id NOT IN (SELECT season_id FROM season)")
    IF orphanSquads.notEmpty:
        issues.add(Error("ORPHAN_SQUAD_MEMBERSHIP", orphanSquads.size))

    // 3. match 外键
    orphanMatches = db.query("SELECT * FROM match m
                              WHERE m.season_id NOT IN (SELECT season_id FROM season)
                                 OR m.competition_id NOT IN (SELECT competition_id FROM competition)
                                 OR m.home_club_id NOT IN (SELECT club_id FROM club)
                                 OR m.away_club_id NOT IN (SELECT club_id FROM club)
                                 OR m.home_club_id = m.away_club_id")
    IF orphanMatches.notEmpty:
        issues.add(Error("ORPHAN_MATCH", orphanMatches.size))

    // 4. historical_prospect_pool.player_id 必须存在
    orphanProspects = db.query("SELECT * FROM historical_prospect_pool
                                WHERE player_id NOT IN (SELECT player_id FROM player)")
    IF orphanProspects.notEmpty:
        issues.add(Error("ORPHAN_PROSPECT", orphanProspects.size))

    // 5. 每个俱乐部在 2002/03 赛季必须有 ≥7 球员（凑得上首发+替补）
    thinSquads = db.query("SELECT club_id, COUNT(*) AS cnt FROM squad_membership
                           WHERE season_id = 2002_03_season_id
                           GROUP BY club_id HAVING cnt < 7")
    IF thinSquads.notEmpty:
        issues.add(Warning("THIN_SQUAD", thinSquads))

    RETURN issues
```

### 3. 批量写入（事务 + 分批 + 错误恢复）

```
FUNCTION writeBatch(entities, batchSize, inserter, batchInserter):
    success = 0; failed = 0
    db.beginTransaction()
    FOR batchIndex IN 0..totalBatches:
        batch = entities.subList(batchIndex * batchSize, min((batchIndex+1)*batchSize, entities.size))
        TRY:
            batchInserter(batch)
            success += batch.size
        CATCH e:
            // 批量失败：回退为逐条插入，定位错误行
            db.execSQL("SAVEPOINT batch_$batchIndex")
            TRY:
                FOR entity IN batch:
                    TRY:
                        inserter(entity)
                        success++
                    CATCH e2:
                        failed++
                db.execSQL("RELEASE SAVEPOINT batch_$batchIndex")
            CATCH e3:
                db.execSQL("ROLLBACK TO SAVEPOINT batch_$batchIndex")
                failed += batch.size
    db.setTransactionSuccessful()
    db.endTransaction()
    RETURN BatchResult(success, failed, 0)
```

## 九、性能要求

| 操作 | 目标 | 实现策略 |
|------|------|----------|
| 全量导入（10000 球员 + 500 俱乐部 + 100 联赛 + 全部派生表） | ≤30 秒 | 流式解析 + 批量事务（500/批）+ 关闭 journal/synchronous + VACUUM 末尾 |
| 增量更新（替换某表） | ≤5 秒 | 单表事务 + 单 Importer 执行 |
| 内存占用（峰值） | ≤100 MB | 流式 Sequence + 行处理即释放 + 不全量缓存 Entity List |
| 单表导入（10000 行 player_attributes） | ≤3 秒 | 流式 + 批量 1000/批 |
| 跨表引用校验 | ≤2 秒 | 已加载 ID Set 到内存 + 索引查询 |
| VACUUM | ≤5 秒 | 末尾单次执行，导入期间不开 |
| ImportReport 写日志 | ≤500 ms | 限制错误日志前 1000 条 |

### 性能优化要点

1. **流式解析**：CsvParser 使用 BufferedReader 逐行读，Sequence yield 逐条产出，避免 10000+ Entity 全量加载到 List。
2. **批量事务**：BatchWriter 内 `runInTransaction` 包裹全部批次，每批 500 行调用 DAO `@Insert List<T>`，减少 SQLite commit 开销。
3. **导入期间 PRAGMA 优化**：
   - `journal_mode = OFF`：关闭 WAL/truncate journal，写入不记日志
   - `synchronous = OFF`：不强制 fsync
   - `foreign_keys = ON`：保留外键检查（不可关，保证完整性）
4. **导入后 VACUUM**：重组碎片，压缩文件体积
5. **历史 ID 缓存**：player_ids / club_ids 等 Set<Int> 在跨表校验阶段加载到内存，外键校验 O(1)
6. **错误日志限制**：单阶段最多记 1000 条错误，避免错误爆炸拖慢导入
7. **并行化考虑**：V1 不并行（SQLite 单写），后续可考虑按表分库

## 十一、验收标准

依据 T01 任务范围与 V0.1 `12_数据导入清洗规范` §十四：

1. **CSV 解析**：能流式解析 16 种 CSV 文件，支持引号转义、UTF-8 BOM。
2. **数据校验**：必填 / 范围 / 外键 / 唯一 / 枚举 / 日期 6 类规则全部生效，错误码可识别。
3. **批量写入**：事务 + 分批 + Savepoint 错误恢复，单批失败不阻塞整体。
4. **数据规模**：支持 10000+ 球员 / 500+ 俱乐部 / 100+ 联赛 / 30000+ 阵容关系全量导入。
5. **性能达标**：全量导入 ≤30 秒，增量更新 ≤5 秒，内存峰值 ≤100 MB，VACUUM 后体积 ≤15 MB。
6. **跨表完整性**：导入后跨表校验 0 SEVERE 错误（孤儿记录为 0）。
7. **历史新星**：100+ 历史新星配置正确导入，与 T15 衔接。
8. **经济配置**：1992-2030 通胀指数 + 9 联赛商业系数正确导入，与 T17 衔接。
9. **蝴蝶基线**：关键转会路径标记写入 transfer_history，与 T20 衔接。
10. **数据包版本管理**：manifest.json 含 pack_id/schema_version/checksum，可识别数据包身份。
11. **双模式导入**：虚构包默认加载 + 本地包免责声明必签 + 资源目录分层隔离。
12. **导入报告**：成功/失败/跳过统计 + 错误明细前 1000 条 + 写入 logs/import_report_*.txt。
13. **进度 UI**：首次启动显示 ImportProgressScreen，13 阶段进度可见，完成后跳转主菜单。
14. **错误恢复**：单 Importer 失败可继续后续 / abortOnError 可配置 / VACUUM 后只读模式恢复。
15. **配置化**：所有参数走 `import_config.json`，修改不重编译生效。

## 十四、风险与应对

### 1. 10000 球员导入内存溢出

- **表现**：OOM 异常，导入中途崩溃
- **应对**：
  - CsvParser 流式 BufferedReader 逐行读
  - Sequence yield 逐条产出，避免 List 缓存
  - BatchWriter.writeStream 每 1000 行清空 buffer
  - 单测强制校验内存峰值 ≤100MB
- **降级**：若仍 OOM，可分片导入（player_id 范围分 10 批）

### 2. CSV 格式不规范（BOM / 引号 / 换行符）

- **表现**：解析错位，字段串行
- **应对**：
  - CsvParser 自动跳过 UTF-8 BOM
  - 支持引号内嵌入逗号、引号转义
  - 自动识别 `\n` / `\r\n` 换行符
- **降级**：使用 opencsv / commons-csv 第三方库

### 3. history.db 只读约束失效

- **表现**：导入时写入被 PRAGMA query_only 拒绝
- **应对**：
  - HistoryDbWriter.openForWrite 显式 `PRAGMA query_only = OFF`
  - 导入完成后 closeAndRestoreReadOnly 恢复 `PRAGMA query_only = ON`
  - 单测强制校验：导入前 query_only=OFF，导入后 query_only=ON

### 4. 跨表引用完整性失败

- **表现**：player_attributes 引用不存在的 player_id，导致存档创建连锁失败
- **应对**：
  - 严格按依赖顺序导入（基础表 → 关系表 → 派生表）
  - CrossTableValidator 最终校验 5 项孤儿检查
  - 任何 SEVERE 错误标记导入失败，不允许进入游戏
- **降级**：若数据包本身有缺陷，提供「自动删除孤儿记录」选项（V2）

### 5. 数据包版本不兼容

- **表现**：用户导入旧版数据包，schema_version 不匹配
- **应对**：
  - ManifestLoader 校验 schema_version 与 SchemaVersion.CURRENT 兼容
  - 不兼容时拒绝导入，提示用户升级数据包
  - V2 可加 migration 脚本支持旧数据包升级

### 6. 性能不达标

- **表现**：全量导入 >30 秒
- **应对**：
  - 关闭 journal_mode / synchronous 提速
  - 增大 batch_size（500 → 1000）
  - 减少校验规则（关闭 enum_check_strict）
  - 末尾 VACUUM 改为异步后台执行
- **降级**：极端情况可关闭部分校验，但跨表完整性校验不可关

### 7. 数据包 checksum 损坏

- **表现**：用户传输数据包时文件损坏
- **应对**：
  - ManifestLoader 校验每个文件 SHA-256 checksum
  - 损坏时拒绝导入，提示用户重新下载/复制
  - 提供配置开关 `enable_checksum_verify` 可关闭（不推荐）

### 8. 版权合规风险

- **表现**：用户上传含真实球员姓名/队徽的数据包被公开传播
- **应对**：
  - 虚构包默认加载，不含任何真实素材
  - 本地包导入强制免责声明签署
  - 资源目录分层（official_safe / user_local_pack / placeholders）
  - 应用商店描述避免「官方授权」字样
  - 不内置任何未授权数据下载入口
