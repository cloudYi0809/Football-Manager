## 一、业务目标与范围

### 1. 业务目标

完成 V0.1 09 §三转会流程 12 步中第 3-12 步（搜索/球探报告由 T10/T14 完成），让玩家能：

- 向任意有主/无主球员提交 5 种类型的报价。
- 与卖方俱乐部进行多轮拉锯谈判（接受/拒绝/还价）。
- 与球员经纪人进行个人条款谈判（基础 9 项 + 特殊 7 项 + 角色承诺 5 档）。
- 经历体检（小概率失败流产），最终完成转会并触发历史路径更新。
- 打断历史新星默认路径并触发蝴蝶效应（T15/T20 集成）。

### 2. V1 做的范围

- 5 种报价类型：永久转会、租借、租借带买断、自由签约、预合同。
- 卖方评估完整 6 因子公式 + 心理价位计算 + 5 档球员重要性。
- 球员加盟意愿完整 8 因子公式。
- 合同基础条款 9 项 + 特殊条款 7 项 + 角色承诺 5 档。
- 经纪人系统：5 种类型 + 8 项属性 + 7 种事件。
- 租借系统：7 项参数 + 租借成长 8 因子公式。
- 体检系统（含失败概率）。
- 多轮谈判（玩家出价 → 卖方还价 → 玩家再出价…，支持破裂）。
- 转会完成后触发 T15 提前签约打断 + T20 蝴蝶效应。
- 全部参数配置化（独立 `negotiation_config.json`）。

### 3. V1 不做（推迟）

- 球员交换（V0.1 09 §二.3 明确"后期可做"）。
- 转会窗截止日竞价大战（最后 24 小时多队抢人）。
- 经纪人主动上门推荐球员的完整任务流（仅保留事件触发与关系记录）。
- 体检详细医学报告（仅保留通过/失败/带条件通过三档）。
- 新闻发布会的多选项互动（仅生成固定新闻条目）。
- 第三方球队搅局竞价（V1 卖方只与玩家单线谈判）。

## 三、数据模型（Room Entity + DAO）

> 三库分离原则：
> - `history.db`（只读）：`Agent`（经纪人基础数据）、`HistoricalTransfer`（历史转会记录模板）
> - `save.db`（存档）：`TransferOffer`、`OfferRound`、`ContractTerms`、`NegotiationSession`、`MedicalExam`、`TransferRecord`、`LoanTerms`、`AgentPlayerRelation`（关系动态）、`SavePlayerContract`（合同快照）
> - `cache.db`：谈判中间态缓存（可选，V1 直接走 save.db）

### 1. TransferOffer（报价单）

```kotlin
// offer/TransferOffer.kt
@Entity(
    tableName = "transfer_offer",
    indices = [
        Index(value = ["saveId", "playerId"]),
        Index(value = ["saveId", "fromClubId"]),
        Index(value = ["saveId", "status"])
    ]
)
data class TransferOfferEntity(
    @PrimaryKey(autoGenerate = true) val offerId: Int,
    val saveId: Int,
    val playerId: Int,
    val fromClubId: Int,                    // 买方（玩家俱乐部）
    val toClubId: Int?,                     // 卖方（自由球员为 null）
    val offerType: String,                  // OfferType.name
    val transferFee: Int,                   // 转会费（自由签约为 0）
    val wageOffer: Int,                     // 周薪报价
    val contractYears: Int,                 // 合同年限
    val signingBonus: Int,                  // 签字费
    val agentCommission: Int,               // 经纪人佣金
    val rolePromise: String,                // RolePromise.name
    val status: String,                     // OfferStatus.name
    val createdDate: String,                // ISO 日期
    val expiresDate: String?,               // 报价有效期
    val currentRound: Int = 0,              // 当前谈判轮次
    val relatedOfferId: Int? = null         // 关联报价（预合同关联未来转会）
)

enum class OfferType {
    PERMANENT,          // 永久转会
    LOAN,               // 租借
    LOAN_WITH_BUYOUT,   // 租借带买断
    FREE_SIGNING,       // 自由签约
    PRE_CONTRACT        // 预合同（合同剩 ≤6 个月接触）
}

enum class OfferStatus {
    DRAFT,              // 草稿
    SUBMITTED,          // 已提交
    SELLER_EVALUATING,  // 卖方评估中
    SELLER_ACCEPTED,    // 卖方接受
    SELLER_REJECTED,    // 卖方拒绝
    SELLER_COUNTERED,   // 卖方还价
    PLAYER_NEGOTIATING, // 球员谈判中
    PLAYER_ACCEPTED,    // 球员接受
    PLAYER_REJECTED,    // 球员拒绝
    MEDICAL_PENDING,    // 体检待进行
    MEDICAL_PASSED,     // 体检通过
    MEDICAL_FAILED,     // 体检失败
    COMPLETED,          // 转会完成
    COLLAPSED,          // 谈判破裂
    WITHDRAWN           // 玩家撤回
}

@Dao
interface TransferOfferDao {
    @Insert suspend fun insert(offer: TransferOfferEntity): Long
    @Update suspend fun update(offer: TransferOfferEntity)
    @Query("SELECT * FROM transfer_offer WHERE offerId = :offerId AND saveId = :saveId")
    suspend fun getById(saveId: Int, offerId: Int): TransferOfferEntity?
    @Query("SELECT * FROM transfer_offer WHERE saveId = :saveId AND playerId = :playerId ORDER BY createdDate DESC")
    suspend fun getByPlayer(saveId: Int, playerId: Int): List<TransferOfferEntity>
    @Query("SELECT * FROM transfer_offer WHERE saveId = :saveId AND fromClubId = :clubId AND status IN (:statuses) ORDER BY createdDate DESC")
    suspend fun getByClubAndStatus(saveId: Int, clubId: Int, statuses: List<String>): List<TransferOfferEntity>
    @Query("SELECT * FROM transfer_offer WHERE saveId = :saveId AND toClubId = :clubId AND status = 'SUBMITTED'")
    suspend fun getPendingForClub(saveId: Int, clubId: Int): List<TransferOfferEntity>
    @Query("SELECT COUNT(*) FROM transfer_offer WHERE saveId = :saveId AND fromClubId = :clubId AND status = 'COMPLETED' AND createdDate BETWEEN :windowStart AND :windowEnd")
    suspend fun countCompletedInWindow(saveId: Int, clubId: Int, windowStart: String, windowEnd: String): Int
    @Query("SELECT * FROM transfer_offer WHERE saveId = :saveId AND playerId = :playerId AND status = 'COMPLETED' ORDER BY createdDate DESC LIMIT 1")
    suspend fun getLastCompletedByPlayer(saveId: Int, playerId: Int): TransferOfferEntity?
}
```

```kotlin
class OfferSubmitter(
    private val saveDb: SaveDatabase,
    private val historyDb: HistoryDatabase,
    private val economyService: EconomyService,        // T17

    /**
     * V0.1 09 §三 第 3 步：提交报价
     * 支持 5 种报价类型
     */
    suspend fun submitOffer(
        ctx: NegotiationContext,
        request: OfferRequest
    ): OfferSubmitResult {
        // 1. 校验转会窗状态
        val windowStatus = TransferWindowManager(config).getStatus(ctx.currentDate)
        if (windowStatus == TransferWindowStatus.CLOSED &&
            request.offerType != OfferType.PRE_CONTRACT &&
            request.offerType != OfferType.FREE_SIGNING) {
            return OfferSubmitResult.Failed("转会窗已关闭")
        }

        // 2. 校验球员可交易性
        val player = saveDb.savePlayerStateDao().getByIdWithAttributes(ctx.saveId, request.playerId)
            ?: return OfferSubmitResult.Failed("球员不存在")
        validateOfferTypeForPlayer(request.offerType, player, ctx.currentDate)

        // 3. 校验预算（转会费 + 签字费 + 经纪人佣金）
        val buyerClub = saveDb.saveClubStateDao().getByClub(ctx.saveId, ctx.buyerClubId)!!
        val totalUpfront = request.transferFee + request.signingBonus + request.agentCommission
        if (totalUpfront > buyerClub.transferBudget) {
            return OfferSubmitResult.Failed("预算不足（需要 $totalUpfront，剩余 ${buyerClub.transferBudget}）")
        }

        // 4. 校验工资预算（年化）
        val annualWageImpact = request.wageOffer * 52
        if (annualWageImpact > buyerClub.wageBudgetRemaining) {
            return OfferSubmitResult.Failed("工资预算不足")
        }

        // 5. 预合同特殊校验：合同剩余 ≤ 6 个月
        if (request.offerType == OfferType.PRE_CONTRACT) {
            val monthsLeft = monthsUntilContractEnd(player.contractUntil, ctx.currentDate)
            if (monthsLeft > 6) {
                return OfferSubmitResult.Failed("预合同仅允许合同剩余 ≤6 个月的球员")
            }
        }

        // 6. 创建报价单
        val offerId = saveDb.transferOfferDao().insert(TransferOfferEntity(
            offerId = 0,
            saveId = ctx.saveId,
            playerId = request.playerId,
            fromClubId = ctx.buyerClubId,
            toClubId = player.currentClubId,
            offerType = request.offerType.name,
            transferFee = request.transferFee,
            wageOffer = request.wageOffer,
            contractYears = request.contractYears,
            signingBonus = request.signingBonus,
            agentCommission = request.agentCommission,
            rolePromise = request.rolePromise.name,
            status = OfferStatus.SUBMITTED.name,
            createdDate = ctx.currentDate.toString(),
            expiresDate = ctx.currentDate.plusDays(config.offerValidityDays).toString(),
            currentRound = 0
        )).toInt()

        // 7. 创建谈判会话
        val agentId = resolveAgentId(request.playerId)
        saveDb.negotiationSessionDao().insert(NegotiationSessionEntity(
            sessionId = 0,
            saveId = ctx.saveId,
            offerId = offerId,
            playerId = request.playerId,
            agentId = agentId,
            stage = NegotiationStage.SELLER_EVALUATION.name,
            currentRound = 0,
            maxRounds = config.maxNegotiationRounds,
            buyerPatience = 100,
            sellerPatience = 100,
            playerPatience = 100,
            relationshipImpact = 0,
            startedDate = ctx.currentDate.toString(),
            lastUpdatedDate = ctx.currentDate.toString()
        ))

        return OfferSubmitResult.Success(offerId)
    }

    /**
     * 自由签约无卖方，跳过卖方评估直接进入合同谈判

    private fun validateOfferTypeForPlayer(

            OfferType.PRE_CONTRACT -> require(monthsUntilContractEnd(player.contractUntil, currentDate) <= 6)
            OfferType.LOAN, OfferType.LOAN_WITH_BUYOUT -> require(player.currentClubId != null)
            OfferType.PERMANENT -> require(player.currentClubId != null)
        }
    }
```

```kotlin
// seller/SellerClubEvaluator.kt
class SellerClubEvaluator(
    private val priceCalculator: PsychologicalPriceCalculator,
    private val counterOfferGenerator: CounterOfferGenerator,
    private val saveDb: SaveDatabase,
    private val economyService: EconomyService,
    private val config: NegotiationConfig
) {

    /**
     * V0.1 09 §四 卖方俱乐部评估
     * 决策树：接受 / 拒绝 / 还价
     */
    suspend fun evaluate(
        ctx: NegotiationContext,
        offer: TransferOfferEntity
    ): SellerDecision {
        val player = saveDb.savePlayerStateDao().getByIdWithAttributes(ctx.saveId, offer.playerId)
        val sellerClub = offer.toClubId?.let { saveDb.saveClubStateDao().getByClub(ctx.saveId, it) }
            ?: return SellerDecision.AcceptDirectly  // 无卖方（自由签约）

        // 1. 计算心理价位
        val psychologicalPrice = priceCalculator.calculate(ctx, player, sellerClub)

        // 2. 计算 6 因子接受概率
        val acceptanceProbability = calculateAcceptanceProbability(
            ctx, offer, player, sellerClub, psychologicalPrice
        )

        // 3. 决策树
        return when {
            acceptanceProbability >= config.acceptThreshold -> {
                // 高概率直接接受
                SellerDecision.Accept(psychologicalPrice, acceptanceProbability)
            }
            acceptanceProbability <= config.rejectThreshold -> {
                // 低概率：检查是否直接拒绝还是给一次还价
                if (offer.currentRound >= config.maxNegotiationRounds - 1) {
                    SellerDecision.Reject("报价与心理价位差距过大")
                } else {
                    val counter = counterOfferGenerator.generate(
                        ctx, offer, psychologicalPrice, sellerClub
                    )
                    SellerDecision.Counter(counter, acceptanceProbability)
                }
            }
            else -> {
                // 中间区间：还价
                val counter = counterOfferGenerator.generate(
                    ctx, offer, psychologicalPrice, sellerClub
                )
                SellerDecision.Counter(counter, acceptanceProbability)
            }
        }
    }

    /**
     * V0.1 09 §四 接受概率公式（严格按文档）
     * 接受概率 =
     *   报价/心理价位 × 0.35
     * + 球员合同剩余影响 × 0.15
     * + 球员离队意愿 × 0.15
     * + 俱乐部财政压力 × 0.15
     * + 球员重要性反向 × 0.15
     * + 俱乐部关系 × 0.05
     */
    private suspend fun calculateAcceptanceProbability(
        ctx: NegotiationContext,
        offer: TransferOfferEntity,
        player: SavePlayerStateWithAttributes,
        sellerClub: SaveClubStateEntity,
        psychologicalPrice: Int
    ): Double {
        // 因子 1：报价/心理价位（0-1.5 clamp）
        val priceRatio = (offer.transferFee.toDouble() / psychologicalPrice.coerceAtLeast(1))
            .coerceIn(0.0, 1.5)

        // 因子 2：合同剩余影响（剩余越短越想卖）
        val contractRemainFactor = calculateContractRemainFactor(player, ctx.currentDate)

        // 因子 3：球员离队意愿（0-1，越高越想卖）
        val leaveDesire = calculateLeaveDesire(player, sellerClub, ctx)

        // 因子 4：俱乐部财政压力（0-1，越高越想卖）
        val financialPressure = calculateFinancialPressure(sellerClub, ctx)

        // 因子 5：球员重要性反向（越重要越不想卖，反向归一）
        val importanceReverse = calculateImportanceReverse(player, sellerClub, ctx)

        // 因子 6：俱乐部关系（0-1，关系好更易成交）
        val clubRelation = calculateClubRelation(ctx.buyerClubId, offer.toClubId!!, ctx)

        val probability = (
            priceRatio * 0.35 +
            contractRemainFactor * 0.15 +
            leaveDesire * 0.15 +
            financialPressure * 0.15 +
            importanceReverse * 0.15 +
            clubRelation * 0.05
        )

        return probability.coerceIn(0.0, 1.0)
    }

    private fun calculateContractRemainFactor(
        player: SavePlayerStateWithAttributes,
        currentDate: LocalDate
    ): Double {
        val months = monthsUntilContractEnd(player.contractUntil, currentDate)
        return when {
            months <= 6 -> 1.0    // 半年到期，强烈想卖
            months <= 12 -> 0.8
            months <= 24 -> 0.5
            else -> 0.2           // 长合同不急
        }
    }

    private suspend fun calculateLeaveDesire(
        player: SavePlayerStateWithAttributes,
        sellerClub: SaveClubStateEntity,
        ctx: NegotiationContext
    ): Double {
        // 基于球员士气、上场时间、野心
        val moraleFactor = when {
            player.morale < 30 -> 0.9
            player.morale < 50 -> 0.6
            player.morale < 70 -> 0.3
            else -> 0.1
        }
        val playtimeFactor = if (player.squadRole in listOf("backup", "prospect")) 0.6 else 0.1
        return (moraleFactor * 0.6 + playtimeFactor * 0.4).coerceIn(0.0, 1.0)
    }

    private fun calculateFinancialPressure(
        sellerClub: SaveClubStateEntity,
        ctx: NegotiationContext
    ): Double {
        val ratio = sellerClub.wageToIncomeRatio
        return when {
            ratio > 0.85 -> 1.0
            ratio > 0.70 -> 0.6
            ratio > 0.55 -> 0.3
            else -> 0.1
        }
    }

    private suspend fun calculateImportanceReverse(
        player: SavePlayerStateWithAttributes,
        sellerClub: SaveClubStateEntity,
        ctx: NegotiationContext
    ): Double {
        val importance = PlayerImportanceResolver.resolve(player, sellerClub)
        // 反向：越重要越不想卖
        return when (importance) {
            PlayerImportance.KEY -> 0.0       // 关键球员很难卖
            PlayerImportance.STARTER -> 0.2   // 主力较难卖
            PlayerImportance.ROTATION -> 0.5  // 轮换可谈
            PlayerImportance.BACKUP -> 0.8    // 替补容易卖
            PlayerImportance.LISTED -> 1.0    // 挂牌低价可卖
        }
    }

    private suspend fun calculateClubRelation(
        buyerClubId: Int,
        sellerClubId: Int,
        ctx: NegotiationContext
    ): Double {
        return saveDb.clubRelationDao().getRelation(ctx.saveId, buyerClubId, sellerClubId)?.let {
            (it.relationValue / 100.0).coerceIn(0.0, 1.0)
        } ?: 0.5
    }
}

sealed class SellerDecision {
    object AcceptDirectly : SellerDecision()
    data class Accept(val psychologicalPrice: Int, val probability: Double) : SellerDecision()
    data class Reject(val reason: String) : SellerDecision()
    data class Counter(val counter: CounterOffer, val probability: Double) : SellerDecision()
}
```

```kotlin
    private val economyService: EconomyService,        // T17 提供市场价值

    /**
     * V0.1 09 §四.1 心理价位
     * 心理价位 = 市场价值 × 球员重要性系数 × 合同系数 × 潜力系数
     */
    suspend fun calculate(
        ctx: NegotiationContext,
        player: SavePlayerStateWithAttributes,
        sellerClub: SaveClubStateEntity
    ): Int {
        // 1. 市场价值（T17 计算）
        val marketValue = economyService.estimateMarketValue(player)

        // 2. 球员重要性系数
        val importance = PlayerImportanceResolver.resolve(player, sellerClub)
        val importanceMultiplier = config.importancePriceMultiplier[importance]!!

        // 3. 合同系数（剩余越长，卖方越敢要价）
        val contractMultiplier = calculateContractMultiplier(player, ctx.currentDate)

        // 4. 潜力系数（高潜年轻球员加价）
        val potentialMultiplier = calculatePotentialMultiplier(player)

        val psychologicalPrice = (marketValue *
            importanceMultiplier *
            contractMultiplier *
            potentialMultiplier).toInt()

        return psychologicalPrice.coerceAtLeast(config.minTransferFee)
    }

    /**
     * 合同系数（与身价修正方向一致但卖方视角更激进）
     */
    private fun calculateContractMultiplier(
        player: SavePlayerStateWithAttributes,
        currentDate: LocalDate
    ): Double {
        val months = monthsUntilContractEnd(player.contractUntil, currentDate)
        return when {
            months <= 6 -> 0.5    // 半年到期大幅降价
            months <= 12 -> 0.75
            months <= 24 -> 1.0
            months <= 36 -> 1.15
            else -> 1.25          // 长合同敢要价
        }
    }

    /**
     * 潜力系数
     */
    private fun calculatePotentialMultiplier(player: SavePlayerStateWithAttributes): Double {
        val age = calculateAge(player.birthDate)
        val potentialGap = player.potentialPa - player.currentCa
        return when {
            age <= 19 && potentialGap > 20 -> 1.40   // 高潜小妖加价 40%
            age <= 22 && potentialGap > 15 -> 1.20
            age <= 24 && potentialGap > 10 -> 1.10
            else -> 1.0
        }
    }
}
```

### 5. ContractNegotiator（合同谈判）

```kotlin
// contract/ContractNegotiator.kt
class ContractNegotiator(
    private val saveDb: SaveDatabase,
    private val economyService: EconomyService,
    private val agentNegotiator: AgentNegotiator,
    private val config: NegotiationConfig
) {

    /**
     * V0.1 09 §六 合同谈判
     * 玩家出价 → 经纪人反应 → 球员决定
     */
    suspend fun negotiateRound(
        ctx: NegotiationContext,
        offer: TransferOfferEntity,
        playerTerms: ContractTermsEntity
    ): ContractRoundResult {
        val player = saveDb.savePlayerStateDao().getByIdWithAttributes(ctx.saveId, offer.playerId)

        // 1. 经纪人评估条款（贪婪度/谈判能力影响）
        val agentReaction = agentNegotiator.evaluateTerms(ctx, offer, playerTerms, player)

        // 2. 球员综合评估（工资 + 角色 + 特殊条款）
        val playerScore = scoreTermsForPlayer(playerTerms, player, ctx)

        return when {
            agentReaction.accepted && playerScore >= config.playerAcceptThreshold -> {
                ContractRoundResult.Accepted(playerTerms)
            }
            agentReaction.walkAway || playerScore < config.playerRejectThreshold -> {
                ContractRoundResult.Rejected(agentReaction.message)
            }
            else -> {
                // 还价：生成经纪人/球员的还价条款
                val counterTerms = generateCounterTerms(playerTerms, agentReaction, player, ctx)
                ContractRoundResult.Counter(counterTerms, agentReaction.message)
            }
        }
    }

    private fun scoreTermsForPlayer(
        terms: ContractTermsEntity,
        player: SavePlayerStateWithAttributes,
        ctx: NegotiationContext
    ): Double {
        val expectedWage = economyService.calculateExpectedWage(player)
        // 工资得分（0-1.5）
        val wageScore = (terms.weeklyWage.toDouble() / expectedWage.coerceAtLeast(1))
            .coerceIn(0.0, 1.5) * 0.40

        // 年限得分（球员偏好 3-5 年）
        val yearsScore = when (terms.contractYears) {
            in 3..5 -> 1.0
            in 2..6 -> 0.7
            else -> 0.4
        } * 0.20

        // 角色承诺得分
        val roleScore = when (terms.rolePromise) {
            RolePromise.KEY_PLAYER -> 1.0
            RolePromise.STARTER -> 0.9
            RolePromise.ROTATION -> 0.6
            RolePromise.BACKUP -> 0.4
            RolePromise.ACADEMY_DEV -> 0.3
        } * 0.20

        // 签字费得分
        val bonusScore = (terms.signingBonus.toDouble() /
            (expectedWage * 4).coerceAtLeast(1)).coerceIn(0.0, 1.0) * 0.10

        // 特殊条款得分（解约金/涨薪对球员有利）
        val specialScore = scoreSpecialTerms(terms) * 0.10

        return (wageScore + yearsScore + roleScore + bonusScore + specialScore).coerceIn(0.0, 1.5)
    }

    private fun scoreSpecialTerms(terms: ContractTermsEntity): Double {
        var score = 0.0
        // 解约金对球员有利（可离队）
        terms.releaseClause?.let { score += 0.25 }
        // 降级解约金
        terms.relegationReleaseClause?.let { score += 0.15 }
        // 欧冠涨薪
        terms.uclRaisePercent?.let { score += 0.15 }
        // 年度涨薪
        terms.annualRaisePercent?.let { score += 0.15 }
        // 续约选项
        if (terms.contractExtensionOption) score += 0.10
        // 回购（对球员略不利，卖方保留）
        terms.buybackClause?.let { score -= 0.10 }
        // 二次转会分成（对球员有利）
        terms.sellOnPercent?.let { score += 0.10 }
        return score.coerceIn(0.0, 1.0)
    }

    private fun generateCounterTerms(
        playerTerms: ContractTermsEntity,
        agentReaction: AgentReaction,
        player: SavePlayerStateWithAttributes,
        ctx: NegotiationContext
    ): ContractTermsEntity {
        val expectedWage = economyService.calculateExpectedWage(player)
        // 还价工资：在玩家出价和期望工资之间，偏向期望工资上浮
        val counterWage = ((playerTerms.weeklyWage + expectedWage * (1 + agentReaction.greedAdjustment)) / 2)
            .toInt().coerceAtLeast(playerTerms.weeklyWage)

        return playerTerms.copy(
            weeklyWage = counterWage,
            signingBonus = (playerTerms.signingBonus * 1.2).toInt(),
            agentCommission = (playerTerms.agentCommission * 1.15).toInt()
        )
    }
}
```

```kotlin
class AgentNegotiator(
    private val historyDb: HistoryDatabase,
    private val saveDb: SaveDatabase,
    private val eventEngine: AgentEventEngine,
    private val config: NegotiationConfig
) {

    /**
     * V0.1 09 §七 经纪人评估合同条款
     * 5 种类型 + 8 项属性影响
     */
    suspend fun evaluateTerms(
        ctx: NegotiationContext,
        offer: TransferOfferEntity,
        terms: ContractTermsEntity,
        player: SavePlayerStateWithAttributes
    ): AgentReaction {
        val agent = offer.let {
            saveDb.negotiationSessionDao().getByOffer(ctx.saveId, it.offerId)?.agentId
        }?.let { historyDb.agentDao().getById(it) }
            ?: return AgentReaction(accepted = true, greedAdjustment = 0.0, message = "无经纪人，球员自行决定")

        // 1. 按经纪人类型计算期望工资上浮
        val greedAdjustment = when (AgentType.valueOf(agent.agentType)) {
            AgentType.GREEDY -> config.agentGreedMultiplier[AgentType.GREEDY]!!   // 默认 +0.20
            AgentType.STAR -> config.agentGreedMultiplier[AgentType.STAR]!!       // 默认 +0.15
            AgentType.DISRUPTIVE -> config.agentGreedMultiplier[AgentType.DISRUPTIVE]!! // +0.10
            AgentType.RELATIONAL -> config.agentGreedMultiplier[AgentType.RELATIONAL]!! // -0.05
            AgentType.ACADEMY -> config.agentGreedMultiplier[AgentType.ACADEMY]!!       // -0.05
        } + (agent.greed - 50) / 200.0  // 贪婪属性修正

        // 2. 评估佣金满意度
        val expectedCommission = (offer.transferFee * config.agentCommissionRate).toInt()
        val commissionRatio = terms.agentCommission.toDouble() / expectedCommission.coerceAtLeast(1)

        // 3. 关系值修正
        val relation = saveDb.agentPlayerRelationDao()
            .get(ctx.saveId, agent.agentId, ctx.buyerClubId)
        val relationBonus = relation?.let { (it.relationshipValue - 50) / 200.0 } ?: 0.0

        // 4. 谈判能力影响（高谈判能力更难满足）
        val negotiationPenalty = (agent.negotiationSkill - 50) / 300.0

        // 5. 综合满意度
        val satisfaction = (commissionRatio + relationBonus - negotiationPenalty).coerceIn(0.0, 1.5)

        // 6. 触发经纪人事件
        val event = eventEngine.tryTriggerEvent(ctx, agent, offer, terms, satisfaction)

        return when {
            satisfaction >= config.agentAcceptThreshold -> AgentReaction(
                accepted = true,
                greedAdjustment = greedAdjustment,
                message = "经纪人 ${agent.agentName} 对条款满意",
                triggeredEvent = event
            )
            satisfaction <= config.agentRejectThreshold && agent.disruptionTendency > 60 -> AgentReaction(
                accepted = false,
                walkAway = true,
                greedAdjustment = greedAdjustment,
                message = "经纪人 ${agent.agentName} 谈判破裂，威胁离队",
                triggeredEvent = event
            )
            else -> AgentReaction(
                accepted = false,
                greedAdjustment = greedAdjustment,
                message = "经纪人 ${agent.agentName} 要求提高工资和佣金",
                triggeredEvent = event
            )
        }
    }
}
```

```kotlin
        val baseFailRate = config.medicalBaseFailRate  // 默认 0.03

        val ageBonus = if (calculateAge(player.birthDate) > 32) 0.05 else 0.0
        val recentInjuryBonus = if (player.hasRecentSeriousInjury) 0.10 else 0.0

        val failProbability = (baseFailRate + injuryRiskBonus + ageBonus + recentInjuryBonus)
            .coerceIn(0.01, 0.25)

        // 2. 掷骰
        val roll = Random.nextDouble()
        val result = when {
            roll < failProbability -> {
                // 体检失败
                MedicalResult.FAILED
            }
            roll < failProbability + config.medicalConditionRate -> {
                // 带条件通过（需调整合同/降价）
                MedicalResult.PASSED_WITH_CONDITIONS
            }
            else -> MedicalResult.PASSED
        }

        val riskFlags = buildRiskFlags(player, result)

        // 4. 持久化
        val exam = MedicalExamEntity(
            medicalId = 0,
            saveId = ctx.saveId,
            offerId = offer.offerId,
            playerId = offer.playerId,
            result = result.name,
            failureReason = if (result == MedicalResult.FAILED) generateFailureReason(player) else null,
            riskFlags = riskFlags,
            conductedDate = ctx.currentDate.toString(),
            conductedByClubId = ctx.buyerClubId
        )
        saveDb.medicalExamDao().insert(exam)

        return exam
    }
```

```kotlin
        }

        // 2. 工资承担比例（0-100）
        if (loanTerms.wageSharePercent !in 0..100) {

        }

        // 3. 买断金额合理性（如设置）
        loanTerms.buyoutFee?.let { buyout ->
            val marketValue = economyService.estimateMarketValue(player)
            if (buyout < marketValue * 0.5) {
                return LoanValidationResult.Failed("买断金额过低（低于市值 50%）")
            }
        }

        // 4. 强制买断需有买断金额
        if (loanTerms.mandatoryBuyout && loanTerms.buyoutFee == null) {
            return LoanValidationResult.Failed("强制买断必须设置买断金额")
        }

        // 5. 年轻球员租借校验（U21 更易获批）
        val age = calculateAge(player.birthDate)
        if (age > 24 && !loanTerms.mandatoryBuyout) {
            return LoanValidationResult.Failed("24 岁以上球员租借需带强制买断")
        }

        return LoanValidationResult.Valid
    }

    /**
     * V0.1 09 §八.2 租借成长影响（严格按 8 因子公式）
     * 租借成长 =
     *   出场时间 × 0.30
     * + 联赛强度 × 0.20
     * + 教练水平 × 0.15
     * + 训练设施 × 0.10
     * + 战术适配 × 0.10
     * + 语言适应 × 0.05
     * + 士气 × 0.05
     * - 伤病风险 × 0.05
     */
    fun calculateLoanGrowth(
        playingTime: Double,        // 0-1，实际出场时间比例
        leagueStrength: Double,     // 0-1，租借联赛强度
        coachLevel: Int,            // 0-100，租借俱乐部教练水平
        trainingFacility: Int,      // 0-100，训练设施等级
        tacticalFit: Double,        // 0-1，战术适配度
        languageAdaptation: Double, // 0-1，语言适应
        morale: Int,                // 0-100，球员士气
        injuryRisk: Double          // 0-1，伤病风险
    ): Double {
        val growth = (
            playingTime * 0.30 +
            leagueStrength * 0.20 +
            (coachLevel / 100.0) * 0.15 +
            (trainingFacility / 100.0) * 0.10 +
            tacticalFit * 0.10 +
            languageAdaptation * 0.05 +
            (morale / 100.0) * 0.05 -
            injuryRisk * 0.05
        )
        return growth.coerceIn(-0.1, 1.0)
    }

    /**
     * 租借结束结算（由 T07 月度推进调用）
     */
    suspend fun settleLoanReturn(
        ctx: NegotiationContext,
        loanTerms: LoanTermsEntity,
        offer: TransferOfferEntity

        val loanClub = saveDb.saveClubStateDao().getByClub(ctx.saveId, offer.toClubId!!)

        // 1. 计算租借期间成长
        val playingTime = getActualPlayingTime(ctx.saveId, offer.playerId, offer.transferDate)
        val leagueStrength = getLeagueStrength(loanClub)
        val growth = calculateLoanGrowth(
            playingTime = playingTime,
            leagueStrength = leagueStrength,
            coachLevel = loanClub.coachLevel,
            trainingFacility = loanClub.trainingFacilityLevel,
            tacticalFit = 0.6,  // V1 简化
            languageAdaptation = 0.7,
            morale = player.morale,
            injuryRisk = player.injuryProneness / 100.0
        )

        // 2. 强制买断检查
        if (loanTerms.mandatoryBuyout && playingTime >= config.mandatoryBuyoutThreshold) {
            // 触发强制买断
            return LoanSettlementResult.MandatoryBuyoutTriggered(
                buyoutFee = loanTerms.buyoutFee!!,
                growth = growth
            )
        }

        // 3. 普通租借结束，球员返回
        saveDb.savePlayerStateDao().updateClub(
            offer.playerId, ctx.saveId, offer.fromClubId
        )

        return LoanSettlementResult.Returned(growth = growth)
    }
}
```

```text
function evaluateOffer(ctx, offer, player, sellerClub):
    if offer.toClubId == null:   # 自由签约无卖方
        return ACCEPT_DIRECTLY

    psychologicalPrice = calcPsychologicalPrice(marketValue, importance, contract, potential)
    probability = calcAcceptanceProbability(offer, player, sellerClub, psychologicalPrice)

    if probability >= ACCEPT_THRESHOLD (0.70):
        return ACCEPT(psychologicalPrice, probability)

    if probability <= REJECT_THRESHOLD (0.25):
        if offer.currentRound >= MAX_ROUNDS - 1:
            return REJECT("报价与心理价位差距过大")
        else:
            counter = generateCounterOffer(offer, psychologicalPrice)
            return COUNTER(counter, probability)

    # 中间区间 0.25-0.70：还价
    counter = generateCounterOffer(offer, psychologicalPrice)
    return COUNTER(counter, probability)


function calcAcceptanceProbability(offer, player, sellerClub, psychPrice):
    priceRatio = clamp(offer.transferFee / psychPrice, 0.0, 1.5)
    contractRemain = contractRemainFactor(player)          # 0.2-1.0
    leaveDesire = calcLeaveDesire(player, sellerClub)      # 0-1
    financialPressure = calcFinancialPressure(sellerClub)  # 0-1
    importanceReverse = importanceReverseFactor(player)    # 0-1（关键球员=0）
    clubRelation = getClubRelation(buyer, seller)          # 0-1

        priceRatio * 0.35 +
        contractRemain * 0.15 +
        leaveDesire * 0.15 +
        financialPressure * 0.15 +
        importanceReverse * 0.15 +
        clubRelation * 0.05,
        0.0, 1.0
```

```text
function calcPsychologicalPrice(player, sellerClub, currentDate):

    importance = resolveImportance(player, sellerClub)  # 5 档
    importanceMultiplier = IMPORTANCE_PRICE_MULTIPLIER[importance]

    contractMultiplier =
        if monthsLeft <= 6:  0.50
```

```text
function calcPlayerWillingness(ctx, offer, player, rolePromise):
    buyerClub = getClub(ctx.buyerClubId)
    expectedWage = economyService.calculateExpectedWage(player)

    clubReputation = buyerClub.reputation / 100.0                      # 0-1
    wageAttractiveness = clamp(offer.wage / expectedWage, 0.0, 1.5)    # 0-1.5
    playingChance = calcPlayingChance(rolePromise, player, buyerClub)  # 0-1
    euroQual = calcEuropeanQualification(buyerClub)                    # 0-1
    leagueAttract = LEAGUE_ATTRACTIVENESS[buyerClub.leagueId]          # 0-1
    ambition = player.ambition / 100.0                                 # 0-1
    adaptation = calcAdaptation(player.nationality, buyerClub.country) # 0-1
    agentRelation = getAgentRelation(player.agentId, ctx)              # 0-1

    return clamp(
        clubReputation * 0.20 +
        wageAttractiveness * 0.20 +
        playingChance * 0.15 +
        euroQual * 0.15 +
        leagueAttract * 0.10 +
        ambition * 0.10 +
        adaptation * 0.05 +
        agentRelation * 0.05,
        0.0, 1.0
    )
```

### 4. 合同谈判回合（玩家出价 → 经纪人反应 → 球员决定）

```text
function contractNegotiationRound(ctx, offer, playerTerms, player):
    # 1. 经纪人评估
    agent = getAgent(offer)
    if agent == null:
        agentReaction = ACCEPT  # 无经纪人
    else:
        greedAdj = AGENT_GREED[agent.type] + (agent.greed - 50) / 200
        expectedCommission = offer.transferFee * COMMISSION_RATE
        commissionRatio = playerTerms.agentCommission / expectedCommission
        relationBonus = (getRelation(agent) - 50) / 200
        negotiationPenalty = (agent.negotiationSkill - 50) / 300
        satisfaction = clamp(commissionRatio + relationBonus - negotiationPenalty, 0, 1.5)

        if satisfaction >= AGENT_ACCEPT_THRESHOLD: agentReaction = ACCEPT
        elif satisfaction <= AGENT_REJECT_THRESHOLD and agent.disruption > 60: agentReaction = WALK_AWAY
        else: agentReaction = COUNTER(greedAdj)

    # 2. 球员评估
    playerScore = scoreTermsForPlayer(playerTerms, player)
        # = wageScore*0.40 + yearsScore*0.20 + roleScore*0.20 + bonusScore*0.10 + specialScore*0.10

    # 3. 综合决策
    if agentReaction == ACCEPT and playerScore >= PLAYER_ACCEPT_THRESHOLD:
        return ACCEPTED(playerTerms)
    if agentReaction == WALK_AWAY or playerScore < PLAYER_REJECT_THRESHOLD:
        return REJECTED("谈判破裂")
    # 还价
    counterTerms = generateCounter(playerTerms, agentReaction, player)
    return COUNTER(counterTerms)
```

### 5. 还价算法（卖方/经纪人还价区间）

```text
function generateCounterOffer(offer, psychologicalPrice):
    # 卖方还价区间：[心理价位 × (1 - 让步幅度), 心理价位]
    roundsUsed = offer.currentRound
    concessionRate = COUNTER_CONCESSION_RATE * (1 + roundsUsed * 0.1)  # 越往后越让步
    lowerBound = psychologicalPrice * (1 - concessionRate)
    upperBound = psychologicalPrice

    # 还价 = 在区间内偏向心理价位（卖方想多要）
    counterFee = clamp(
        lerp(offer.transferFee, psychologicalPrice, COUNTER_LERP_FACTOR),
        lowerBound, upperBound
    )
    # 工资/佣金按比例调整
    counterWage = offer.wageOffer * 0.95   # 卖方不关心工资，略降
    return CounterOffer(fee=counterFee, wage=counterWage, ...)


function generateAgentCounter(playerTerms, agentReaction, player):
    expectedWage = economyService.calculateExpectedWage(player)
    # 经纪人还价：玩家出价与期望工资(上浮贪婪)的中间值
    targetWage = expectedWage * (1 + agentReaction.greedAdjustment)
    counterWage = (playerTerms.weeklyWage + targetWage) / 2
    counterBonus = playerTerms.signingBonus * 1.2
    counterCommission = playerTerms.agentCommission * 1.15
    return playerTerms.copy(wage=counterWage, bonus=counterBonus, commission=counterCommission)
```

## 七、交互流程

### 1. 完整 12 步转会流程状态机

```text
[1.搜索球员(T10)] → [2.球探报告(T14)] → [3.提交报价(T11)]
    ↓
[4.卖方评估]
    ├── 接受 → [5.卖方接受]
    ├── 拒绝 → [结束：转会失败]
    └── 还价 → [玩家选择]
        ├── 接受还价 → [5.卖方接受]
        ├── 修改重报 → [4.卖方评估]（回到评估，轮次+1）
        └── 撤回 → [结束：转会撤回]
    ↓
[5.卖方接受] → [6.玩家确认] → [7.经纪人介入]
    ↓
[8.合同谈判]
    ├── 接受 → [9.体检]
    ├── 还价 → [玩家选择]（循环）
    └── 破裂 → [结束：谈判破裂，关系受损]
    ↓
[9.体检]
    ├── 通过 → [10.转会完成]
    ├── 带条件 → [玩家选择：调整合同/放弃]
    └── 失败 → [结束：转会流产]
    ↓
[10.转会完成]
    ├── 更新球员俱乐部/合同
    ├── 扣减/增加预算
    ├── 触发 T15 提前签约打断
    └── 触发 T20 蝴蝶效应（如打断）
    ↓
[11.新闻发布] → [12.阵容与历史路径更新]
```

### 2. 卖方评估流程

```text
提交报价
    ↓
计算心理价位（市场价值 × 重要性 × 合同 × 潜力）
    ↓
计算 6 因子接受概率
    ↓
┌─ 概率 ≥ 0.70 ──→ 卖方接受 → 进入球员谈判
│
├─ 概率 ≤ 0.25 ──→ 检查轮次
│                  ├─ 已达上限 → 拒绝（结束）
│                  └─ 未达上限 → 生成还价
│
└─ 0.25 < 概率 < 0.70 ──→ 生成还价
                            ↓
                        玩家选择
                            ├─ 接受还价 → 卖方接受
                            ├─ 修改重报 → 轮次+1，重新评估
                            └─ 撤回 → 结束
```

### 3. 合同谈判回合流程

```text
卖方接受后
    ↓
经纪人介入（如有）
    ↓
玩家提出合同条款（9 基础 + 7 特殊 + 角色）
    ↓
经纪人评估（贪婪度/佣金/关系）
    ├─ 满意 → 球员评估
    ├─ 不满意且搅局 → 谈判破裂
    └─ 不满意 → 生成经纪人还价
    ↓
球员评估（工资/年限/角色/特殊条款）
    ├─ 满意 → 合同达成 → 体检
    ├─ 极不满 → 拒绝（结束）
    └─ 中间 → 生成球员还价
    ↓
玩家选择
    ├─ 接受还价 → 合同达成 → 体检
    ├─ 修改重报 → 重新评估（轮次+1）
    └─ 撤回 → 谈判破裂（关系受损）
```

### 4. 还价流程

```text
卖方/经纪人发起还价
    ↓
展示还价条款（转会费/工资/佣金/特殊条款）
    ↓
玩家查看还价 vs 自己的预算/期望
    ↓
玩家选择
    ├─ 接受还价 → 按还价条款继续
    ├─ 修改后重报 → 在还价基础上调整，提交新轮次
    └─ 拒绝还价 → 谈判破裂或撤回
```

### 5. 谈判破裂流程

```text
触发条件：
- 任一方耐心归零
- 经纪人 walkAway（搅局型 + 满意度极低）
- 玩家主动撤回
- 体检失败

破裂后果：
1. 报价状态置为 COLLAPSED 或 WITHDRAWN
2. 球员关系值 -10（如玩家主动撤回且球员意愿高）
3. 经纪人关系值 -5（如玩家多次撤回）
4. 卖方俱乐部关系值 -3（如谈判拖沓）
5. 球员可能在下一个转会窗降低对玩家俱乐部的意愿
6. 生成"转会流产"新闻
```

## 九、性能要求

| 指标 | 目标 | 说明 |
|------|------|------|
| 报价评估响应 | ≤200ms | 单次卖方评估（心理价位 + 6 因子概率 + 决策） |
| 谈判回合响应 | ≤500ms | 单轮合同谈判（经纪人评估 + 球员评估 + 还价生成） |
| 转会完成（含历史更新） | ≤1s | 含球员俱乐部更新 + 预算结算 + T15 打断 + T20 蝴蝶 + 新闻 + 记录 |

| 体检执行 | ≤50ms | 概率计算 + 持久化 |
| 谈判会话加载 | ≤200ms | 多轮历史加载 |

**性能保障措施**：
- 心理价位/期望工资在报价提交时缓存到 NegotiationSession，避免每轮重算。
- 球员俱乐部/合同更新走单事务，确保原子性。
- T15/T20 触发为异步事件，不阻塞主流程（先完成转会，再触发蝴蝶）。
- OfferRound 历史列表分页加载（>10 轮时）。

## 十四、风险与应对

### 1. 谈判死循环

- **表现**：玩家与卖方无限还价，无法达成
- **应对**：
  - maxNegotiationRounds = 5，达到上限后卖方不再还价（直接拒绝）
  - 每轮扣减耐心，耐心归零自动破裂
  - 报价有效期 7 天，过期自动撤回

### 2. 转会费/工资失衡

- **表现**：玩家用低价签高价值球员，或经纪人要价天价
- **应对**：
  - 心理价位 = 市场价值 × 重要性系数（关键球员 ×1.5），防止低价强挖
  - 经纪人贪婪度上浮有上限（+0.20）
  - 工资/佣金参考 T17 期望工资，偏离过大触发还价

### 3. T15/T20 集成阻塞

- **表现**：T15/T20 未实现导致转会完成卡住
- **应对**：
  - 通过接口契约（HistoricalProspectService/ButterflyEventService）解耦
  - T20 未实现时注入 no-op 实现，转会先完成，蝴蝶后补
  - T15 已在 W53-W55 实现，但 T11（W42-W46）早于 T15，故 T11 阶段先用 mock

### 4. 体检失败玩家体验差

- **表现**：谈了很久体检失败，玩家挫败
- **应对**：
  - 基础失败率仅 3%，仅高风险球员（伤病倾向高/老将/重伤史）失败率高
  - 带条件通过（PASSED_WITH_CONDITIONS）允许玩家调整合同继续
  - 失败后球员关系不受损（非玩家责任）

### 5. 多轮谈判性能退化

- **表现**：第 5 轮谈判响应 >500ms
- **应对**：
  - 心理价位/期望工资在会话开始时缓存
  - OfferRound 历史分页加载
  - 同位置竞争查询结果缓存

### 6. 经纪人事件过频

- **表现**：每轮谈判都触发经纪人事件，干扰体验
- **应对**：
  - eventTriggerRate = 0.15（15% 概率触发）
  - 同一谈判会话同一事件不重复触发
  - 关系型/青训型事件触发率减半
