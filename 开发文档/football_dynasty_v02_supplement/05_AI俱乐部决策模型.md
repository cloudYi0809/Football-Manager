# 05_AI 俱乐部决策模型

## 一、设计目标

AI 俱乐部不需要一开始做到人类级别，但必须让世界长期模拟后仍然“像足球”。

AI 需要解决：

1. 什么时候买人。
2. 买什么位置。
3. 出多少钱。
4. 是否卖人。
5. 是否续约。
6. 是否换教练。
7. 是否培养年轻人。
8. 如何在预算内做选择。

## 二、AI 俱乐部画像

每家俱乐部需要一个 club_ai_profile。

字段：

- ambition：野心
- financial_power：财力
- youth_preference：青训偏好
- star_preference：巨星偏好
- resale_preference：低买高卖偏好
- domestic_preference：本土球员偏好
- tactical_identity：战术倾向
- risk_tolerance：风险容忍
- wage_strictness：工资控制
- patience_with_manager：主帅耐心

## 三、俱乐部类型

### 1. 豪门争冠型

行为：

- 偏好高 CA 球星。
- 愿意支付溢价。
- 欧冠成绩差会换帅。
- 不容易卖核心。

### 2. 青训培养型

行为：

- 偏好 16-22 岁高潜力。
- 愿意给年轻人出场。
- 卖人获得收入后继续投资青训。

### 3. 黑店型

行为：

- 挖低价小妖。
- 培养后高价出售。
- 设置较高违约金或转售分成。

### 4. 保级务实型

行为：

- 偏好即战力和低工资球员。
- 多签自由球员、租借球员。
- 不轻易高价买小妖。

### 5. 金元扩张型

行为：

- 短期大量买入球星。
- 工资溢价明显。
- 更看重商业价值和声望。

## 四、阵容短板识别

每月和转会窗口前，AI 执行 squad_needs_analysis。

### 1. 位置深度检查

每个位置需要：

- 主力数量
- 替补数量
- 青年储备
- 伤病覆盖

```text
position_need_score =
  starter_gap * 0.40
+ backup_gap * 0.20
+ average_age_risk * 0.15
+ injury_risk * 0.10
+ contract_expiry_risk * 0.10
+ tactical_importance * 0.05
```

### 2. 年龄结构检查

风险：

- 主力平均年龄过高。
- 某位置全是年轻人。
- 核心球员合同快到期。
- 老将下滑无替代者。

### 3. 战术适配检查

AI 主帅战术需要特定位置和属性。

例如：

- 高位压迫需要耐力、工作率、速度。
- 控球需要传球、技术、视野。
- 防守反击需要速度、强壮、门将。

## 五、转会目标评分

AI 对候选球员计算 target_score。

```text
target_score =
  position_need * 0.25
+ current_ability_fit * 0.20
+ potential_fit * 0.15
+ price_value * 0.15
+ wage_affordability * 0.10
+ age_fit * 0.05
+ tactical_fit * 0.05
+ nationality_fit * 0.03
+ commercial_value * 0.02
```

不同俱乐部类型调整权重：

- 豪门：current_ability_fit 更高。
- 青训型：potential_fit 更高。
- 黑店型：price_value 和 resale 更高。
- 保级型：wage_affordability 更高。
- 金元型：commercial_value 和 reputation 更高。

## 六、报价逻辑

### 1. 估价

```text
max_offer = market_value
  * club_need_multiplier
  * player_reputation_multiplier
  * contract_remaining_multiplier
  * selling_club_pressure_multiplier
  * competition_multiplier
```

### 2. 预算限制

```text
if max_offer > transfer_budget_remaining:
    max_offer = transfer_budget_remaining * risk_tolerance
```

### 3. 报价策略

| 俱乐部类型 | 初始报价 | 加价次数 |
|---|---:|---:|
| 豪门 | 90% 市值 | 3 |
| 黑店 | 60% 市值 | 2 |
| 保级队 | 70% 市值 | 1 |
| 金元队 | 110% 市值 | 4 |

## 七、卖人逻辑

AI 决定出售球员时考虑：

```text
sell_score =
  offer_value_ratio * 0.25
+ player_unhappy * 0.20
+ contract_expiry_risk * 0.15
+ squad_depth_cover * 0.15
+ financial_pressure * 0.15
+ age_decline_risk * 0.10
```

达到阈值即可出售。

特殊规则：

- 队长、核心球员、青训旗帜默认 sell_score 降低。
- 财政危机时 sell_score 提高。
- 球员强烈要求转会时 sell_score 提高。

## 八、续约逻辑

AI 每月检查合同。

```text
renew_priority =
  current_ability * 0.25
+ potential * 0.20
+ squad_importance * 0.20
+ resale_value * 0.10
+ loyalty * 0.10
+ age_factor * 0.10
+ wage_reasonableness * 0.05
```

处理：

- 高优先级：提前 18-24 个月续约。
- 中优先级：提前 12 个月续约。
- 低优先级：合同到期放走或挂牌。

## 九、换帅逻辑

```text
manager_sack_score =
  board_expectation_gap * 0.35
+ recent_form_bad * 0.20
+ dressing_room_unrest * 0.15
+ fan_pressure * 0.10
+ financial_underperformance * 0.10
+ derby_failure * 0.05
+ european_failure * 0.05
```

董事会耐心越低，阈值越低。

## 十、AI 计算频率

为控制性能：

| AI 行为 | 频率 |
|---|---|
| 阵容短板分析 | 每月一次，转会窗每周一次 |
| 球员续约检查 | 每月一次 |
| 转会目标搜索 | 转会窗每周一次 |
| 报价谈判 | 转会窗每日处理少量 |
| 换帅检查 | 每月一次，连败后触发 |
| 青训提拔 | 每赛季 2 次 |
| 战术调整 | 每 5 场或换帅后 |

## 十一、防止世界崩坏的约束

1. 每个转会窗口限制 AI 俱乐部最大交易数量。
2. 俱乐部工资不得长期超过收入安全线。
3. 豪门不会无理由卖出全部核心。
4. 保级队不会买入远超预算的球星。
5. 年轻天才不会全部流向同一家豪门。
6. 位置短板优先级必须高于囤积同类球员。
7. AI 每次大额交易后重新计算预算。

## 十二、调试输出

开发版在每笔 AI 转会后记录：

- 购买原因
- 目标评分
- 预算状态
- 位置需求
- 替代目标
- 失败原因

用于调参。
