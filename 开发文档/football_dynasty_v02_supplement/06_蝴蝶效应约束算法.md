# 06_蝴蝶效应约束算法

## 一、设计目标

蝴蝶效应不是无限模拟整个世界的因果链，而是在有限预算内生成可信替代历史。

目标：

1. 玩家改变关键节点后，世界有反应。
2. 反应要可信，不荒谬。
3. 反应不能无限扩散。
4. 计算量可控。
5. 结果可解释给玩家。

## 二、核心原则

### 1. 只处理关键节点

不是所有转会都触发蝴蝶效应。

触发条件：

- 历史新星被提前签下。
- 真实关键转会失败。
- 顶级球员伤病命运大幅改变。
- 豪门主帅提前离任。
- 重大资本事件被改变。
- 国家队核心球员路径改变。
- 欧冠/世界杯关键结果改变。

### 2. 影响逐层衰减

每个事件最多扩散 3 层。

```text
第一层：直接受影响俱乐部/球员
第二层：替代转会对象、竞争俱乐部
第三层：长期声望、国家队、奖项微调
```

第三层之后不再继续扩散，只写入世界趋势修正。

### 3. 使用影响预算

每个赛季最多处理固定数量蝴蝶事件。

建议：

```text
每赛季重大蝴蝶事件 ≤ 20
每个事件最多生成 5 条直接影响
每条直接影响最多生成 3 条二级影响
每赛季总影响处理节点 ≤ 200
```

## 三、事件结构

```json
{
  "event_id": "butterfly_cr7_not_to_manutd_2003",
  "trigger_type": "historical_transfer_broken",
  "source_player_id": 1007,
  "source_club_id": "SportingCP",
  "expected_club_id": "ManchesterUnited",
  "trigger_date": "2003-08-01",
  "importance": 95,
  "impact_budget": 100,
  "max_depth": 3,
  "status": "pending"
}
```

## 四、影响类型

### 1. Transfer Replacement：替代转会

例如 C 罗未去曼联，曼联寻找边锋替代。

规则：

- 从同位置、同年龄层或即战力候选池中搜索。
- 按 AI 俱乐部决策模型选择。
- 不强制指定唯一结果。
- 允许“未找到合适目标”。

### 2. Financial Impact：财政影响

例如葡萄牙体育未收到转会费。

影响：

- 当前预算降低。
- 后续转会能力下降。
- 财政压力上升。

### 3. Career Path Shift：球员成长路径改变

例如小妖去了不同联赛。

影响：

- 联赛强度不同。
- 出场时间不同。
- 训练设施不同。
- 成长曲线可能改变。
- 伤病风险可能改变。

### 4. Club Strategy Shift：俱乐部策略变化

例如巴萨失去梅西。

影响：

- 增加青训投入。
- 转向买入其他攻击手。
- 战术核心改变。
- 未来声望变化。

### 5. National Team Shift：国家队路线变化

影响：

- 国家队战术核心改变。
- 大赛成绩概率调整。
- 球员声望和奖项概率调整。

## 五、影响衰减公式

```text
impact_strength(depth) = base_importance * decay_rate ^ depth
```

默认：

```text
decay_rate = 0.55
```

例：

- 深度 0：100
- 深度 1：55
- 深度 2：30
- 深度 3：16

低于 15 的影响不生成具体事件，只记录为趋势修正。

## 六、替代决策池

每个关键历史转会需要提前定义 replacement_pool_rule。

示例：C 罗未去曼联。

```json
{
  "event": "cr7_manutd_2003_failed",
  "affected_club": "ManchesterUnited",
  "need": {
    "position": ["RW", "LW", "AMR"],
    "age_min": 18,
    "age_max": 25,
    "ca_min": 70,
    "pa_min": 82,
    "league_scope": ["Portugal", "Netherlands", "France", "England", "Brazil"]
  },
  "fallback_actions": [
    "promote_youth",
    "loan_player",
    "delay_transfer_to_winter",
    "buy_lower_reputation_winger"
  ]
}
```

## 七、回退机制

如果替代逻辑失败：

1. 降低搜索标准。
2. 选择租借。
3. 提拔青训。
4. 维持现有阵容。
5. 记录“历史分歧未产生重大替代”。

不允许为了蝴蝶效应强行制造荒谬转会。

## 八、玩家可见反馈

蝴蝶效应不需要全部展示，只展示重大节点。

示例：

```text
由于 C 罗没有加盟曼联，曼联在夏窗转向寻找其他年轻边锋。
葡萄牙体育缺少原本的转会收入，本赛季预算被下调。
C 罗在你的球队获得更多出场机会，他的成长路径已经偏离真实历史。
```

## 九、存档记录

必须记录：

- 哪个历史节点被打断。
- 原本路径是什么。
- 当前替代路径是什么。
- 已产生的影响。
- 是否已归档。

## 十、开发顺序

第一版只做三类蝴蝶：

1. 关键转会被打断。
2. 历史新星提前签约。
3. 豪门关键主帅提前离任。

后续再扩展国家队、奖项、资本事件。
