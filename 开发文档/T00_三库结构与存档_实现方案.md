# T00 三库结构与存档系统 - 详细实现方案

> 任务编号：T00（W1-W2）+ T03（W9-W11）合并细化
> 周期：W1-W2 三库结构（2 周）+ W9-W11 存档系统（3 周）= 共 5 周
> 优先级：P0（项目地基，A 轨最先启动）
> 依据：V0.2 `09_长程性能_存档_版本迁移.md` + V0.2 `14_增补SQL` + V0.1 `04_数据库设计` + V0.1 `05_database_schema.sql`
> 铁律：三库严格分离（history 只读 / save 可写 / cache 可重建），存档 schema_version 必须支持迁移

## 一、三库架构总览

### 1. 数据库分层

```
┌─────────────────────────────────────────────────────────────┐
│  history.db (只读)                                           │
│  随安装包/数据包提供，玩家不可修改                              │
│  ─────────────────────────────────────────                   │
│  player / club / season / competition / match               │
│  player_attributes / squad_membership / transfer_history   │
│  historical_prospect_pool / historical_event               │
│  scout / staff / agent / youth_academy                     │
│  data_pack_manifest（V0.2）                                 │
└─────────────────────────────────────────────────────────────┘
                          │
                          │ 外键只读引用
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  save_xxx.db (可写，每存档一个)                                │
│  玩家所有修改都在这里                                          │
│  ─────────────────────────────────────────                   │
│  save_manifest / save_world_state                          │
│  save_player_state / save_club_state                       │
│  save_news / save_injury / save_transfer_offer             │
│  scout_assignment / scout_report（按 save_id 隔离）          │
│  club_ai_profile / ai_decision_log（V0.2）                  │
│  butterfly_event / butterfly_impact_node（V0.2）            │
│  economy_index / league_economy_profile（V0.2，可放 history）│
│  season_archive / checkpoint / perf_log（V0.2）             │
└─────────────────────────────────────────────────────────────┘
                          │
                          │ 写入优化缓存
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  cache.db (可重建)                                            │
│  可随时删除重建，不影响存档                                     │
│  ─────────────────────────────────────────                   │
│  player_search_index / ranking_cache / stats_cache         │
│  image_path_cache / ft Virtual Table                       │
└─────────────────────────────────────────────────────────────┘
```

### 2. 三库职责边界

| 库 | 读写性 | 生命周期 | 体积控制 | 备份策略 |
|----|--------|----------|----------|----------|
| history.db | 只读 | 随版本更新 | 固定，约 5-15MB | 随 APK 提供 |
| save_xxx.db | 可写 | 玩家创建后持续增长 | 20 年 ≤80MB | checkpoint + 手动备份 |
| cache.db | 可写可删 | 随时重建 | 自动清理 | 无需备份 |

### 1. HistoryDatabase

```kotlin
// history/HistoryDatabase.kt
@Database(
    entities = [
        PlayerEntity::class,
        PlayerAttributesEntity::class,
        ClubEntity::class,
        SeasonEntity::class,
        CompetitionEntity::class,
        ClubCompetitionSeasonEntity::class,
        SquadMembershipEntity::class,
        TransferHistoryEntity::class,
        MatchEntity::class,
        HistoricalProspectPoolEntity::class,
        HistoricalEventEntity::class,
        ScoutEntity::class,
        ScoutRegionKnowledgeEntity::class,
        StaffEntity::class,
        AgentEntity::class,
        PlayerAgentEntity::class,
        YouthAcademyEntity::class,
        DataPackManifestEntity::class
    ],
    version = 1,
    exportSchema = true
)
abstract class HistoryDatabase : RoomDatabase() {
    abstract fun playerDao(): PlayerDao
    abstract fun clubDao(): ClubDao
    abstract fun seasonDao(): SeasonDao
    abstract fun matchDao(): MatchDao
    abstract fun prospectDao(): HistoricalProspectDao
    abstract fun dataPackDao(): DataPackManifestDao

    companion object {
        const val DATABASE_NAME = "history.db"

        fun create(context: Context, dbFile: File): HistoryDatabase {
            return Room.databaseBuilder(context, HistoryDatabase::class.java, dbFile.absolutePath)
                .setJournalMode(JournalMode.TRUNCATE)  // 只读库用 TRUNCATE 节省空间
                .addCallback(object : RoomDatabase.Callback() {
                    override fun onOpen(db: SupportSQLiteDatabase) {
                        super.onOpen(db)
                        db.execSQL("PRAGMA query_only = ON")  // 强制只读
                    }
                })
                .build()
        }
    }
}
```

### 3. 迁移流程（V0.2 §五）

```kotlin
// api/MigrationManager.kt
class MigrationManager(private val saveDb: SaveDatabase) {

    /**
     * 执行迁移（V0.2 §五 流程）
     * 1. 检测 schema_version
     * 2. 创建完整备份
     * 3. 校验剩余存储空间
     * 4. 执行迁移脚本
     * 5. 校验关键表完整性
     * 6. 失败回滚
     * 7. 成功后更新版本号
     */
    fun migrate(fromVersion: Int, toVersion: Int): MigrationResult {
        // 1. 创建备份（migration_backup checkpoint）
        val backupPath = CheckpointManager(saveDb).createCheckpoint(CheckpointType.MIGRATION)

        // 2. 校验存储空间（至少 100MB）
        if (getAvailableStorage() < 100 * 1024 * 1024) {
            return MigrationResult.Failed("存储空间不足，至少需要 100MB")
        }

        // 3. 逐版本执行迁移
        try {
            var currentVersion = fromVersion
            while (currentVersion < toVersion) {
                val migration = findMigration(currentVersion, currentVersion + 1)
                    ?: return MigrationResult.Incompatible(currentVersion, currentVersion + 1)

                when (migration.type) {
                    MigrationType.INCOMPATIBLE -> {
                        return MigrationResult.Incompatible(currentVersion, currentVersion + 1)
                    }
                    else -> {
                        saveDb.openHelper.writableDatabase.run {
                            beginTransaction()
                            try {
                                migration.executor.migrate(this)
                                setTransactionSuccessful()
                            } finally {
                                endTransaction()
                            }
                        }
                        currentVersion++
                    }
                }
            }

            // 4. 校验关键表完整性
            val integrity = SaveIntegrityChecker(saveDb).check()
            if (integrity.isSevere()) {
                // 5. 失败回滚
                CheckpointManager(saveDb).restoreFromCheckpoint(backupPath)
                return MigrationResult.Failed("迁移后完整性校验失败，已回滚：${integrity.summary}")
            }

            // 6. 更新 schema_version
            saveDb.saveManifestDao().updateSchemaVersion(toVersion)

            return MigrationResult.Success(fromVersion, toVersion)

        } catch (e: Exception) {
            // 回滚
            CheckpointManager(saveDb).restoreFromCheckpoint(backupPath)
            return MigrationResult.Failed("迁移异常，已回滚：${e.message}")
        }
    }

    private fun findMigration(from: Int, to: Int): MigrationStep? {
        // 查找注册表
        return null  // 简化
    }

    private fun getAvailableStorage(): Long {
        val stat = StatFs(Environment.getDataDirectory().path)
        return stat.availableBlocksLong * stat.blockSizeLong
    }
}

sealed class MigrationResult {
    data class Success(val from: Int, val to: Int) : MigrationResult()
    data class Failed(val reason: String) : MigrationResult()
    data class Incompatible(val from: Int, val to: Int) : MigrationResult()
}
```

```kotlin
// save/CheckpointManager.kt
class CheckpointManager(private val saveDb: SaveDatabase) {

    /**
     * 创建 checkpoint
     * V0.2 §六：
     * - light_checkpoint: 每 30 天，日期+关键状态+队伍表
     * - season_checkpoint: 每赛季，完整赛季结束状态
     * - migration_backup: 迁移前，完整备份
     * - user_backup: 用户手动，完整备份
     */
    fun createCheckpoint(type: CheckpointType): String {
        val checkpointId = UUID.randomUUID().toString()
        val now = LocalDateTime.now().format(ISO_LOCAL_DATE)
        val manifest = saveDb.saveManifestDao().get()

        // 1. 复制当前 save.db 到备份目录
        val backupDir = File(getCheckpointDir(), checkpointId)
        backupDir.mkdirs()
        val currentDbFile = saveDb.openHelper.writableDatabase.path?.let { File(it) }
        val backupDbFile = File(backupDir, "save_${manifest.saveId}.db")
        if (currentDbFile != null && currentDbFile.exists()) {
            currentDbFile.copyTo(backupDbFile, overwrite = true)
        }

        // 2. 计算 checksum
        val checksum = calculateChecksum(backupDbFile)

        // 3. 写入 checkpoint 记录
        saveDb.checkpointDao().insert(CheckpointEntity(
            checkpointId = checkpointId,
            saveId = manifest.saveId,
            checkpointType = type.name.lowercase(),
            checkpointDate = manifest.currentDate,
            filePath = backupDbFile.absolutePath,
            checksum = checksum
        ))

        // 4. 更新 save_manifest 的 last_checkpoint_id
        saveDb.saveManifestDao().updateLastCheckpoint(manifest.saveId, checkpointId)

        // 5. 清理过期 checkpoint
        cleanOldCheckpoints(type)

        return checkpointId
    }

    /**
     * 从 checkpoint 恢复
     */
    fun restoreFromCheckpoint(checkpointId: String): Boolean {
        val cp = saveDb.checkpointDao().getById(checkpointId) ?: return false
        val backupFile = File(cp.filePath)
        if (!backupFile.exists()) return false

        // 校验 checksum
        if (calculateChecksum(backupFile) != cp.checksum) return false

        // 关闭当前 db 连接，覆盖文件，重新打开
        saveDb.close()
        val currentDbFile = File(saveDb.openHelper.writableDatabase.path!!)
        backupFile.copyTo(currentDbFile, overwrite = true)

        return true
    }

    private fun cleanOldCheckpoints(type: CheckpointType) {
        val maxCount = when (type) {
            CheckpointType.LIGHT -> 4      // 保留最近 4 个 30 天 checkpoint
            CheckpointType.SEASON -> 5     // 保留最近 5 个赛季 checkpoint
            CheckpointType.MIGRATION -> 2  // 保留最近 2 个迁移备份
            CheckpointType.USER -> 10      // 用户备份保留 10 个
        }
        val oldCheckpoints = saveDb.checkpointDao().getOldByType(type.name.lowercase(), maxCount)
        oldCheckpoints.forEach { cp ->
            File(cp.filePath).delete()
            saveDb.checkpointDao().delete(cp)
        }
    }

    private fun calculateChecksum(file: File): String {
        return MessageDigest.getInstance("MD5").digest(file.readBytes()).joinToString("") { "%02x".format(it) }
    }

    private fun getCheckpointDir(): File {
        val dir = File(context.filesDir, "checkpoints")
        if (!dir.exists()) dir.mkdirs()
        return dir
    }
}

enum class CheckpointType { LIGHT, SEASON, MIGRATION, USER }
```

## 七、存档完整性检测（V0.2 §八）

```kotlin
// integrity/SaveIntegrityChecker.kt
class SaveIntegrityChecker(private val saveDb: SaveDatabase) {

    /**
     * V0.2 §八 检测项：
     * 1. manifest 是否存在
     * 2. 关键表是否存在
     * 3. save_world_state 是否只有一条当前状态
     * 4. 当前执教俱乐部是否存在
     * 5. 当前日期是否合法
     * 6. 球员状态外键是否完整
     * 7. 比赛表是否有孤儿记录
     */
    fun check(): IntegrityReport {
        val issues = mutableListOf<IntegrityIssue>()

        // 1. manifest
        val manifest = saveDb.saveManifestDao().get()
        if (manifest == null) {
            issues.add(IntegrityIssue(Severity.SEVERE, "save_manifest 不存在"))
            return IntegrityReport(issues)
        }

        // 2. 关键表（通过 Room schema 校验）
        val requiredTables = listOf(
            "save_manifest", "save_world_state", "save_player_state",
            "save_club_state", "save_injury", "save_transfer_offer"
        )
        val existingTables = saveDb.openHelper.readableDatabase.query("SELECT name FROM sqlite_master WHERE type='table'")
            .use { cursor ->
                generateSequence { if (cursor.moveToNext()) cursor.getString(0) else null }.toSet()
            }
        requiredTables.forEach { table ->
            if (table !in existingTables) {
                issues.add(IntegrityIssue(Severity.SEVERE, "关键表缺失：$table"))
            }
        }

        // 3. save_world_state 只有一条
        val worldStateCount = saveDb.saveWorldStateDao().count()
        if (worldStateCount == 0) {
            issues.add(IntegrityIssue(Severity.SEVERE, "save_world_state 为空"))
        } else if (worldStateCount > 1) {
            issues.add(IntegrityIssue(Severity.MEDIUM, "save_world_state 有 $worldStateCount 条，应为 1"))
        }

        // 4. 当前执教俱乐部存在
        val worldState = saveDb.saveWorldStateDao().get()
        if (worldState != null) {
            val managerClub = saveDb.saveClubStateDao().getByClub(worldState.save_id, worldState.managerClubId)
            if (managerClub == null) {
                issues.add(IntegrityIssue(Severity.SEVERE, "当前执教俱乐部状态不存在：${worldState.managerClubId}"))
            }
        }

        // 5. 当前日期合法
        if (manifest.currentDate != null) {
            try {
                LocalDate.parse(manifest.currentDate)
            } catch (e: Exception) {
                issues.add(IntegrityIssue(Severity.SEVERE, "当前日期非法：${manifest.currentDate}"))
            }
        }

        // 6. 球员状态外键（简化：检查 save_player_state 是否有 player_id 不在 history.player 中的）
        // 此项需要 historyDb 联合检查，此处跳过

        // 7. 孤儿比赛记录（简化）
        // 比赛表在 history.db 中，存档本身不存比赛，跳过

        return IntegrityReport(issues)
    }
}

data class IntegrityReport(val issues: List<IntegrityIssue>) {
    fun isSevere(): Boolean = issues.any { it.severity == Severity.SEVERE }
    fun hasMinorIssues(): Boolean = issues.any { it.severity == Severity.MINOR }
    val summary: String get() = issues.joinToString("; ") { "${it.severity}: ${it.description}" }
}

data class IntegrityIssue(val severity: Severity, val description: String)
```

```kotlin
class SaveRepairService(private val saveDb: SaveDatabase) {

    /**
     * V0.2 §九 三级修复
     */
    fun repairMinor(report: IntegrityReport) {
        report.issues.filter { it.severity == Severity.MINOR }.forEach { issue ->
            when {
                issue.description.contains("孤儿日志") -> {
                    saveDb.openHelper.writableDatabase.execSQL("DELETE FROM save_news WHERE save_id NOT IN (SELECT save_id FROM save_manifest)")
                }
                issue.description.contains("索引") -> {
                    saveDb.openHelper.writableDatabase.execSQL("REINDEX")
                }
                issue.description.contains("头像") -> {
                    // 头像回退到默认
                }
            }
        }
    }

    fun repairMedium(report: IntegrityReport): RepairResult {
        // 使用最近 checkpoint 恢复
        val lastCheckpoint = saveDb.checkpointDao().getLatest()
            ?: return RepairResult.Failed("无可用 checkpoint")

        val restored = CheckpointManager(saveDb).restoreFromCheckpoint(lastCheckpoint.checkpointId)
        return if (restored) {
            // 重建积分榜和统计
            rebuildRankingCache()
            rebuildStatsCache()
            RepairResult.Success("已从 checkpoint ${lastCheckpoint.checkpointId} 恢复")
        } else {
            RepairResult.Failed("checkpoint 恢复失败")
        }
    }

    fun repairSevere(): RepairResult {
        // 提示用户恢复备份，导出可救援数据
        return RepairResult.NeedUserBackup("存档严重损坏，请从用户备份恢复")
    }
}
```

## 十、关键风险与应对

### 1. Room 编译性能

- **表现**：35+ Entity 导致 Room KSP 编译缓慢（>2 分钟）
- **应对**：拆分 HistoryDatabase / SaveDatabase / CacheDatabase 三个独立 Database，避免单库过大
- **降级**：若 Room 编译仍慢，可对部分表使用 raw SQL 而非 @Entity

### 2. history.db 强制只读失效

- **表现**：Room 事务写入 history.db 成功（不应允许）
- **应对**：onOpen 回调中执行 `PRAGMA query_only = ON`
- **验证**：单测尝试写入 history.db 应抛出异常

### 3. 存档体积膨胀

- **表现**：单存档 20 年后 > 80MB
- **应对**：
  - save_player_state 只存当前赛季状态，旧赛季归档到 season_archive
  - 比赛事件压缩（仅保留比分+进球+红黄牌）
  - 定期 vacuum
- **监控**：perf_log 记录每次 daily_advance 的 db_size_mb

### 4. 迁移回滚失败

- **表现**：迁移失败后 checkpoint 恢复也失败
- **应对**：
  - 迁移前双备份（checkpoint + 文件复制到 backup 目录）
  - 失败时优先尝试 checkpoint 恢复，再尝试文件复制
  - 最坏情况：标记存档为只读，提示用户

### 5. 多存档并发

- **表现**：同时加载两个存档导致 save_db 引用混乱
- **应对**：DatabaseManager 单例，currentSaveId 单值，同时只能加载一个存档
- **限制**：切换存档必须先 close 当前 saveDb

- 三库结构（history + save + cache）
- 全部 Entity 和 DAO（V0.1 + V0.2 表）
- 存档创建/加载/列表
- V1→V2 迁移脚本
- 4 种 checkpoint
- 完整性检测 7 项

- 存档云同步
- 存档压缩加密
- 多存档并发加载
- 数据库编辑器 UI（T25）
- 自动备份到外部存储
