## 三、四条铁律

1. **V0.1 是产品全集，不是第一个版本的开工范围。** 第一个版本只做 Gate 1 + Gate 2 涉及的系统。
2. **五大算法按 V0.2 文档实现，不得自行发明。** 比赛/AI/经济/蝴蝶/成长的公式都已定义，只调参不改架构。
3. **所有数值参数配置化，不写死代码。** 评分权重、xG 系数、成长因子、经济指数、AI 权重全部走配置表。
4. **Gate 没过不往上叠。** 比赛引擎 10000 场没校准通过，不接赛季循环；赛季闭环没跑通，不接 AI/经济/蝴蝶。

---

## 四、三个验收门控

| Gate | 时间 | 验收内容 | 不通过的后果 |
|------|------|---------|------------|
| **Gate 1** | 第 4-6 周 | 比赛引擎 10000 场统计合理 | 暂停 B/C/D 集成，只修引擎 |
| **Gate 2** | 第 12-16 周 | 1 赛季完整闭环可玩 | 暂停 D/E 集成，修到跑通 |
| **Gate 3** | 第 24-32 周 | 20 年长程模拟无崩溃 | 定位 AI/经济/蝴蝶哪个失控 |

通过标准详见并行开发方案。

## 五、第一周该做什么

| 轨道 | 第一周任务 |
|------|-----------|
| **A 引擎核心** | 搭建 Room 三库结构 + 比赛引擎评分层原型 |

| **C 经营系统** | 设计转会搜索算法 + 合同数据结构 |
| **D 世界系统** | 研读 V0.2 `05_AI决策` + 设计 AI 计算频率分批方案 |
| **E 蝴蝶效应** | 研读 V0.2 `06_蝴蝶效应` + 设计替代决策池数据结构 |
| **F UI 与平台** | 搭建 Compose 导航骨架 + 版权合规 UI 原型 |
| **数据准备** | 开始造 4 队 100 人虚构数据集（Gate 1 用） |

## 六、技术栈

- Kotlin + Jetpack Compose + Room + SQLite
- 三库结构：history.db（只读）+ save_xxx.db（存档）+ cache.db（可重建）
- Room 负责数据库抽象层和版本迁移
- 最低 Android 版本、性能指标详见 V0.2 `03_非功能需求`

---

## 七、版权底线

- 公开版默认虚构数据，真实数据只走本地导入包
- 不内置真实队徽、球衣、头像、授权赛事名称
- 导入真实数据时需用户签署免责声明
- UI 文案不得暗示与真实俱乐部/球员存在授权关系
