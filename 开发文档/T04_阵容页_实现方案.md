## 一、业务目标与范围

### 1. 业务目标

为玩家提供一个**高频、稳定、信息密度合理**的球队管理入口，覆盖：

1. **梯队浏览**：5 个梯队（一线队 / 预备队 / U21 / U18 / 外租球员）一键切换。
2. **快速识别**：通过列表字段（CA/PA/状态/体能/士气/合同/身价）一眼看出球员价值与风险。
3. **多维筛选**：8 项筛选（位置 / 年龄 / 能力 / 潜力 / 合同 / 伤病 / 国籍 / 状态）组合查询。
4. **快捷操作**：长按球员行弹出 7 种操作（设置首发 / 放入替补 / 续约 / 挂牌 / 外租 / 设置训练 / 设置导师）。
5. **深度查看**：进入球员详情页查看 10 个模块（基础信息 / 属性 / 位置适应 / 成长曲线 / 赛季数据 / 合同 / 转会记录 / 伤病记录 / 球探报告 / 训练计划 / 关系状态）。
6. **状态完备**：6 种 UI 状态（loading / empty / error / normal / locked / warning / success）覆盖所有边界。

### 2. V1 做的范围

| 模块 | V1 范围 |
|------|---------|
| 梯队切换 | 5 个梯队（一线队 / 预备队 / U21 / U18 / 外租球员） |
| 列表字段 | 姓名 / 年龄 / 国籍 / 位置 / CA / PA / 状态 / 体能 / 士气 / 合同到期 / 身价 |
| 筛选 | 8 项单选或多选组合筛选 |
| 排序 | 8 种排序（CA / PA / 年龄 / 身价 / 合同到期 / 体能 / 士气 / 姓名拼音） |
| 长按操作 | 7 种操作（设置首发 / 放入替补 / 续约 / 挂牌 / 外租 / 设置训练 / 设置导师） |
| 详情页模块 | 10 个模块全部展示（部分操作跳转下游任务页面） |
| 详情页操作 | 7 种操作（设置训练 / 设置导师 / 续约 / 挂牌 / 租借 / 设为队长 / 加入观察名单） |
| 状态 UI | 6 种标准状态 |
| 球员搜索 | normalized_name 模糊匹配 + 位置/年龄/能力快速筛选 |
| 成长曲线 | 基于 player_attributes 多赛季快照绘制 CA/PA 曲线 |

### 3. V1 不做的范围（推迟到 M6/M7 或后续迭代）

- **真实头像照片**：V1 仅显示姓氏首字占位图（版权合规）。
- **3D 球员模型 / 高亮动画**：V1 仅静态卡片。
- **球员对比页**：2-3 名球员属性对比，归入 T10 转会搜索。
- **属性雷达图动画**：V1 仅静态 Canvas 雷达图。
- **自定义列**：玩家拖拽自定义列表显示字段，V1 固定列。
- **批量操作**：批量挂牌 / 批量续约，V1 仅单球员长按操作。
- **观察名单管理页**：加入观察名单后由 T10 转会页管理。
- **历史职业生涯完整回放**：仅展示成长曲线，不提供逐赛季回放。
- **球员社交媒体 / 个人生活事件**：归入 M6 沉浸系统。

```kotlin
// data/SquadRepository.kt
class SquadRepository(
    private val savePlayerStateDao: SavePlayerStateDao,
    private val playerDao: PlayerDao,
    private val injuryDao: SaveInjuryDao,
    private val cacheDao: SquadListCacheDao,
    private val filterEngine: SquadFilterEngine,
    private val sorter: PlayerSorter,
    private val historyDb: HistoryDatabase,
    private val saveDb: SaveDatabase,
    private val cacheDb: CacheDatabase,
    private val config: SquadConfig
) {

    /**
     * 分页加载某梯队球员列表
     * 性能：≤200ms（50 条）
     */
    suspend fun loadSquadPage(
        saveId: Int,
        clubId: Int,
        tab: SquadTab,
        filter: SquadFilter,
        sortField: SquadSortField,
        page: Int
    ): SquadPageResult {
        val pageSize = config.pageSize
        val offset = page * pageSize

        // 1. 检查 cache.db（仅当无筛选时使用缓存）
        if (filter == SquadFilter() && page == 0) {
            val cacheKey = "${saveId}_${clubId}_${tab.roleId}_${filter.toHash()}"
            val cached = cacheDao.get(saveId, clubId, tab.roleId, filter.toHash())
            if (cached != null && isCacheFresh(cached.updatedAt, config.cacheTtlMinutes)) {
                val list = deserializePlayers(cached.snapshotJson)
                val sorted = sorter.sort(list, sortField)
                return SquadPageResult(sorted.take(pageSize), cached.playerCount, false, true)
            }
        }

        // 2. 从 save.db 查询 SavePlayerState（按梯队）
        val states = if (tab == SquadTab.LOAN_OUT) {
            savePlayerStateDao.getLoanedOutPlayers(saveId, clubId, pageSize, offset)
        } else {
            savePlayerStateDao.getSquadByRole(saveId, clubId, tab.roleId, pageSize, offset)
        }

        if (states.isEmpty()) {
            return SquadPageResult(emptyList(), 0, false, false)
        }

        // 3. 批量查询 history.db 球员基础资料
        val playerIds = states.map { it.playerId }
        val players = playerDao.getByIds(playerIds).associateBy { it.playerId }

        // 4. 拼接 PlayerListItem
        val currentDate = getCurrentDate(saveId)
        var items = states.mapNotNull { state ->
            val player = players[state.playerId] ?: return@mapNotNull null
            buildPlayerListItem(state, player, currentDate)
        }

        // 5. 应用筛选（位置/年龄/能力/潜力/合同/伤病/国籍/状态）
        items = filterEngine.apply(items, filter)

        // 6. 排序
        items = sorter.sort(items, sortField)

        // 7. 总数（用于分页）
        val totalCount = savePlayerStateDao.countSquadByRole(saveId, clubId, tab.roleId)

        // 8. 写入 cache.db（仅无筛选时）
        if (filter == SquadFilter() && page == 0) {
            cacheDao.upsert(SquadListCacheEntity(
                saveId = saveId, clubId = clubId, squadRole = tab.roleId,
```

```kotlin
// filter/SquadFilterEngine.kt
class SquadFilterEngine {

    /**
     * 多条件筛选组合
     * V0.1 03 §阵容页筛选 8 项
     */
    fun apply(players: List<PlayerListItem>, filter: SquadFilter): List<PlayerListItem> {
        return players.filter { p ->
            // 1. 位置筛选（含主位置 + 副位置）
            filter.positions.isEmpty() ||
                p.primaryPosition in filter.positions ||
                p.secondaryPositions.any { it in filter.positions }

            // 2. 年龄筛选
            && p.age in filter.ageRange

            // 3. 能力筛选
            && p.currentCa in filter.caRange

            // 4. 潜力筛选
            && p.currentPa in filter.paRange

            // 5. 合同剩余月数筛选
            && (filter.contractRemainingMonths == null ||
                computeRemainingMonths(p.contractUntil) in filter.contractRemainingMonths)

            // 6. 伤病筛选
            && when (filter.injuryFilter) {
                InjuryFilter.ANY -> true
                InjuryFilter.HEALTHY_ONLY -> p.injuryStatus == "healthy"
                InjuryFilter.INJURED_ONLY -> p.injuryStatus != "healthy"
            }

            // 7. 国籍筛选
            && (filter.nationalities.isEmpty() || p.nationality in filter.nationalities)

            // 8. 状态筛选（多选）
            && (filter.statusFilter.isEmpty() || matchStatusFilter(p, filter.statusFilter))
        }
    }

    private fun matchStatusFilter(p: PlayerListItem, status: Set<StatusFilter>): Boolean {
        // 多选时满足任意一个即匹配
        return status.any { flag ->
            when (flag) {
                StatusFilter.CAPTAIN -> p.isCaptain
                StatusFilter.LISTED -> p.isListed
                StatusFilter.LOANED -> false  // PlayerListItem 不直接含此字段，外租 Tab 独立
                StatusFilter.CONTRACT_EXPIRING ->
                    computeRemainingMonths(p.contractUntil) <= 12
                StatusFilter.LOW_MORALE -> p.morale < 30
                StatusFilter.LOW_CONDITION -> p.condition < 30
            }
        }
    }

    private fun computeRemainingMonths(contractUntil: String?): Int {
        if (contractUntil == null) return Int.MAX_VALUE
        return ChronoUnit.MONTHS.between(LocalDate.now(), LocalDate.parse(contractUntil)).toInt()
    }
}
```

```kotlin
class PlayerSorter {

    fun sort(players: List<PlayerListItem>, field: SquadSortField): List<PlayerListItem> {
        val comparator = when (field) {
            SquadSortField.CA -> compareByDescending<PlayerListItem> { it.currentCa }
            SquadSortField.PA -> compareByDescending { it.currentPa }
            SquadSortField.AGE -> compareBy { it.age }
            SquadSortField.MARKET_VALUE -> compareByDescending { it.marketValue }
            SquadSortField.CONTRACT -> compareBy {
                it.contractUntil ?: "9999-12-31"
            }
            SquadSortField.CONDITION -> compareByDescending { it.condition }
            SquadSortField.MORALE -> compareByDescending { it.morale }
            SquadSortField.NAME -> compareBy { it.displayName }
        }
        // 同分时按位置 → CA → 姓名二级排序，保证稳定
        val tiebreaker = compareBy<PlayerListItem> {
            positionOrder(it.primaryPosition)
        }.thenByDescending { it.currentCa }.thenBy { it.displayName }
        return players.sortedWith(comparator.then(tiebreaker))
    }

    private fun positionOrder(pos: String): Int = when (pos) {
        "GK" -> 0
        "CB" -> 1
        "LB", "RB" -> 2
        "DM" -> 3
        "CM" -> 4
        "AM" -> 5
        "LW", "RW" -> 6
        "ST" -> 7
        else -> 99
    }
}
```

```kotlin
// growth/GrowthCurveGenerator.kt
class GrowthCurveGenerator {

    /**
     * 基于历史 player_attributes 多赛季快照生成成长曲线
     * V0.2 08 球员成长算法：CA/PA 随年龄变化
     * 性能：≤50ms（单球员）
     */
    fun generate(
        allAttributes: List<PlayerAttributesEntity>,
        player: PlayerEntity,
        currentDate: LocalDate
    ): List<GrowthCurvePoint> {
        if (allAttributes.isEmpty()) return emptyList()

        val birthDate = player.birthDate?.let { LocalDate.parse(it) }
        val points = mutableListOf<GrowthCurvePoint>()

        // 1. 按赛季顺序构建历史点
        val sorted = allAttributes.sortedBy { it.seasonId }
        sorted.forEach { attr ->
            val seasonLabel = lookupSeasonLabel(attr.seasonId)
            val age = if (birthDate != null) {
                // 由 season.start_date - birthDate 估算
                lookupSeasonStartDate(attr.seasonId)?.let {
                    ChronoUnit.YEARS.between(birthDate, LocalDate.parse(it)).toInt()
                } ?: 25
            } else 25

            points.add(GrowthCurvePoint(
                seasonId = attr.seasonId,
                seasonLabel = seasonLabel,
                age = age,
                ca = attr.ca,
                pa = attr.pa,
                isProjected = false
            ))
        }

        // 2. 未来预测点（基于 V0.2 08 年龄基础成长表）
        val lastPoint = points.last()
        val futureYears = config.growthCurveProjectYears   // 默认 5
        for (i in 1..futureYears) {
            val projectedAge = lastPoint.age + i
            val baseGrowth = ageBaseGrowth(projectedAge)   // 查表 14-34+
            val projectedCa = (lastPoint.ca + baseGrowth * i).coerceIn(0, lastPoint.pa)
            points.add(GrowthCurvePoint(
                seasonId = -1,
                seasonLabel = "+${i}年",
                age = projectedAge,
                ca = projectedCa,
                pa = lastPoint.pa,
                isProjected = true
            ))
        }

        return points
    }

    /**
     * V0.2 08 §四 年龄基础成长表
     */
    private fun ageBaseGrowth(age: Int): Double = when (age) {
        in 14..16 -> 0.40
        in 17..19 -> 0.80
        in 20..23 -> 0.55
        in 24..27 -> 0.20
        in 28..30 -> 0.05
        in 31..33 -> -0.20
        else -> -0.50
    }
}
```

```text
function loadSquadPage(saveId, clubId, tab, filter, sort, page):
    pageSize = config.pageSize  # 50
    offset = page * pageSize

    # 1. 优先读 cache.db（仅无筛选时）
    if filter is empty and page == 0:
        cached = cacheDao.get(saveId, clubId, tab.roleId, filter.hash)
        if cached != null and isFresh(cached.updatedAt, config.cacheTtlMinutes):
            list = deserialize(cached.snapshotJson)
            return sort(list, sort).take(pageSize)

    # 2. save.db 查询 SavePlayerState
    if tab == LOAN_OUT:
        states = savePlayerStateDao.getLoanedOut(saveId, clubId, pageSize, offset)
    else:
        states = savePlayerStateDao.getSquadByRole(saveId, clubId, tab.roleId, pageSize, offset)

    if states is empty:
        return emptyResult

    # 3. 批量查 history.db.player
    playerIds = states.map { it.playerId }
    players = playerDao.getByIds(playerIds).associateBy { it.playerId }

    # 4. 拼接 PlayerListItem
    items = states.map { buildListItem(it, players[it.playerId], currentDate) }

    # 5. 应用筛选（位置/年龄/能力/潜力/合同/伤病/国籍/状态）
    items = filterEngine.apply(items, filter)

    # 6. 排序
    items = sorter.sort(items, sort)

    # 7. 写 cache.db（仅无筛选时）
    if filter is empty and page == 0:
        cacheDao.upsert(saveId, clubId, tab.roleId, filter.hash, serialize(items))

    totalCount = savePlayerStateDao.countSquadByRole(saveId, clubId, tab.roleId)
    hasMore = (offset + pageSize) < totalCount

    return SquadPageResult(items, totalCount, hasMore)
```

```text
        and (filter.nationalities is empty or p.nationality in filter.nationalities)

        # 8. 状态（多选，满足任意一项即匹配）
        and (filter.statusFilter is empty
```

```kotlin
// ui/detail/panel/GrowthCurveChart.kt
@Composable
fun GrowthCurveChart(
    points: List<GrowthCurvePoint>,
    modifier: Modifier = Modifier
) {
    if (points.size < 2) {
        Box(modifier = modifier.height(200.dp), contentAlignment = Alignment.Center) {
            Text("数据不足以绘制曲线", color = Color.Gray)
        }
        return
    }

    val caColor = Color(0xFF2196F3)
    val paColor = Color(0xFF4CAF50)
    val projectedColor = Color(0xFF9E9E9E)

    Canvas(modifier = modifier.fillMaxWidth().height(220.dp).padding(16.dp)) {
        val width = size.width
        val height = size.height
        val padding = 40f

        // 1. 计算坐标系
        val minCa = points.minOf { minOf(it.ca, it.pa) }
        val maxCa = points.maxOf { maxOf(it.ca, it.pa) }
        val caRange = (maxCa - minCa).coerceAtLeast(10)

        // 2. 绘制网格
        val gridCount = 5
        val gridColor = Color(0xFFEEEEEE)
        for (i in 0..gridCount) {
            val y = padding + (height - 2 * padding) * i / gridCount
            drawLine(
                color = gridColor,
                start = Offset(padding, y),
                end = Offset(width - padding, y),
                strokeWidth = 1f
            )
        }

        // 3. 绘制 X 轴标签（年龄）
        points.forEachIndexed { idx, point ->
            val x = padding + (width - 2 * padding) * idx / (points.size - 1).coerceAtLeast(1)
            // 标签由 drawText 完成（需 nativeCanvas + Paint）
        }

        // 4. 绘制 PA 折线（背景线，理论潜力上限）
        val paPath = Path().apply {
            points.forEachIndexed { idx, point ->
                val x = padding + (width - 2 * padding) * idx / (points.size - 1).coerceAtLeast(1)
                val y = height - padding - (height - 2 * padding) * (point.pa - minCa) / caRange
                if (idx == 0) moveTo(x, y) else lineTo(x, y)
            }
        }
        drawPath(paPath, color = paColor, style = Stroke(width = 2f))

        // 5. 绘制 CA 折线（实线 + 虚线分段）
        // 5.1 实线部分（历史点）
        val solidPoints = points.filter { !it.isProjected }
        val solidPath = Path().apply {
            solidPoints.forEachIndexed { idx, point ->
                val x = padding + (width - 2 * padding) * points.indexOf(point) /
                    (points.size - 1).coerceAtLeast(1)
                val y = height - padding - (height - 2 * padding) * (point.ca - minCa) / caRange
                if (idx == 0) moveTo(x, y) else lineTo(x, y)
            }
        }
        drawPath(solidPath, color = caColor, style = Stroke(width = 3f))

        // 5.2 虚线部分（预测点）
        val projectedPoints = points.filter { it.isProjected }
        if (projectedPoints.isNotEmpty()) {
            val lastSolid = solidPoints.last()
            val projStart = points.indexOf(lastSolid)
            val projectedPath = Path().apply {
                val firstProjIdx = points.indexOf(projectedPoints.first())
                val startX = padding + (width - 2 * padding) * (firstProjIdx - 1) /
                    (points.size - 1).coerceAtLeast(1)
                val startY = height - padding - (height - 2 * padding) *
                    (lastSolid.ca - minCa) / caRange
                moveTo(startX, startY)
                projectedPoints.forEach { point ->
                    val idx = points.indexOf(point)
                    val x = padding + (width - 2 * padding) * idx /
                        (points.size - 1).coerceAtLeast(1)
                    val y = height - padding - (height - 2 * padding) *
                        (point.ca - minCa) / caRange
                    lineTo(x, y)
                }
            }
            drawPath(projectedPath, color = projectedColor, style = Stroke(width = 2f))
        }

        // 6. 数据点圆圈
        points.forEach { point ->
            val idx = points.indexOf(point)
            val x = padding + (width - 2 * padding) * idx /
                (points.size - 1).coerceAtLeast(1)
            val y = height - padding - (height - 2 * padding) * (point.ca - minCa) / caRange
            drawCircle(
                color = if (point.isProjected) projectedColor else caColor,
                radius = 4f,
                center = Offset(x, y)
            )
        }
    }
}
```

## 十、性能要求

| 操作 | 目标 | 实现策略 |
|------|------|----------|
| 列表加载（分页 50 条） | ≤200ms | cache.db 优先读 + save.db 索引 `(save_id, current_club_id, squad_role)` |
| 详情页加载（10 表聚合） | ≤300ms | 10 个 DAO 并发查询（`async + awaitAll`）+ cache.db 属性缓存 |
| 筛选响应 | ≤100ms | 筛选在内存中执行（数据已加载），不重新查库 |
| 排序响应 | ≤50ms | 排序在内存中执行，仅切换 Comparator |
| 梯队切换 | ≤200ms | 复用 cache.db + 后台预加载相邻梯队 |
| 滚动流畅度 | 60fps | LazyColumn + PlayerRow 复用 + 避免每行 Bitmap 解码 |
| 长按操作回写 | ≤100ms | 单条 UPDATE + 失效该球员相关缓存 |
| 成长曲线绘制 | ≤50ms | Compose Canvas 一次性绘制，无动画 |
| 球员搜索 | ≤500ms | normalized_name LIKE + 索引 `idx_player_normalized_name` |
| 1200 球员全量加载 | ≤1s | 分页 + 索引（Gate 2 验收要求） |

### 性能优化要点

1. **三库分离读取**：history.db 只读不锁，save.db 写入用 WAL 模式，cache.db 可重建。
2. **批量查询**：球员基础资料用 `WHERE player_id IN (...)` 一次取回，避免 N+1。
3. **索引完备**：
   - `save_player_state(save_id, current_club_id, squad_role)`
   - `save_player_state(save_id, loan_club_id)`（外租查询）
   - `player_attributes(player_id, season_id)`
   - `player(normalized_name)`
4. **cache.db 加速**：列表无筛选时优先读 JSON 快照，TTL 5 分钟。
5. **并发聚合**：详情页 10 个 DAO 查询并发执行（`coroutineScope.async`），总耗时 ≈ max(单查询)。
6. **LazyColumn 复用**：PlayerRow 不持有 Context，状态由外部传入。
7. **避免主线程 IO**：所有 DAO 调用走 `suspend` + IO Dispatcher。
8. **筛选内存化**：首次加载后，筛选/排序在内存中执行，不重复查库。
9. **图片懒加载**：球员头像用 Coil/Glide 懒加载 + 缓存到 cache.db 路径。
10. **后台预加载**：梯队切换时后台预加载相邻梯队（如一线队→预备队切换时预加载 U21）。

## 十五、风险与应对

### 1. 详情页多表聚合性能不达标

- **表现**：10 个 DAO 串行查询导致详情页加载 >300ms
- **应对**：
  - 10 个查询并发执行（`coroutineScope.async + awaitAll`），总耗时 ≈ max(单查询)
  - cache.db 缓存属性聚合结果（player_attribute_cache），TTL 5 分钟
  - 球员基础资料用 `WHERE player_id IN (...)` 批量查询，避免 N+1
  - 单测强制校验详情页加载 ≤300ms

### 2. 列表筛选后性能下降

- **表现**：1200 球员全量加载后内存筛选 >100ms
- **应对**：
  - **筛选下沉到 SQL 层**：所有 8 项筛选条件拼接到 DAO 查询的 `WHERE` 子句，依赖 `idx_save_player_state_echelon` + `idx_save_player_state_position` + `idx_save_player_state_ca` 索引完成过滤，避免 1200 条全量入内存后再筛
  - **分页加载**：LazyColumn 每页 50 条，通过 `LIMIT 50 OFFSET ?` 拉取，滚动到底部自动加载下一页
  - **筛选缓存**：筛选结果写入 `cache.db.squad_list_cache`，key = `echelon + filter_hash + sort_field`，TTL 5 分钟，相同筛选组合直接命中缓存
  - **索引兜底**：在 `save_player_state` 上为 `(echelon, position, ca, age, contract_end)` 建复合索引，保证 WHERE 多条件查询走索引而非全表扫描
  - **单测阈值**：`SquadFilterEngine` 单测强制校验 1200 球员 × 8 项筛选组合 <100ms，超出即 fail

### 3. 成长曲线历史快照缺失

- **表现**：新存档开局时 `player_attributes` 只有当前赛季一条记录，成长曲线无历史可画
- **应对**：
  - **初始快照回填**：新存档创建时（T00 SaveEngine.createSave），从 `history.db.player_attributes` 读取该球员最近 3 个赛季的属性快照，批量写入 `save.db.player_attributes`，作为成长曲线起点
  - **合成历史点**：若 history.db 也无历史快照（如本作新生成球员），按 V0.2 `08_球员成长_训练_伤病细化算法.md` §四年龄基础成长表反推过去 3 年的合成 CA 点，并在曲线上以虚线 + "合成数据"标记区分
  - **月度快照**：每月推进时（GrowthEngine.processMonthlyGrowth）写入一条 `player_attributes` 快照，保证后续赛季有真实历史
  - **降级展示**：若历史点 <2 个，成长曲线区域显示"数据积累中，至少需要 2 个赛季"占位文案，不画曲线只画当前 CA 单点
  - **预测虚线**：基于 V0.2 年龄基础成长表 + 当前 CA + PA 差距，预测未来 3 年 CA 走势以虚线展示，让玩家看到潜力兑现预期

### 4. 长按操作后缓存与列表状态不一致

- **表现**：玩家对 A 球员执行"挂牌"后，列表仍显示"未挂牌"，或外租后梯队未刷新
- **应对**：
  - **同步回写双库**：`PlayerActionRepository` 执行操作时，先写 `save.db.save_player_state`（更新 `is_listed` / `squad_role` 等字段），再同步删除 `cache.db.squad_list_cache` 中该梯队对应 key，最后让 `player_attribute_cache` 中该球员条目失效
  - **StateFlow 推送**：操作完成后通过 `SquadUiState.refreshToken` 自增，触发 SquadViewModel 重新拉取当前梯队列表
  - **乐观更新**：UI 层先本地更新 PlayerRow 状态（如挂牌图标立即变亮），再异步落库；若落库失败回滚 UI 并 Toast 提示
  - **跨梯队联动**：外租操作需将球员从原梯队移除并写入"外租球员"梯队，触发两个梯队的列表刷新
  - **二次确认闸门**：所有 7 种长按操作必须经 `PlayerActionConfirmDialog` 二次确认后才执行回写，避免误触导致状态错乱
  - **操作日志**：每次操作写入 `save.db.player_action_log`（player_id / action_type / before_value / after_value / timestamp），便于异常回查

### 5. 三库跨库查询一致性

- **表现**：外租球员基础资料在 `history.db.player`，当前状态在 `save.db.save_player_state`（且 `club_id` 已不是本队），跨库 JOIN 失败或结果错乱
- **应对**：
  - **ATTACH DATABASE 方案**：在打开存档时执行 `ATTACH DATABASE 'history.db' AS hist`，DAO 中使用 `hist.player` + `main.save_player_state` 跨库 JOIN，由 SQLite 优化器统一执行
  - **两段式查询兜底**：若 ATTACH 受 Room 限制不稳定，先在 `save.db.save_player_state` 查出本队外租球员的 `player_id` 列表（`WHERE squad_role = 'loaned_out'`），再用 `WHERE player_id IN (...)` 批量回 `history.db.player` 拉基础资料，避免 N+1
  - **外租球员识别**：在 `save_player_state` 上加 `squad_role` 字段（`first_team` / `reserve` / `u21` / `u18` / `loaned_out`），外租球员的 `club_id` 是租借方俱乐部，但 `owner_club_id` 仍是本队，查询条件用 `owner_club_id = ? AND squad_role = 'loaned_out'`
  - **回租同步**：外租到期时由 TransferEngine 触发 `squad_role` 改回 `first_team` 或 `reserve`，同时刷新 `cache.db.squad_list_cache` 两个梯队
  - **测试用例**：覆盖"球员外租 → 提前召回 → 重新外租"全链路，校验列表显示与详情页数据一致
  - **资源缺失兜底**：若 history.db 中 player 记录缺失（数据包不完整），显示 "[数据缺失]" 占位并写入 `local_data_issue` 表，不阻塞列表加载

### 6. 1200 球员内存压力

- **表现**：5 个梯队全量切换 + 列表 + 详情页聚合后，堆内存占用 >200MB，低端机型 OOM
- **应对**：
  - **LazyColumn 复用**：列表项严格使用 `key(player.id)` 复用 Composable，避免重组时重建视图树
  - **轻量列表模型**：`SquadListItem` 只持有列表 11 字段（不含完整属性集），完整属性延迟到详情页加载
  - **相邻梯队预加载**：当前梯队渲染完成后，后台协程预加载相邻梯队第一页 50 条到 `cache.db`，但不入内存
  - **图片弱引用缓存**：球员头像使用 LruCache（容量 100 张，每张 ≤20KB），超出走磁盘缓存
  - **详情页属性释放**：离开详情页时 `PlayerDetailViewModel.onCleared()` 主动释放完整属性集引用，仅保留列表模型
  - **梯队切换时 GC**：切换梯队前调用 `System.gc()` 提示回收上一梯队的 PlayerRow Composable
  - **内存监控**：单测中通过 `Debug.getNativeHeapAllocatedSize()` 校验单梯队加载后堆增长 <20MB，超出即告警
