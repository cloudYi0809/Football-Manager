# 16. 资源素材规范

---

# 一、资源类型

游戏本地资源包括：

- 球员头像
- 俱乐部队徽
- 球衣
- 球场图片
- 国家旗帜
- 赛事图标
- 新闻默认图
- UI 图标

---

# 二、目录结构

```text
assets/
├── portraits/
│   ├── players/
│   └── default_player.png
├── logos/
│   ├── clubs/
│   ├── competitions/
│   └── default_club.png
├── kits/
│   ├── club_id_home_year.png
│   └── club_id_away_year.png
├── stadiums/
├── flags/
├── news/
└── ui/
```

---

# 三、命名规范

## 1. 球员头像

```text
player_{player_id}.png
```

示例：

```text
player_100001.png
```

## 2. 队徽

```text
club_{club_id}.png
```

## 3. 球衣

```text
club_{club_id}_{season_id}_home.png
club_{club_id}_{season_id}_away.png
club_{club_id}_{season_id}_third.png
```

## 4. 国家旗帜

```text
flag_{country_code}.png
```

---

# 四、尺寸建议

| 类型 | 建议尺寸 |
|---|---|
| 球员头像 | 256x256 或 512x512 |
| 队徽 | 256x256 |
| 球衣 | 512x512 |
| 球场图 | 1280x720 |
| 国家旗帜 | 128x88 |
| 赛事图标 | 256x256 |

---

# 五、缺省资源

必须准备默认资源：

- default_player.png
- default_club.png
- default_kit.png
- default_stadium.png
- default_flag.png

资源缺失时不能崩溃。

---

# 六、数据库路径字段

player.portrait_path 示例：

```text
portraits/players/player_100001.png
```

club.logo_path 示例：

```text
logos/clubs/club_200001.png
```

---

# 七、个人单机使用说明

本项目默认个人本地使用。真实头像、队徽、球衣等素材如使用，应作为本地资源包，不做公开上架、不做商业分发。

---

# 八、UI 风格建议

整体风格：

- 深色足球经理后台感
- 绿色球场色为主
- 金色用于荣誉和重点数据
- 卡片式信息层级
- 数据表格清晰可筛选

主色建议：

- 深绿：#0B3D2E
- 草地绿：#1E7A4D
- 金色：#C9A45C
- 背景深灰：#121614
- 卡片深灰：#1D2420
- 文字白：#F5F5F0

---

# 九、验收标准

1. 所有资源路径可解析。
2. 缺图时显示默认图。
3. 头像、队徽在列表和详情页显示正常。
4. 大量资源不会导致首屏严重卡顿。
5. 资源包可替换。
