# 03. 页面结构与交互说明

本文档定义 Android 端页面结构、入口关系、关键交互和状态流。

---

# 一、整体导航

## 1. 底部主导航

底部主导航固定 5 个入口：

1. 首页
2. 阵容
3. 战术
4. 转会
5. 赛程

不建议底部放太多入口。青训、球探、董事会、训练、媒体、国家队等通过首页快捷入口或二级页面进入。

---

# 二、一级页面

## 1. 首页：经理办公室

### 页面目标

首页是玩家推进游戏和处理待办事项的中心。

### 页面模块

1. 顶部状态栏
   - 当前日期
   - 当前赛季
   - 当前球队
   - 当前联赛排名
   - 当前余额

2. 下一场比赛卡片
   - 赛事
   - 对手
   - 日期
   - 主客场
   - 对手排名
   - 快捷按钮：进入赛前准备

3. 待处理事项
   - 球员受伤
   - 续约提醒
   - 转会报价
   - 球探报告
   - 青训推荐
   - 董事会消息
   - 媒体发布会

4. 球队状态
   - 士气
   - 体能
   - 更衣室
   - 董事会满意度
   - 球迷满意度

5. 快捷入口
   - 青训学院
   - 球探中心
   - 训练中心
   - 董事会
   - 更衣室
   - 医疗中心
   - 数据分析
   - 财务中心

6. 今日新闻
   - 转会新闻
   - 伤病新闻
   - 比赛新闻
   - 历史事件
   - 其他球队动态

### 核心按钮

- 继续一天
- 跳到下一场比赛
- 查看新闻
- 处理待办

### 交互规则

- 点击“继续一天”后，进入日推进流程。
- 如有必须处理事项，弹出确认或跳转。
- 如果当天有比赛，继续按钮变为“进入比赛日”。

1. 阵型区

   - 替补拖入首发

2. 战术风格区

   - 防守反击
   - 边路传中
   - 中路渗透
   - 长传冲吊
   - 巨星自由发挥

3. 参数区
   - 节奏
   - 压迫
   - 防线
   - 传球
   - 进攻方向
   - 防守方式

   - 点球手
   - 任意球手

# 四、关键流程

## 1. 开始新游戏流程

1. 启动游戏。
2. 选择新游戏。
3. 选择剧本。
4. 选择模式。
5. 选择联赛加载范围。
6. 选择俱乐部。
7. 设置开关。
8. 创建存档。
9. 进入首页。

## 2. 每日推进流程

1. 点击继续一天。
2. 训练结算。
3. 伤病恢复。
4. 球探推进。
5. 青训成长。
6. 转会处理。
7. AI 俱乐部行动。
8. 新闻生成。
9. 如果有比赛，进入比赛日提示。
10. 更新首页。

## 3. 比赛日流程

1. 赛前准备。
2. 阵容确认。
3. 战术确认。
4. 赛前讲话。
5. 比赛模拟。
6. 中场调整。
7. 赛后报告。
8. 士气和评价结算。

## 4. 球探发现流程

1. 进入球探中心。
2. 选择球探。
3. 选择地区。
4. 设置任务。
5. 等待报告。
6. 发现球员。
7. 加入观察名单。
8. 深度观察。
9. 尝试签约。

## 5. 转会流程

1. 搜索球员。
2. 查看报告。
3. 提交报价。
4. 俱乐部回应。
5. 经纪人谈判。
6. 合同条款确认。
7. 体检。
8. 转会完成。
9. 历史路径更新。

# 五、UI 状态标准

每个页面都需要处理以下状态：

- loading：加载中
- empty：无数据
- error：数据错误
- normal：正常显示
- locked：功能未解锁
- warning：有风险提示
- success：操作成功

---

# 六、Android 交互建议

- 列表页优先使用搜索 + 筛选 + 排序。
- 关键操作必须二次确认，例如出售核心球员、强行复出、删除存档。
- 比赛模拟支持跳过。
- 赛季推进支持停止在关键事件。
- 所有报告类页面要有“加入待办”功能。
- 所有球员名、俱乐部名点击都应可跳详情。
