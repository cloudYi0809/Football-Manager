## 二、开发方式建议

不要一开始做完整 UI。建议顺序：

1. 先把 SQLite 数据库和存档跑通。
2. 再做球员、俱乐部、阵容、赛季推进。
3. 再做比赛模拟。
4. 再做转会、青训、球探。
5. 最后做历史事件、蝴蝶效应、挑战剧本、编辑器。

## 三、项目边界

本项目默认用于个人本地单机娱乐，不做公开上架、不做联网服务、不做商业分发。真实球员、真实队伍、真实图片、真实队徽等素材如使用，应只在个人本地资源包中使用。

## 四、核心一句话

> 开局是真实足球历史，过程由玩家亲手改写。

# 01.《绿茵王朝：2002》项目总览与 PRD

## 1. 项目名称

中文名：绿茵王朝：2002  
英文代号：Football Dynasty 2002  
平台：Android  
类型：单机足球经理模拟游戏  
数据路线：真实历史数据库  
核心体验：真实开局、自由改写、长期王朝、青训挖宝、历史模拟。

---

## 2. 产品定位

《绿茵王朝：2002》是一款基于 Android 的单机足球经理模拟游戏。游戏以真实足球历史为基础，使用真实球员、真实俱乐部、真实赛季、真实转会、真实比赛、真实国家队赛事和真实球员成长轨迹作为底层资料库，让玩家从 1990 年代或 2002 年黄金年代开始执教俱乐部，通过转会、青训、球探、战术、比赛、董事会、更衣室、历史事件、国家队、数据分析等系统，改变真实足球历史。

游戏不是实时操控比赛，而是长期经营与模拟推演。核心体验是：

> 从真实足球历史出发，亲手改写世界足坛。

---

## 3. 核心卖点

### 3.1 真实历史开局

玩家可以从指定年代开始：

- 1992/93：传奇种子时代，可以早期发现罗纳尔多、齐达内、菲戈、贝克汉姆、皮耶罗等。
- 2002/03：黄金年代，罗纳尔多、齐达内、亨利、内斯塔、舍甫琴科、小罗、卡恩等处于高光期，卡卡、C 罗、鲁尼、梅西即将进入主线。
- 2006/07：双骄前夜，卡卡巅峰，梅西与 C 罗高速成长。
- 2014/15：现代王朝，梅西、C 罗、内马尔、苏亚雷斯、莱万、德布劳内等进入现代足球阶段。
- 自定义沙盒：自定义数据库、资金、开关、规则。

### 3.2 历史改写

玩家可以改变关键历史：

- 让大罗不离开国米。
- 让卡卡不去 AC 米兰。
- 让 C 罗不去曼联。
- 提前签下梅西。
- 阻止 AC 米兰老化。
- 延续阿森纳不败王朝。
- 让皇马银河战舰不崩盘。
- 让中国球员进入欧洲豪门。

### 3.3 青训与球探挖宝

核心爽点不是只买成名球星，而是在他们成名前发现他们。

> 你不只是签下巨星，你可以在世界还不知道他是谁的时候，发现他。

### 3.4 长期世界模拟

游戏支持从 1992 玩到 2026、2050，甚至更久。球员会成长、巅峰、下滑、退役、转教练；俱乐部会崛起、衰落、被收购、财政危机、重建；历史会被玩家改变并产生连锁反应。

---

## 4. 目标用户

1. 足球经理类玩家：喜欢 Football Manager、冠军足球经理、实况大师联赛、FIFA 经理模式。
2. 足球历史爱好者：喜欢大罗、卡卡、小罗、齐达内、亨利、舍甫琴科、内斯塔、马尔蒂尼等黄金年代球星。
3. 数据模拟玩家：喜欢长期存档、沙盒推演、历史改写、数据成长。
4. 单机沉浸玩家：不需要联网，不需要竞技，只需要完整可长期游玩的私人游戏。

---

## 5. 游戏模式

### 5.1 真实历史模式

默认沿真实历史推进。

规则：

- 真实转会默认开启。
- 真实退役默认开启。
- 真实新星默认开启。
- 真实历史事件默认开启。
- 真实赛程可开启。
- 玩家可以干预关键节点。

适合历史还原玩家。

### 5.2 自由改写模式

开局真实，但之后世界自由模拟。

规则：

- 初始阵容、年龄、俱乐部、联赛真实。
- 历史新星按真实出生和可发现年份进入世界。
- 转会由 AI 和玩家自由推进。
- 比赛结果由模拟引擎生成。
- 球员职业轨迹可完全改变。

这是主模式。

### 5.3 沙盒编辑模式

玩家可以自由修改：

- 球员能力、潜力、年龄、合同、国籍、位置。
- 俱乐部资金、声望、球场、青训、训练设施。
- 联赛规则、赛制、转会窗口。
- 真实转会、真实退役、传奇重生等开关。

适合个人长期娱乐和调试。

## 9. 成功标准

### 9.1 可玩性标准

- 玩家可以完整玩完一个赛季。
- 玩家可以从 2002 玩到至少 2010，不发生数据崩坏。
- 玩家可以签约、续约、出售、租借球员。
- 玩家可以通过球探发现历史新星。
- 玩家可以通过青训培养球员。
- 玩家干预真实转会后，世界仍能继续模拟。

### 9.2 技术标准

- 本地数据库可加载。
- 存档可创建、读取、覆盖、删除。
- 每日推进不超过可接受等待时间。
- 赛季结算不破坏数据完整性。
- 比赛模拟结果具有基本可信度。
- Android 离线可运行。

### 9.3 数据标准

- 球员、俱乐部、转会、阵容、赛季之间关系正确。
- 历史新星不会重复生成。
- 真实转会被玩家打断后不会强制回滚。
- 退役、成长、伤病、合同不会出现明显异常。

# 三、字段设计原则

## 1. ID 设计

所有主表使用 INTEGER PRIMARY KEY。  
历史数据 ID 与存档状态 ID 分离。  
外部数据源 ID 单独保存 source_id，不作为内部主键。

## 2. 时间字段

统一使用 TEXT ISO 日期：YYYY-MM-DD。  
日期推进由游戏内 DateEngine 处理。

## 3. 金额字段

金额统一用整数，单位建议为欧元或英镑。  
例如 120000 表示 120,000。

## 4. 能力字段

能力值建议范围：1-100。

如果以后要兼容 FM 1-200，可增加 raw_ca/raw_pa 字段。

## 5. JSON 字段

SQLite 可使用 TEXT 存储 JSON，用于：

- 默认转会路径
- 事件条件
- 战术配置
- 新闻参数

# 06. 比赛模拟引擎规则

本文档定义 V1 比赛模拟逻辑。目标是可信、可调、可解释，而不是一开始追求复杂物理模拟。

---

# 一、比赛模拟目标

比赛结果应满足：

1. 强队更容易赢，但弱队有爆冷空间。
2. 战术选择会影响比赛走势。
3. 球星能改变比赛。
4. 伤病、体能、士气影响表现。
5. 主客场有差异。
6. 定位球有实际价值。
7. 赛后数据要与比分基本匹配。

---

# 二、比赛输入

一场比赛需要输入：

- 主队阵容
- 客队阵容
- 主队战术
- 客队战术
- 球员属性
- 球员体能
- 球员士气
- 伤病和停赛
- 主客场
- 天气，可后期加入
- 赛事重要性
- 德比或淘汰赛压力

---

# 三、球队评分

## 1. 进攻评分

```text
进攻评分 =
前锋射门均值 × 0.25
+ 边锋/前腰创造力 × 0.20
+ 中场传球与视野 × 0.20
+ 速度与突破 × 0.10
+ 战术适配 × 0.10
+ 球员状态 × 0.10
+ 核心球员加成 × 0.05
```

## 2. 防守评分

```text
防守评分 =
中卫防守与站位 × 0.25
+ 后腰拦截 × 0.15
+ 门将能力 × 0.20
+ 边后卫防守 × 0.10
+ 阵型防守加成 × 0.10
+ 体能状态 × 0.10
+ 战术纪律 × 0.10
```

## 3. 中场控制

```text
中场控制 =
中场传球 × 0.25
+ 控球技术 × 0.20
+ 视野 × 0.15
+ 决断 × 0.15
+ 团队合作 × 0.10
+ 战术适配 × 0.10
+ 状态 × 0.05
```

## 4. 主场优势

基础主场优势：3%-8%。

影响因素：

- 球场容量
- 球迷满意度
- 德比战
- 欧冠夜
- 对手客场适应

---

# 四、战术克制

## 1. 控球组织

优势：压制弱队，提升传球成功率。  
劣势：面对快速反击时有被打身后风险。

加成：

- 中场控制 +8%
- 射门数量 +5%
- 被反击风险 +5%

## 2. 快速反击

优势：克制高位压迫和高防线。  
劣势：控球率低，面对低位防守机会少。

加成：

- 反击机会 +12%
- 单刀概率 +8%
- 控球率 -8%

## 3. 高位压迫

优势：提高抢断和前场机会。  
劣势：体能消耗大，后场空间大。

加成：

- 前场抢断 +12%
- 对手失误 +8%
- 体能消耗 +15%
- 被身后球机会 +6%

## 4. 防守反击

优势：弱队保分、杯赛爆冷。  
劣势：落后后追分困难。

加成：

- 防守稳定 +10%
- 反击质量 +8%
- 射门数量 -10%

## 5. 边路传中

加成：

- 边路进攻 +10%
- 高中锋头球机会 +12%
- 对方强中卫时效率下降

## 6. 中路渗透

加成：

- 前腰和中场创造力 +10%
- 禁区前沿机会 +8%
- 面对密集中路防守效率下降

## 7. 长传冲吊

加成：

- 身体对抗 +8%
- 第二落点机会 +8%
- 传球成功率下降

## 8. 巨星自由发挥

加成：

- 核心球员关键事件 +12%
- 团队稳定 -5%
- 防守纪律 -5%

---

# 五、xG 生成

## 1. 基础 xG

```text
主队基础 xG = 主队进攻评分 / 客队防守评分 × 1.35 × 主场系数
客队基础 xG = 客队进攻评分 / 主队防守评分 × 1.20
```

## 2. 调整项

- 体能差异
- 士气差异
- 红牌
- 伤病
- 战术克制
- 比赛重要性
- 天气
- 主客场

## 3. 最终 xG 限制

为了避免极端数据：

- 最低 xG：0.1
- 常规最高 xG：4.5
- 特殊大胜最高 xG：7.0

# 07. 球员成长、训练、伤病与退役算法

---

# 一、球员生命周期

球员生命周期分为：

| 年龄 | 阶段 | 说明 |
|---:|---|---|
| 14-16 | 青训苗子 | 属性波动大，潜力判断不稳定 |
| 17-19 | 青年突破 | 成长最快，需要比赛时间 |
| 20-23 | 一线队成型 | 需要稳定出场与训练 |
| 24-28 | 巅峰形成 | 能力稳定上升 |
| 29-32 | 巅峰维持 | 经验提升，身体开始下降 |
| 33+ | 下滑期 | 身体下降，精神属性保留 |

---

# 二、成长总公式

```text
月度成长值 =
潜力空间 × 0.20
+ 职业态度 × 0.20
+ 出场时间 × 0.20
+ 训练质量 × 0.15
+ 教练能力 × 0.10
+ 联赛强度 × 0.05
+ 士气 × 0.05
+ 导师影响 × 0.05
- 伤病影响 × 0.15
```

其中：

```text
潜力空间 = 当前潜力 - 当前能力
```

潜力空间越大，成长可能性越高。

---

# 三、成长影响因素

## 1. 潜力

潜力不是绝对保证，只代表上限。球员需要通过训练、比赛、职业态度兑现潜力。

## 2. 职业态度

职业态度影响长期成长。

高职业态度：

- 成长稳定
- 不容易迷失
- 伤后恢复更好
- 巅峰期更长

低职业态度：

- 容易训练懈怠
- 成长断档
- 媒体吹捧后下滑
- 更衣室风险高

## 3. 出场时间

年轻球员需要比赛。

- 17-19 岁：适量一线队或租借最重要。
- 20-23 岁：需要稳定首发或轮换。
- 长期坐板凳会浪费潜力。

## 4. 联赛强度

联赛强度越高，成长经验越好，但压力和伤病风险也更高。

## 5. 战术适配

球员在适合的战术中更容易成长。

例如：

- 技术型中场适合控球体系。
- 速度型边锋适合反击体系。
- 高中锋适合边路传中或长传冲吊。

---

# 四、训练系统

## 1. 全队训练

类型：

- 平衡训练
- 体能训练
- 战术训练
- 进攻训练
- 防守训练
- 控球训练
- 定位球训练
- 恢复训练

## 2. 个人训练

专项：

- 射门
- 盘带
- 速度
- 身体对抗
- 弱脚
- 新位置
- 防守意识
- 传球视野
- 任意球
- 体能恢复

## 3. 训练强度

- 低强度：成长慢，伤病低，体能恢复快。
- 标准：平衡。
- 高强度：成长快，伤病风险高，体能消耗大。
- 恢复型：用于密集赛程。

---

# 五、导师系统

## 1. 导师条件

导师需要：

- 年龄较大
- 声望较高
- 职业态度较好
- 领导力较高
- 同位置或相近位置最佳

## 2. 导师影响

正面影响：

- 职业态度提升
- 稳定性提升
- 领导力提升
- 战术理解提升
- 更衣室融入提升

负面影响：

- 坏性格传染
- 年轻人不满
- 老将影响派系

## 3. 导师组合例子

- 马尔蒂尼指导年轻中卫。
- 皮尔洛指导年轻中场。
- 劳尔指导年轻前锋。
- 卡恩指导年轻门将。

---

# 六、伤病系统

## 1. 伤病类型

| 类型 | 时间 | 影响 |
|---|---:|---|
| 轻伤 | 3-7 天 | 轻微体能下降 |
| 肌肉伤 | 2-6 周 | 速度、爆发短期下降 |

| 疲劳伤 | 1-4 周 | 赛程密集触发 |
| 复发伤 | 不固定 | 与旧伤相关 |

## 2. 伤病概率

```text

+ 年龄因素 × 0.10

```

- 复发风险上升。
- 球员可能不满或感谢。
- 长期伤病倾向可能上升。

## 4. 手术

# 六、真实路径与打断逻辑

- Sporting CP 获得不同资金。

# 五、ID 映射

由于不同数据源球员 ID 不同，必须建立内部 ID。

## 1. player_id 生成规则

优先使用人工映射。若无映射，则根据：

- 姓名
- 出生日期
- 国籍
- 位置

匹配生成。

## 2. club_id 生成规则

根据：

- 俱乐部名称
- 国家
- 城市
- 创建年份

进行映射。

## 3. 去重规则

疑似重复条件：

- 姓名相同 + 出生日期相同
- 姓名相似 + 国籍相同 + 出生年份相同
- 数据源 ID 不同但履历相同

重复数据需要输出到 duplicate_report.csv。

# 16. 资源素材规范

---

# 一、资源类型

游戏本地资源包括：

- 球员头像
- 俱乐部队徽
- 球衣
- 球场图片
- 国家旗帜
- 赛事图标
- 新闻默认图
- UI 图标

---

# 二、目录结构

```text
assets/
├── portraits/
│   ├── players/
│   └── default_player.png
├── logos/
│   ├── clubs/
│   ├── competitions/
│   └── default_club.png
├── kits/
│   ├── club_id_home_year.png
│   └── club_id_away_year.png
├── stadiums/
├── flags/
├── news/
└── ui/
```

---

# 三、命名规范

## 1. 球员头像

```text
player_{player_id}.png
```

示例：

```text
player_100001.png
```

## 2. 队徽

```text
club_{club_id}.png
```

## 3. 球衣

```text
club_{club_id}_{season_id}_home.png
club_{club_id}_{season_id}_away.png
club_{club_id}_{season_id}_third.png
```

## 4. 国家旗帜

```text
flag_{country_code}.png
```

---

# 四、尺寸建议

| 类型 | 建议尺寸 |
|---|---|
| 球员头像 | 256x256 或 512x512 |
| 队徽 | 256x256 |
| 球衣 | 512x512 |
| 球场图 | 1280x720 |
| 国家旗帜 | 128x88 |
| 赛事图标 | 256x256 |

---

# 五、缺省资源

必须准备默认资源：

- default_player.png
- default_club.png
- default_kit.png
- default_stadium.png
- default_flag.png

资源缺失时不能崩溃。

player.portrait_path 示例：

```text
portraits/players/player_100001.png
```

club.logo_path 示例：

```text
logos/clubs/club_200001.png
```

# 七、个人单机使用说明

本项目默认个人本地使用。真实头像、队徽、球衣等素材如使用，应作为本地资源包，不做公开上架、不做商业分发。

# 八、UI 风格建议

整体风格：

- 深色足球经理后台感
- 绿色球场色为主
- 金色用于荣誉和重点数据
- 卡片式信息层级
- 数据表格清晰可筛选

主色建议：

- 深绿：#0B3D2E
- 草地绿：#1E7A4D
- 金色：#C9A45C
- 背景深灰：#121614
- 卡片深灰：#1D2420
- 文字白：#F5F5F0

1. 所有资源路径可解析。
