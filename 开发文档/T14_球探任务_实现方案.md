# T14 球探任务 - 详细实现方案

> 任务编号：T14
> 周期：W50-W52（3 周）
> 优先级：P0（C 轨核心，依赖 T10 转会搜索；下游 T15 历史新星池强依赖）
> 依据：V0.1 `08_青训_球探_历史新星系统.md` §三全球球探系统 + §四球探报告信息解锁 + §七青年赛事发现
> 铁律：球探任务随日期推进 / 报告等级随观察时长逐步解锁 / 所有参数配置化 / 球探可发现历史新星
> 定位：C 轨差异化卖点的入口，玩家通过球探发现隐藏天才与历史新星，并产出影响转会搜索与谈判的报告

## 一、业务目标与范围

### 1. 业务目标

1. 让玩家能够雇佣球探、派遣任务到 15 个地区，按 8 种任务类型进行球员侦察。
2. 球探任务随 T07 每日推进逐日推进，到期产出报告；报告按观察时长分 5 级逐步解锁信息。
3. 球探可发现普通球员与历史新星（与 T15 衔接），发现后产出初次报告并加入观察名单。
4. 球探报告等级直接影响 T10 转会搜索的球员卡片显示精度（CA/PA 区间、隐藏标签、签约难度等）。
5. 青年赛事（U17/U20 等 5 项）作为特殊任务类型，可触发小妖帽子戏法、豪门争夺等事件。

### 2. V1 范围（做）

- 球探雇佣系统（球探池 + 工资预算）
- 8 种任务类型派遣 + 30/60/90 天周期 + 低/中/高预算
- 15 个地区知识独立维护
- 7 因子发现概率公式
- 5 级报告信息解锁
- 球探中心页 UI（二级页面，首页快捷入口进入）
- 与 T07 每日推进集成（球探推进作为每日推进子任务）
- 与 T15 历史新星池集成（球探任务触发历史新星发现）
- 与 T10 转会搜索集成（报告等级影响搜索结果显示）
- 青年赛事任务 + 4 类事件
- 观察名单（与 T10 观察名单复用）

### 3. V1 不做（推迟）

- 球探知识随任务执行自动提升（V1 仅随事件/年资缓慢提升）
- 球探多语言/签证等经营细节
- 球探个人剧情
- 球探被其他俱乐部挖角
- 全 3D 地区地图（V1 用 2D 区域图）

## 三、数据模型（Room Entity）

球探数据遵循三库分离原则：
- **history.db**（只读）：`scout_template`（可雇佣球探池模板，由数据包提供）
- **save.db**（存档）：`scout`、`scout_region_knowledge`、`scout_task`、`scout_report`、`scout_event`（玩家存档私有状态）
- **cache.db**：`scout_report_index`（报告检索索引，加速 T10 搜索联查）

```kotlin
// core/ScoutManager.kt
class ScoutManager(
    private val saveDb: SaveDatabase,
    private val historyDb: HistoryDatabase,
    private val config: ScoutConfig
) {

    /**
     * 雇佣球探（从 history.scout_template 拷贝到 save.scout）
     * 限制：每俱乐部球探数 ≤ config.maxScoutsPerClub（默认 6）
     */
    suspend fun hireScout(saveId: Int, clubId: Int, templateId: Int, currentDate: LocalDate): HireResult {
        val currentCount = saveDb.scoutDao().countByClub(saveId, clubId)
        if (currentCount >= config.maxScoutsPerClub) {
            return HireResult(false, "球探数量已达上限 ${config.maxScoutsPerClub}")
        }

        val template = historyDb.scoutTemplateDao().getById(templateId)
            ?: return HireResult(false, "球探模板不存在")

        // 创建球探
        val scout = ScoutEntity(
            saveId = saveId,
            clubId = clubId,
            templateId = templateId,
            name = template.name,
            nationality = template.nationality,
            age = template.age,
            currentAbilityJudgment = template.currentAbilityJudgment,
            potentialJudgment = template.potentialJudgment,
            adaptability = template.adaptability,
            network = template.network,
            negotiationAssist = template.negotiationAssist,
            reputation = template.reputation,
            wage = template.wageBase,
            hiredDate = currentDate.toString(),
            contractExpireDate = currentDate.plusYears(2).toString()
        )
        val scoutId = saveDb.scoutDao().insert(scout).toInt()

        // 初始化地区知识（从模板 JSON 解析）
        val regionKnowledge = parseRegionKnowledgeJson(template.regionKnowledgeJson)
        for ((region, value) in regionKnowledge) {
            saveDb.scoutDao().insertRegionKnowledge(
                RegionKnowledgeEntity(
                    saveId = saveId, scoutId = scoutId,
                    regionCode = region, knowledgeValue = value
                )
            )
        }

        return HireResult(true, "雇佣成功", scoutId)
    }

    /** 解雇球探：取消进行中任务，删除地区知识，标记为 RELEASED */
    suspend fun releaseScout(saveId: Int, scoutId: Int, currentDate: LocalDate) {
        saveDb.scoutDao().cancelTasksByScout(saveId, scoutId, currentDate.toString())
        saveDb.scoutDao().deleteRegionKnowledge(saveId, scoutId)
        saveDb.scoutDao().updateStatus(saveId, scoutId, "RELEASED")
    }

    /** 获取球探对某地区的知识值（无记录返回 config.baseRegionKnowledge） */
    suspend fun getRegionKnowledge(saveId: Int, scoutId: Int, regionCode: String): Int {
        return saveDb.scoutDao().getRegionKnowledge(saveId, scoutId, regionCode)
            ?: config.baseRegionKnowledge
    }

    /** 列出某俱乐部所有可用球探（含地区知识汇总） */
    suspend fun listScouts(saveId: Int, clubId: Int): List<ScoutWithKnowledge> {
        val scouts = saveDb.scoutDao().getByClub(saveId, clubId)
        return scouts.map { scout ->
            val knowledgeList = saveDb.scoutDao().getKnowledgeByScout(saveId, scout.scoutId)
            ScoutWithKnowledge(scout, knowledgeList)
        }
    }
}

data class HireResult(val success: Boolean, val message: String, val scoutId: Int? = null)
data class ScoutWithKnowledge(val scout: ScoutEntity, val knowledge: List<RegionKnowledgeEntity>)
```

### 2. ScoutTaskExecutor - 任务推进（每日）

```kotlin
// core/ScoutTaskExecutor.kt
class ScoutTaskExecutor(
    private val saveDb: SaveDatabase,
    private val discoveryCalculator: DiscoveryCalculator,
    private val reportBuilder: ScoutReportBuilder,
    private val youthScanner: YouthTournamentScanner,
    private val prospectBridge: ProspectDiscoveryBridge,
    private val config: ScoutConfig
) {

    /**
     * 由 T07 每日推进调用（V0.1 03 §四.2 每日推进流程第 4 步：球探推进）
     * 性能：≤50ms / 球探
     */
    suspend fun advanceDaily(ctx: AdvanceContext): List<ScoutResult> {
        val results = mutableListOf<ScoutResult>()
        val activeTasks = saveDb.scoutDao().getInProgressTasks(ctx.saveId)

        for (task in activeTasks) {
            val taskStart = System.currentTimeMillis()

            // 1. 递增已推进天数
            val newElapsed = task.elapsedDays + 1
            val isCompleted = newElapsed >= task.durationDays

            // 2. 触发发现判定（每个 tick 概率性发现）
            val discoveries = discoveryCalculator.tryDiscover(task, ctx)
            for (discovery in discoveries) {
                // 3. 生成初次报告（等级 1）
                val report = reportBuilder.generateInitialReport(task, discovery, ctx.currentDate)
                saveDb.scoutDao().insertReport(report)

                // 4. 自动加入观察名单（由 T10 watchlist 复用）
                saveDb.scoutDao().insertWatchListEntry(
                    WatchListEntryEntity(
                        saveId = ctx.saveId, clubId = task.clubId,
                        playerId = discovery.playerId, addedDate = ctx.currentDate.toString(),
                        sourceTaskId = task.taskId
                    )
                )

                // 5. 历史新星发现衔接 T15
                if (discovery.isHistoricalProspect) {
                    prospectBridge.onProspectDiscovered(
                        ctx.saveId, discovery.playerId, task.scoutId, ctx.currentDate
                    )
                }

                results.add(ScoutResult.Discovered(discovery, report))
            }

            // 6. 报告升级判定（按 observationDays 升级现有报告）
            reportBuilder.tryUpgradeReports(task, ctx.currentDate)

            // 7. 青年赛事任务：扫描事件
            if (task.taskType == ScoutTaskType.YOUTH_TOURNAMENT.name) {
                val events = youthScanner.scan(task, ctx.currentDate)
                for (event in events) {
                    saveDb.scoutDao().insertScoutEvent(event)
                    results.add(ScoutResult.YouthEvent(event))
                }
            }

            // 8. 更新任务状态
            if (isCompleted) {
                saveDb.scoutDao().updateTaskStatus(
                    ctx.saveId, task.taskId, "COMPLETED", ctx.currentDate.toString()
                )
                // 释放球探
                saveDb.scoutDao().updateScoutStatus(ctx.saveId, task.scoutId, "IDLE")
                results.add(ScoutResult.TaskCompleted(task))
            } else {
                saveDb.scoutDao().updateTaskElapsed(
                    ctx.saveId, task.taskId, newElapsed, ctx.currentDate.toString()
                )
            }

            // 性能保护：单球探推进超过阈值记录慢日志
            val cost = System.currentTimeMillis() - taskStart
```

### 3. DiscoveryCalculator - 发现概率计算

```kotlin
// core/DiscoveryCalculator.kt
class DiscoveryCalculator(
    private val saveDb: SaveDatabase,
    private val historyDb: HistoryDatabase,
    private val densityProvider: RegionTalentDensityProvider,
    private val prospectBridge: ProspectDiscoveryBridge,
    private val config: ScoutConfig
) {

    /**
     * V0.1 08 §三.5 发现概率公式（7 因子）
     * 每日推进时按概率发现 0-N 名球员
     */
    suspend fun tryDiscover(task: ScoutTaskEntity, ctx: AdvanceContext): List<Discovery> {
        val discoveries = mutableListOf<Discovery>()
        val scout = saveDb.scoutDao().getById(task.scoutId) ?: return discoveries

        // 1. 候选池：地区内未被发现且符合任务条件的球员
        val candidates = getCandidates(task, ctx)
        if (candidates.isEmpty()) return discoveries

        // 2. 任务发现上限（每日 ≤ config.maxDiscoveriesPerDay）
        var foundToday = 0

        for (candidate in candidates.shuffled()) {
            if (foundToday >= config.maxDiscoveriesPerDay) break

            // 3. 已有报告则跳过（避免重复发现）
            val existing = saveDb.scoutDao().getReportByPlayer(ctx.saveId, candidate.playerId, task.clubId)
            if (existing != null) continue

            // 4. 计算发现概率
            val probability = calculateProbability(scout, task, candidate)
            if (Random.nextDouble() < probability) {
                discoveries.add(Discovery(
                    playerId = candidate.playerId,
                    playerName = candidate.name,
                    playerAge = candidate.age,
                    playerPosition = candidate.position,
                    playerRegion = candidate.regionCode,
                    isHistoricalProspect = candidate.isHistoricalProspect,
                    probability = probability
                ))
                foundToday++
            }
        }

        return discoveries
    }

    /**
     * 发现概率 =
     *   地区知识 × 0.25
     * + 潜力判断 × 0.20
     * + 人脉 × 0.15
     * + 任务匹配 × 0.15
     * + 预算 × 0.10
     * + 地区人才密度 × 0.10
     * + 随机因素 × 0.05
     */
    private suspend fun calculateProbability(
        scout: ScoutEntity,
        task: ScoutTaskEntity,
        candidate: CandidatePlayer
    ): Double {
        val regionKnowledge = saveDb.scoutDao()
            .getRegionKnowledge(scout.saveId, scout.scoutId, task.regionCode)
            ?.knowledgeValue ?: config.baseRegionKnowledge
        val potentialJudgment = scout.potentialJudgment
        val network = scout.network

        val taskMatch = calculateTaskMatch(task, candidate)

        val budgetFactor = BudgetLevel.valueOf(task.budgetLevel).displayFactor

        val regionDensity = densityProvider.getDensity(task.regionCode)

        val randomFactor = Random.nextDouble()

        val probability = (
            (regionKnowledge / 100.0) * 0.25 +
            (potentialJudgment / 20.0) * 0.20 +
            (network / 20.0) * 0.15 +
            taskMatch * 0.15 +
            budgetFactor * 0.10 +
            regionDensity * 0.10 +
            randomFactor * 0.05
        )

        return probability.coerceIn(0.0, config.maxProbabilityPerAttempt)  // 单次 ≤ 0.8
    }

    /** 任务匹配度：候选球员与任务类型/位置/年龄的契合度（0-1） */
    private fun calculateTaskMatch(task: ScoutTaskEntity, candidate: CandidatePlayer): Double {
        var match = 0.5  // 基础
        when (ScoutTaskType.valueOf(task.taskType)) {
            ScoutTaskType.POSITION_SEARCH -> {
                if (candidate.position == task.targetPosition) match = 1.0
                else if (samePositionGroup(candidate.position, task.targetPosition)) match = 0.6
                else match = 0.2
            }
            ScoutTaskType.AGE_GROUP_SEARCH -> {
                if (candidate.age in task.ageMin..task.ageMax) match = 1.0 else match = 0.2
            }
            ScoutTaskType.CLUB_OBSERVATION -> {
                if (candidate.clubId == task.targetClubId) match = 1.0 else match = 0.1
            }
            ScoutTaskType.CONTRACT_OPPORTUNITY -> {
                if (candidate.monthsLeft <= 12) match = 1.0
                else if (candidate.monthsLeft <= 24) match = 0.7 else match = 0.3
            }
            ScoutTaskType.LOWER_LEAGUE -> {
                if (candidate.leagueTier >= 2) match = 1.0 else match = 0.3
            }
            ScoutTaskType.STAR_TRACKING -> {
                if (candidate.reputation >= 70) match = 1.0 else match = 0.4
            }
            ScoutTaskType.REGION_SEARCH -> match = 0.8  // 广撒网基础匹配
            ScoutTaskType.YOUTH_TOURNAMENT -> {
                if (candidate.age <= 20) match = 1.0 else match = 0.2
            }
        }
        // 年龄范围叠加过滤
        if (candidate.age !in task.ageMin..task.ageMax && task.ageMin > 0) match *= 0.3
        return match
    }

    /** 候选池：地区内球员 + 历史新星池（T15 提供） */
    private suspend fun getCandidates(task: ScoutTaskEntity, ctx: AdvanceContext): List<CandidatePlayer> {
        val normalCandidates = saveDb.scoutDao()
            .findPlayersByRegion(task.saveId, task.regionCode, task.ageMin, task.ageMax)
            .map { it.toCandidate() }

        // 历史新星候选（T15 提供）
        val prospectCandidates = prospectBridge
            .getDiscoverableProspectsInRegion(ctx.saveId, task.regionCode, ctx.currentDate)
            .map { it.toCandidate() }

        return normalCandidates + prospectCandidates
    }
}
```

```kotlin
     */
    suspend fun tryUpgradeReports(task: ScoutTaskEntity, currentDate: LocalDate) {
        // 获取该任务关联的报告（每名球员一份报告，按 observationDays 升级）
        val reports = saveDb.scoutDao().getReportsByTask(task.saveId, task.taskId)
        val scout = saveDb.scoutDao().getById(task.scoutId) ?: return

        for (report in reports) {
            val newObservationDays = report.observationDays + 1
            val targetLevel = calculateTargetLevel(newObservationDays, scout, task)

            if (targetLevel > report.reportLevel) {
                val upgraded = upgradeReport(report, targetLevel, scout, task, currentDate)
                saveDb.scoutDao().updateReport(upgraded)
            } else {
                saveDb.scoutDao().incrementObservationDays(
                    task.saveId, report.reportId, newObservationDays, currentDate.toString()
                )
            }
        }
    }

    /**
     * 报告升级阈值（按观察天数 + 球探能力）
     * 等级 2：累计观察 ≥ upgradeThresholdL2 天（默认 14）
     * 等级 3：累计观察 ≥ upgradeThresholdL3 天（默认 35）
     * 等级 4：累计观察 ≥ upgradeThresholdL4 天（默认 60）
     * 等级 5：累计观察 ≥ upgradeThresholdL5 天（默认 90）+ 球探潜力判断 ≥ 15
     */
    private fun calculateTargetLevel(
        observationDays: Int,
        scout: ScoutEntity,
        task: ScoutTaskEntity
    ): Int {
        var level = 1
        if (observationDays >= config.upgradeThresholdL2) level = 2
        if (observationDays >= config.upgradeThresholdL3) level = 3
        if (observationDays >= config.upgradeThresholdL4) level = 4
        // 等级 5 需要球探潜力判断达标
        if (observationDays >= config.upgradeThresholdL5 &&
            scout.potentialJudgment >= config.upgradeL5PotentialJudgmentMin) {
            level = 5
        }
        // 预算加成：高预算减 10% 阈值
        if (task.budgetLevel == BudgetLevel.HIGH.name) {
            // 已隐含在阈值配置中，此处不再二次计算
        }
        return level
    }

    /** 升级报告：按目标等级解锁对应字段 */
    private suspend fun upgradeReport(
        report: ScoutReportEntity,
        targetLevel: Int,
        scout: ScoutEntity,
        task: ScoutTaskEntity,
        currentDate: LocalDate
    ): ScoutReportEntity {
        val player = historyDb.playerDao().getById(report.playerId)
        val builder = report.copy(
            reportLevel = targetLevel,
            lastUpdatedDate = currentDate.toString(),
            observationDays = report.observationDays + 1
        )

        return when (targetLevel) {
            2 -> {
                // CA/PA 区间（受球探潜力判断准确度影响）
                val (caLow, caHigh) = computeRange(player.currentAbility, scout.currentAbilityJudgment, spread = 12)
                val (paLow, paHigh) = computeRange(player.potentialAbility, scout.potentialJudgment, spread = 15)
                builder.copy(
                    caRangeLow = caLow, caRangeHigh = caHigh,
                    paRangeLow = paLow, paRangeHigh = paHigh,
                    strengths = deriveStrengths(player).toJson(),
                    risks = deriveRisks(player, scout).toJson()
                )
            }
            3 -> {
                val (caLow, caHigh) = computeRange(player.currentAbility, scout.currentAbilityJudgment, spread = 7)
                val (paLow, paHigh) = computeRange(player.potentialAbility, scout.potentialJudgment, spread = 9)
                builder.copy(
                    caNarrowLow = caLow, caNarrowHigh = caHigh,
                    paNarrowLow = paLow, paNarrowHigh = paHigh,
                    personality = player.personality,
                    signDifficulty = computeSignDifficulty(player, task)
                )
            }
            4 -> {
                builder.copy(
                    growthSpeed = player.growthSpeedLabel,
                    hiddenTags = player.hiddenTagsJson,
                    tacticalFit = deriveTacticalFit(player).toJson()
                )
            }
            5 -> {
                builder.copy(
                    realPa = player.potentialAbility,
                    injuryProneness = player.injuryProneness,
                    professionalism = player.professionalism
                )
            }
            else -> builder
        }
    }

    /** CA/PA 区间计算：球探能力越低，spread 越大 */
    private fun computeRange(realValue: Int, scoutAbility: Int, spread: Int): Pair<Int, Int> {
        val accuracy = scoutAbility / 20.0  // 0-1
        val actualSpread = (spread * (2.0 - accuracy)).toInt().coerceAtLeast(2)
        val low = (realValue - actualSpread).coerceAtLeast(1)
        val high = (realValue + actualSpread).coerceAtMost(200)
        return Pair(low, high)
    }

    private fun computeSignDifficulty(player: Player, task: ScoutTaskEntity): Int {
        // 简化：声望 + 合同剩余 + 俱乐部意愿
        val base = player.reputation.coerceIn(0, 100)
        return base
    }
```

```
            (caLow, caHigh) = computeRange(player.CA, scout.currentAbilityJudgment, spread=12)
            (paLow, paHigh) = computeRange(player.PA, scout.potentialJudgment, spread=15)
            unlock: caRangeLow, caRangeHigh, paRangeLow, paRangeHigh, strengths, risks
        case 3:
            # 较窄 CA/PA + 性格 + 签约难度
            (caLow, caHigh) = computeRange(player.CA, scout.currentAbilityJudgment, spread=7)
            (paLow, paHigh) = computeRange(player.PA, scout.potentialJudgment, spread=9)
            unlock: caNarrowLow, caNarrowHigh, paNarrowLow, paNarrowHigh, personality, signDifficulty
        case 4:
            # 成长速度 + 隐藏标签 + 适配战术
            unlock: growthSpeed, hiddenTags, tacticalFit
        case 5:
            # 真实潜力 + 伤病倾向 + 职业态度
            unlock: realPa, injuryProneness, professionalism
    update: reportLevel, lastUpdatedDate, observationDays

function computeRange(realValue, scoutAbility, spread):
    accuracy = scoutAbility / 20                      # 0-1
    actualSpread = max(2, int(spread * (2 - accuracy)))
    return (max(1, realValue - actualSpread), min(200, realValue + actualSpread))
```

## 七、交互流程

### 1. 球探发现主流程（V0.1 03 §四.4）

```
1. 进入球探中心（首页快捷入口）
2. 查看球探列表 → 选择空闲球探
3. 点击"派遣任务" → 打开 ScoutTaskDispatchDialog
4. 选择任务类型（8 种）
5. 选择地区（15 地区，显示该球探对该地区知识值）
6. 设置参数（位置/年龄/周期 30-60-90/预算 低-中-高）
7. 确认派遣 → 扣预算 → 球探状态置为 ON_TASK
8. 每日推进（T07）→ ScoutService.advanceDaily()
   ├── 任务 elapsedDays++
   ├── DiscoveryCalculator.tryDiscover() 概率性发现
   ├── 发现球员 → ScoutReportBuilder.generateInitialReport()（等级 1）
   ├── 自动加入观察名单
   ├── 历史新星 → ProspectDiscoveryBridge → T15 更新星状态
   └── ScoutReportBuilder.tryUpgradeReports() 升级现有报告
9. 报告升级 → 等级 2/3/4/5 按 observationDays 解锁
10. 玩家在"最新报告" Tab 查看报告
11. 点击报告 → ScoutReportDetailScreen
12. 球探推荐 → 设置推荐星级
13. "提交报价" → 跳转 T10/T11 转会流程
14. 任务到期 → 状态置 COMPLETED → 球探 IDLE
```

```
球探状态：
  IDLE ──派遣──→ ON_TASK ──任务完成/取消──→ IDLE
  IDLE ──解雇──→ RELEASED

任务状态：
  IN_PROGRESS ──到期──→ COMPLETED
  IN_PROGRESS ──取消──→ CANCELLED

报告等级：
  L1(初次发现) ──14天──→ L2(粗略)
  L2 ──35天──→ L3(标准)
  L3 ──60天──→ L4(深度)
  L4 ──90天 + 潜力判断≥15──→ L5(完全掌握)
```

## 九、性能要求

| 场景 | 指标 | 实现要点 |
|------|------|----------|
| 单球探任务每日推进 | ≤ 50ms | 候选池查询走 `idx(save_id, region_code)`；候选数 ≤ 200；DB 批量更新 |
| 报告生成（初次） | ≤ 100ms | 单次插入；属性推导在内存完成，不重复查 DB |
| 报告升级 | ≤ 50ms / 报告 | 仅更新差异字段，使用 Room `@Update` |
| 球探列表加载 | ≤ 300ms | 球探 + 地区知识一次性联查（`@Relation`） |
| 候选池查询 | ≤ 100ms | 走 `idx(save_id, region_code)` + 年龄范围索引 |
| 青年赛事事件扫描 | ≤ 30ms / 任务 | 仅在赛事举办月份触发，事件概率预计算 |
| 整体每日推进中球探子任务 | ≤ 500ms / 俱乐部 | 6 球探 × 50ms = 300ms，留余量 |

性能保护：
- 候选池上限 200，超出按随机采样
- 每日发现上限 `max_discoveries_per_day = 3`，防止报告暴增
- 报告表索引：`(save_id, player_id)`、`(save_id, report_level)`、`(save_id, created_date)`
- 慢操作写入 `perf_log` 表（V0.2 14_SQL 已定义）

| T03 存档 | 球探/任务/报告持久化 | save.db 的 `scout`/`scout_task`/`scout_report`/`scout_region_knowledge` 表 |
| T0X 参数配置 | 配置加载框架 | `scout_config.json` 走统一配置加载与热重载 |

### 2. 下游被依赖

| 下游任务 | 依赖点 | 衔接方式 |
|----------|--------|----------|
| T15 历史新星池 | 球探发现历史新星 | `ProspectDiscoveryBridge` 调用 `HistoricalProspectService.getActiveProspectsInRegion()` 取候选；发现后调用 `markProspectDiscovered()` 更新星状态 |
| T07 每日推进 | 球探推进作为子任务 | T07 `processDailyCheck()` 调用 `ScoutService.advanceDaily()`，在每日推进顺序第 4 步（V0.1 03 §四.2） |
| T10 转会搜索 | 报告等级影响搜索结果显示 | T10 球员卡片渲染时调用 `getReportForPlayer()`，按 `reportLevel` 决定显示 CA/PA 区间、隐藏标签、签约难度等 |
| T11 报价谈判 | 球探谈判辅助属性 | `scout.negotiationAssist` 在谈判难度计算中作为修正因子（T11 调用 `ScoutService` 查询俱乐部球探最高谈判辅助值） |
| T13/T18 AI 转会 | AI 球探报告 | AI 俱乐部可调用简化版 `ScoutService.advanceDaily()` 生成报告（共享核心算法，参数走 AI 配置） |

### 3. 集成点详解

**T15 衔接（球探发现历史新星）**：
- `DiscoveryCalculator.getCandidates()` 同时拉取普通球员 + `ProspectDiscoveryBridge` 提供的历史新星
- 发现历史新星后，`ProspectDiscoveryBridge.onProspectDiscovered()` 通知 T15 将星状态从 ACTIVE → DISCOVERED
- 报告 `isHistoricalProspect = 1` 标记，UI 显示特殊徽章
- 玩家可"继续观察"升级报告，或"提交报价"提前签约（触发 T15 提前签约打断 → T20 蝴蝶效应）

**T10 衔接（报告影响搜索结果）**：
- T10 转会搜索球员卡片渲染时，按 `reportLevel` 决定显示内容：
  - 无报告：CA/PA 显示"未知"
  - 等级 1：仅显示姓名/年龄/地区/位置/初步特点
  - 等级 2：显示 CA/PA 区间（±12）
  - 等级 3：显示较窄 CA/PA（±7）+ 性格 + 签约难度
  - 等级 4-5：显示隐藏标签、成长速度等
- `scout_report` 表索引 `(save_id, player_id)` 已在 V0.2 03 §六 中明确要求

**T07 衔接（每日推进子任务）**：
- T07 每日推进顺序（V0.1 03 §四.2）：训练 → 伤病恢复 → **球探推进** → 青训成长 → 转会处理 → AI 行动 → 新闻
- `ScoutService.advanceDaily(ctx: AdvanceContext)` 接收 T07 的推进上下文（saveId/currentDate/isTransferWindowOpen）
- 返回 `List<ScoutResult>` 由 T07 转换为新闻/待办事项

## 十四、关键风险与应对

### 1. 发现概率失衡

- **表现**：球探每天发现一堆球员，报告暴增
- **应对**：
  - 单次概率上限 0.8
  - 每日发现上限 3
  - 候选池上限 200，超出随机采样
  - 报告表索引保证查询性能

### 2. 报告升级过快

- **表现**：玩家短时间内拿到等级 5 报告，失去探索乐趣
- **应对**：
  - 等级 5 需 90 天 + 球探潜力判断 ≥ 15（双重门槛）
  - 阈值全部配置化，可调

### 3. 候选池过大导致性能下降

- **表现**：地区球员多，每日推进慢
- **应对**：
  - 走 `(save_id, region_code)` 索引
  - 候选池上限 200
  - 已发现球员自动跳过

### 4. T15 集成时序问题

- **表现**：球探发现历史新星时 T15 星状态未激活
- **应对**：
  - `ProspectDiscoveryBridge.getDiscoverableProspectsInRegion()` 只返回 ACTIVE 状态
  - T15 激活由 T07 每日推进在球探推进之前完成（T15 `processDailyCheck` 先于球探推进）

### 5. 青年赛事事件过多

- **表现**：举办月每日触发事件，刷屏
- **应对**：
  - 事件概率默认 4-6%
  - 单任务单日最多触发 1 类事件
  - `scout_event.read` 字段支持未读标记
