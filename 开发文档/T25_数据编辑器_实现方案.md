# T25 数据编辑器 - 详细实现方案

> 任务编号：T25
> 周期：W96-W100（5 周）
> 优先级：P2（D 轨工具类，辅助系统，不影响主游戏流程）
> 依赖：T00 三库结构（上游，history.db / save.db Entity 全集）/ T01 数据导入（上游，复用 CsvParser / DataValidator / EntityMapper / BatchWriter / HistoryDbWriter）/ T15 历史新星池（编辑 historical_prospect_pool 配置）/ T17 经济模型（编辑 economy_index + league_economy_profile 参数）/ T20 蝴蝶效应（编辑 transfer_history.was_interrupted + butterfly_transfers 基线）/ T03 存档系统（存档编辑模式复用 save.db）
> 下游：T0F-2 数据导入版权 UI（编辑后导出的数据包可被 T01 重新导入）/ 玩家自定义数据（玩家手动修正球员/俱乐部/剧本配置）
> 依据：V0.1 `03_页面结构与交互说明.md` §10 数据库编辑器页 + V0.1 `12_数据导入清洗规范.md` + V0.2 `02_数据来源治理与版本管理.md` + V0.2 `06_蝴蝶效应约束算法.md` + V0.2 `07_经济通胀_身价_工资模型.md` + V0.2 `08_球员成长_训练_伤病细化算法.md` + T01 `T01_数据导入_实现方案.md`（逆向复用）+ T00 `T00_三库结构与存档_实现方案.md`
> 铁律：编辑器是辅助工具，不能影响主游戏流程 / 实时编辑要谨慎（避免破坏存档，写入前必须校验）/ 撤销/重做要可靠（编辑历史持久化，避免崩溃丢失）/ 数据校验要严格（复用 T01 校验逻辑，防止玩家创建不合理数据）/ 导出格式必须兼容 T01 导入格式（CSV/JSON 字段对齐）/ 权限分级要清晰（普通/高级/开发者三档）/ 所有参数配置化（editor_config.json）/ 编辑 history.db 前必须自动备份

## 一、业务目标与范围

### 1. 业务目标

为玩家提供完整的游戏数据编辑能力，允许在游戏运行中或启动前自定义球员、俱乐部、联赛、转会、剧本、历史新星、蝴蝶效应、经济模型等全部游戏数据，并支持将编辑后的数据导出为 CSV/JSON 数据包供 T01 重新导入，或在存档内直接修改当前世界状态。编辑器是 T01 数据导入的逆向操作，复用 T01 的解析、校验、写入基础设施，是玩家数据自定义与本地问题修正的统一入口。

### 2. V1 做的范围

| 类别 | 编辑能力 | 目标表 | 编辑模式 |
|------|----------|--------|----------|
| 球员基础信息 | 姓名/国籍/位置/生日/惯用脚/身高体重/性格/退役年龄 | player | 离线 + 实时 + 存档 |
| 球员属性 | CA/PA + 30+ 属性（按赛季） | player_attributes | 离线 + 实时 + 存档 |
| 球员合同 | 合同到期/工资/身价/队内角色 | squad_membership + save_player_state | 离线 + 存档 |
| 俱乐部基础 | 名称/国家/城市/声望/成立年份/球场 | club | 离线 + 实时 + 存档 |
| 俱乐部财政 | 预算/收入/工资上限/财政等级 | save_club_state + club.finance_level | 存档 |
| 俱乐部设施 | 训练/青训/医疗等级 | club + youth_academy | 离线 + 存档 |
| 俱乐部阵容 | 添加/移除球员（按赛季） | squad_membership | 离线 + 存档 |
| 联赛结构 | 球队数/升降级规则/赛事类型 | competition + club_competition_season | 离线 |
| 联赛规则 | 外援限制/报名规则 | competition 规则字段 | 离线 |

| 历史转会路径 | 球员转会记录/蝴蝶基线标记 | transfer_history | 离线 |
| 蝴蝶效应配置 | 关键转会重要性/最大深度/替代决策池 | transfer_history.notes JSON + butterfly_transfers 元数据 | 离线（高级模式） |
| 历史新星配置 | 可发现时间/默认路径/标签/legend_level | historical_prospect_pool | 离线（高级模式） |

| 经济模型参数 | 通胀指数/联赛商业系数/俱乐部财力因子 | economy_index + league_economy_profile | 离线（开发者模式） |
| 数据导入 | CSV/JSON 数据包导入 | 全 history.db 表 | 离线（复用 T01） |
| 数据导出 | 当前数据导出为 CSV/JSON | 全 history.db 表 | 离线 |
| 数据包版本管理 | manifest 元信息查看/编辑 | data_pack_manifest | 离线 |
| 编辑历史 | 操作记录/撤销/重做/变更预览 | editor_edit_log（编辑器内部表） | 全模式 |
| 本地问题标注 | 球员详情页标记数据问题（来自 V0.2 §十） | local_data_issue | 实时 + 存档 |

### 3. V1 不做（推迟）

- **云端数据包同步**：V1 仅支持本地编辑与本地导入/导出，不联网
- **多存档批量编辑**：V1 只能编辑当前存档或 history.db，不支持跨存档批量修改
- **可视化数据编辑**：V1 不提供图表化编辑（如属性雷达图拖拽），仅表单输入
- **AI 数据平衡建议**：V1 仅做平衡性警告（如 CA 过高），不自动生成建议值
- **历史新星自动生成**：V1 仅编辑已有 historical_prospect_pool，不自动生成新球星
- **剧本事件编辑器**：V1 仅编辑剧本开关与参数，不编辑 historical_event 事件脚本
- **图片资源编辑**：V1 仅编辑路径字符串，不裁剪/生成图片
- **数据库结构迁移**：V1 不修改 schema，仅修改数据行
- **国家队数据编辑**：V1 不支持国家队系统数据编辑（国家队系统在 M7 后续扩展）
- **挑战剧本编辑器**：V1 不支持自定义挑战剧本（10_历史事件_蝴蝶效应_挑战剧本.md 中的挑战模式）
- **数据对比工具**：V1 不支持两份数据包的差异对比

### 2. 编辑器内部表（editor.db）

编辑器自身需要一个独立的小型 SQLite 数据库，用于持久化编辑历史与配置，避免污染游戏库。

```kotlin
// db/EditLogEntity.kt
@Entity(
    tableName = "editor_edit_log",
    indices = [
        Index(value = ["session_id"]),
        Index(value = ["target_table", "target_id"]),
        Index(value = ["created_at"])
    ]
)
data class EditLogEntity(
    @PrimaryKey(autoGenerate = true) val logId: Int,
    @ColumnInfo(name = "session_id") val sessionId: String,        // 编辑会话 UUID
    @ColumnInfo(name = "target_db") val targetDb: String,          // history / save_xxx
    @ColumnInfo(name = "target_table") val targetTable: String,    // player / club / ...
    @ColumnInfo(name = "target_id") val targetId: String,          // 主键值（字符串化）
    @ColumnInfo(name = "operation_type") val operationType: String,// INSERT / UPDATE / DELETE
    @ColumnInfo(name = "field_name") val fieldName: String?,       // 修改字段（批量修改时为 null）
    @ColumnInfo(name = "old_value_json") val oldValueJson: String?,// 修改前值
    @ColumnInfo(name = "new_value_json") val newValueJson: String?,// 修改后值
    @ColumnInfo(name = "permission_level") val permissionLevel: String, // 编辑时权限
    @ColumnInfo(name = "edit_mode") val editMode: String,          // OFFLINE / REALTIME / SAVE
    @ColumnInfo(name = "undoable") val undoable: Int = 1,          // 是否可撤销
    @ColumnInfo(name = "undone") val undone: Int = 0,              // 是否已撤销
    @ColumnInfo(name = "created_at") val createdAt: String,        // ISO 时间戳
    @ColumnInfo(name = "user_note") val userNote: String?          // 用户备注
)

// db/EditorSessionEntity.kt
@Entity(tableName = "editor_session")
data class EditorSessionEntity(
    @PrimaryKey @ColumnInfo(name = "session_id") val sessionId: String,
    @ColumnInfo(name = "start_time") val startTime: String,
    @ColumnInfo(name = "end_time") val endTime: String?,
    @ColumnInfo(name = "edit_mode") val editMode: String,
    @ColumnInfo(name = "permission_level") val permissionLevel: String,
    @ColumnInfo(name = "target_save_id") val targetSaveId: String?, // 存档编辑模式记录 save_id
    @ColumnInfo(name = "operation_count") val operationCount: Int = 0,
    @ColumnInfo(name = "backup_path") val backupPath: String?       // 本次编辑前的备份路径
)
```

### 3. 编辑器特有的状态管理（草稿态）

编辑器不直接修改数据库，而是在内存中维护「草稿态」Entity，用户确认保存后才写入。草稿态继承自原 Entity 并附加元信息。

```kotlin
// state/EditableEntity.kt
interface EditableEntity {
    val originalId: Any                // 原始主键
    val isModified: Boolean            // 是否被修改过
    val isDeleted: Boolean             // 是否被标记删除
    val isNew: Boolean                 // 是否新增
    fun diff(): FieldDiff              // 计算与原值的差异
}

data class FieldDiff(
    val fieldName: String,
    val oldValue: Any?,
    val newValue: Any?,
    val isModified: Boolean
)

// state/EditablePlayer.kt
data class EditablePlayer(
    val original: PlayerEntity?,        // null 表示新增
    val draft: PlayerEntity,            // 当前草稿值
    val attributesDraft: Map<Int, PlayerAttributesEntity>, // seasonId → 属性草稿
    override val isDeleted: Boolean = false
) : EditableEntity {
    override val originalId: Int get() = draft.playerId
    override val isNew: Boolean get() = original == null
    override val isModified: Boolean
        get() = original != null && original != draft || isDeleted

    override fun diff(): List<FieldDiff> {
        if (original == null) return listOf(FieldDiff("__new__", null, draft, true))
        val diffs = mutableListOf<FieldDiff>()
        if (original!!.realName != draft.realName)
            diffs.add(FieldDiff("real_name", original!!.realName, draft.realName, true))
        if (original!!.nationality != draft.nationality)
            diffs.add(FieldDiff("nationality", original!!.nationality, draft.nationality, true))
        // ... 其余字段逐一对比
        return diffs
    }
}
```

```kotlin
// api/DataEditorManager.kt
class DataEditorManager(
    private val context: Context,
    private val historyDb: HistoryDatabase,
    private val saveDb: SaveDatabase?,
    private val editorDb: EditorDatabase,
    private val config: EditorConfig,
    private val permissionManager: PermissionManager,
    private val historyManager: EditHistoryManager,
    private val ioDispatcher: CoroutineDispatcher = Dispatchers.IO
) {

    /**
     * 编辑模式（运行时确定）
     */
    private var currentMode: EditMode = EditMode.OFFLINE
    private var currentSessionId: String? = null

    /**
     * 开启编辑会话
     * @param mode 编辑模式
     * @return 会话 ID（持久化到 editor_session）
     */
    suspend fun startSession(mode: EditMode): String = withContext(ioDispatcher) {
        val sessionId = UUID.randomUUID().toString()
        currentMode = mode
        currentSessionId = sessionId

        // 1. 备份当前数据库（离线/存档模式）
        val backupPath = when (mode) {
            EditMode.OFFLINE -> BackupManager(context).backupHistoryDb()
            EditMode.SAVE -> BackupManager(context).backupSaveDb(saveDb!!)
            EditMode.REALTIME -> null // 实时编辑不备份整库，靠 checkpoint 兜底
        }

        // 2. 写入会话记录
        editorDb.editorDao().insertSession(EditorSessionEntity(
            sessionId = sessionId,
            startTime = LocalDateTime.now().format(ISO_LOCAL_DATE_TIME),
            endTime = null,
            editMode = mode.name,
            permissionLevel = permissionManager.currentLevel.name,
            targetSaveId = saveDb?.let { getCurrentSaveId(it) },
            operationCount = 0,
            backupPath = backupPath
        ))

        // 3. 若是 history.db 编辑，临时关闭只读
        if (mode == EditMode.OFFLINE) {
            HistoryDbAccessor(historyDb).openForWrite()
        }

        sessionId
    }

    /**
     * 结束编辑会话
     */
    suspend fun endSession(sessionId: String) = withContext(ioDispatcher) {
        // 1. 若是 history.db 编辑，恢复只读
        if (currentMode == EditMode.OFFLINE) {
            HistoryDbAccessor(historyDb).closeAndRestoreReadOnly()
        }

        // 2. 标记会话结束
        editorDb.editorDao().updateSessionEndTime(
            sessionId = sessionId,
            endTime = LocalDateTime.now().format(ISO_LOCAL_DATE_TIME)
        )

        // 3. 失效 cache.db
        CacheRebuilder(context).markAllDirty()

        currentSessionId = null
    }

    /**
     * 保存编辑（核心入口）
     * @param editable 可编辑 Entity 草稿
     * @param targetTable 目标表名
     * @return EditReport
     */
    suspend fun save(editable: EditableEntity, targetTable: String): EditReport = withContext(ioDispatcher) {
        val report = EditReport()
        val sessionId = currentSessionId ?: error("未开启编辑会话")

        // 1. 权限校验
        if (!permissionManager.canEdit(targetTable)) {
            return@withContext report.fail("权限不足：${permissionManager.currentLevel} 无法编辑 $targetTable")
        }

        // 2. 数据校验（复用 T01 DataValidator）
        val validationResult = EditorValidator(historyDb, config.validation).validate(editable, targetTable)
        if (!validationResult.isValid) {
            return@withContext report.fail("数据校验失败：${validationResult.errors.joinToString { it.message }}")
        }

        // 3. 平衡性检查（仅警告，不阻塞）
        val balanceWarnings = BalanceChecker(config.balance).check(editable, targetTable)
        report.warnings.addAll(balanceWarnings)

        // 4. 记录编辑命令到历史（保存前记录，便于撤销）
        val command = EditCommandFactory.create(editable, targetTable, currentMode)
        historyManager.recordCommand(sessionId, command)

        // 5. 写入数据库（按模式路由）
        val writeResult = when (currentMode) {
            EditMode.OFFLINE -> writeToHistory(editable, targetTable)
            EditMode.REALTIME, EditMode.SAVE -> writeToSave(editable, targetTable)
        }
        report.merge(writeResult)

        // 6. 持久化编辑日志
        historyManager.persistLog(sessionId)

        report
    }

    /**
     * 批量保存
     */
    suspend fun saveBatch(editables: List<EditableEntity>, targetTable: String): EditReport = withContext(ioDispatcher) {
        val report = EditReport()
        historyDb.runInTransaction {
            for (editable in editables) {
                val sub = save(editable, targetTable)
                report.merge(sub)
                if (!sub.success && config.abortOnError) return@runInTransaction
            }
        }
        report
    }

    /**
     * 撤销上一步
     */
    suspend fun undo(sessionId: String): EditReport = withContext(ioDispatcher) {
        historyManager.undo(sessionId, historyDb, saveDb)
    }

    /**
     * 重做
     */
    suspend fun redo(sessionId: String): EditReport = withContext(ioDispatcher) {
        historyManager.redo(sessionId, historyDb, saveDb)
    }

    /**
     * 写入 history.db
     */
    private suspend fun writeToHistory(editable: EditableEntity, targetTable: String): EditReport {
        val writer = BatchWriter(historyDb, config.batchSize)
        return when (targetTable) {
            "player" -> writePlayerToHistory(editable as EditablePlayer, writer)
            "club" -> writeClubToHistory(editable as EditableClub, writer)
            // ... 其余表
            else -> EditReport().fail("不支持的表：$targetTable")
        }
    }

    private suspend fun writePlayerToHistory(player: EditablePlayer, writer: BatchWriter): EditReport {
        val report = EditReport()
        if (player.isDeleted) {
            historyDb.playerDao().delete(player.original!!)
            report.deletedCount = 1
        } else {
            historyDb.playerDao().upsert(player.draft)
            if (player.isNew) report.insertedCount = 1 else report.updatedCount = 1

            // 写入属性
            for ((_, attr) in player.attributesDraft) {
                historyDb.playerAttributesDao().upsert(attr)
            }
        }
        return report
    }

    /**
     * 写入 save_xxx.db
     */
    private suspend fun writeToSave(editable: EditableEntity, targetTable: String): EditReport {
        saveDb ?: return EditReport().fail("save.db 未加载，无法实时/存档编辑")
        // 路由到具体 save 表写入
        return when (targetTable) {
            "save_player_state" -> writePlayerStateToSave(editable, saveDb)
            "save_club_state" -> writeClubStateToSave(editable, saveDb)
            // ... 其余 save 表
            else -> EditReport().fail("不支持的 save 表：$targetTable")
        }
    }

    private fun getCurrentSaveId(db: SaveDatabase): String? {
        return db.saveManifestDao().get()?.saveId
    }
}
```

```kotlin
// validator/EditorValidator.kt
class EditorValidator(
    private val historyDb: HistoryDatabase,
    private val config: ValidationConfig
) {

    private val t01Validator = DataValidator(config)  // 直接复用 T01 校验器

    /**
     * 校验可编辑 Entity
     * @return ValidationResult
     */
    suspend fun validate(editable: EditableEntity, targetTable: String): ValidationResult {
        val errors = mutableListOf<ValidationError>()
        val warnings = mutableListOf<ValidationWarning>()

        // 1. 必填字段检查
        val requiredResult = checkRequiredFields(editable, targetTable)
        errors.addAll(requiredResult.errors)

        // 2. 范围检查（CA 0-200、年龄 14-50、属性 1-100 等）
        val rangeResult = checkRanges(editable, targetTable)
        errors.addAll(rangeResult.errors)
        warnings.addAll(rangeResult.warnings)

        // 3. 引用完整性（球员所属俱乐部必须存在）
        val fkResult = checkForeignKeys(editable, targetTable)
        errors.addAll(fkResult.errors)

        // 4. 唯一性检查（球员 ID 不重复）
        val uniqueResult = checkUniqueness(editable, targetTable)
        errors.addAll(uniqueResult.errors)

        return ValidationResult(errors = errors, warnings = warnings)
    }

    private suspend fun checkRequiredFields(editable: EditableEntity, targetTable: String): ValidationResult {
        val errors = mutableListOf<ValidationError>()
        when (targetTable) {
            "player" -> {
                val p = (editable as EditablePlayer).draft
                if (p.realName.isBlank()) errors.add(ValidationError("real_name", "球员姓名不能为空", "REQUIRED_MISSING"))
                if (p.nationality.isBlank()) errors.add(ValidationError("nationality", "国籍不能为空", "REQUIRED_MISSING"))
                if (p.primaryPosition.isBlank()) errors.add(ValidationError("primary_position", "主要位置不能为空", "REQUIRED_MISSING"))
                if (p.primaryPosition !in listOf("GK","RB","LB","CB","DM","CM","AM","RW","LW","ST"))
                    errors.add(ValidationError("primary_position", "位置枚举非法：${p.primaryPosition}", "ENUM_INVALID"))
            }
            "club" -> {
                val c = (editable as EditableClub).draft
                if (c.clubName.isBlank()) errors.add(ValidationError("club_name", "俱乐部名称不能为空", "REQUIRED_MISSING"))
            }
            // ... 其余表
        }
        return ValidationResult(errors = errors)
    }

    private suspend fun checkRanges(editable: EditableEntity, targetTable: String): ValidationResult {
        val errors = mutableListOf<ValidationError>()
        val warnings = mutableListOf<ValidationWarning>()
        when (targetTable) {
            "player" -> {
                val p = (editable as EditablePlayer).draft
                p.height?.let { if (it !in 150..220) errors.add(ValidationError("height", "身高范围 150-220：$it", "RANGE_OVER")) }
                p.weight?.let { if (it !in 50..110) errors.add(ValidationError("weight", "体重范围 50-110：$it", "RANGE_OVER")) }
                if (p.retireAgeBase !in 30..42) errors.add(ValidationError("retire_age_base", "退役年龄范围 30-42：${p.retireAgeBase}", "RANGE_OVER"))
                // 年龄检查（14-50）
                p.birthDate?.let {
                    val age = calcAge(it)
                    if (age !in 14..50) errors.add(ValidationError("birth_date", "年龄范围 14-50：$age", "RANGE_OVER"))
                }
                // 属性范围检查
                for ((_, attr) in p.attributesDraft) {
                    if (attr.ca !in 1..200) errors.add(ValidationError("ca", "CA 范围 1-200：${attr.ca}", "RANGE_OVER"))
                    if (attr.pa !in 1..200) errors.add(ValidationError("pa", "PA 范围 1-200：${attr.pa}", "RANGE_OVER"))
                    if (attr.pa < attr.ca) warnings.add(ValidationWarning("pa", "PA < CA：${attr.pa} < ${attr.ca}"))
                }
            }
            "club" -> {
                val c = (editable as EditableClub).draft
                if (c.reputation !in 1..100) errors.add(ValidationError("reputation", "声望范围 1-100：${c.reputation}", "RANGE_OVER"))
                if (c.trainingLevel !in 1..100) errors.add(ValidationError("training_level", "训练等级范围 1-100", "RANGE_OVER"))
            }
        }
        return ValidationResult(errors = errors, warnings = warnings)
    }

    private suspend fun checkForeignKeys(editable: EditableEntity, targetTable: String): ValidationResult {
        val errors = mutableListOf<ValidationError>()
        when (targetTable) {
            "squad_membership" -> {
                val clubIds = historyDb.clubDao().getAllIds().toSet()
                val playerIds = historyDb.playerDao().getAllIds().toSet()
                // 校验引用的 club_id 和 player_id 存在
            }
            "historical_prospect_pool" -> {
                val playerIds = historyDb.playerDao().getAllIds().toSet()
                val p = (editable as EditableProspect).draft
                if (p.playerId !in playerIds)
                    errors.add(ValidationError("player_id", "历史新星引用的球员不存在：${p.playerId}", "FK_NOT_FOUND"))
            }
        }
        return ValidationResult(errors = errors)
    }

    private suspend fun checkUniqueness(editable: EditableEntity, targetTable: String): ValidationResult {
        val errors = mutableListOf<ValidationError>()
        when (targetTable) {
            "player" -> {
                val p = (editable as EditablePlayer).draft
                if (editable.isNew || (editable.original != null && editable.original!!.playerId != p.playerId)) {
                    if (historyDb.playerDao().existsById(p.playerId))
                        errors.add(ValidationError("player_id", "球员 ID 重复：${p.playerId}", "UNIQUE_DUPLICATE"))
                }
            }
        }
        return ValidationResult(errors = errors)
    }

    private fun calcAge(birthDate: String): Int {
        val birth = LocalDate.parse(birthDate)
        return Period.between(birth, LocalDate.now()).years
    }
}
```

```kotlin
class BalanceChecker(private val config: BalanceConfig) {

    /**
     * 平衡性检查（仅警告，不阻塞保存）
     * @return 警告列表
     */
    fun check(editable: EditableEntity, targetTable: String): List<BalanceWarning> {
        val warnings = mutableListOf<BalanceWarning>()
        when (targetTable) {
            "player" -> {
                val p = (editable as EditablePlayer).draft
                for ((_, attr) in p.attributesDraft) {
                    // CA 过高警告
                    if (attr.ca > config.caHighThreshold) {
                        warnings.add(BalanceWarning(
                            severity = BalanceSeverity.HIGH,
                            field = "ca",
                            message = "CA=${attr.ca} 超过高水平阈值 ${config.caHighThreshold}，可能影响游戏平衡",
                            suggestion = "建议 CA ≤ ${config.caHighThreshold}"
                        ))
                    }
                    // PA 过高警告
                    if (attr.pa > config.paHighThreshold) {
                        warnings.add(BalanceWarning(
                            severity = BalanceSeverity.HIGH,
                            field = "pa",
                            message = "PA=${attr.pa} 超过高潜力阈值 ${config.paHighThreshold}，球员成长可能过强",
                            suggestion = "建议 PA ≤ ${config.paHighThreshold}"
                        ))
                    }
                    // CA 与 PA 差距过大
                    if (attr.pa - attr.ca > config.maxCaPaGap) {
                        warnings.add(BalanceWarning(
                            severity = BalanceSeverity.MEDIUM,
                            field = "pa",
                            message = "PA-CA=${attr.pa - attr.ca} 差距过大，最大允许 ${config.maxCaPaGap}",
                            suggestion = "建议 PA-CA ≤ ${config.maxCaPaGap}"
                        ))
                    }
                    // 单属性过高
                    checkSingleAttributeTooHigh(attr, warnings)
                }
            }
            "club" -> {
                val c = (editable as EditableClub).draft
                if (c.reputation > config.reputationHighThreshold) {
                    warnings.add(BalanceWarning(
                        severity = BalanceSeverity.MEDIUM,
                        field = "reputation",
                        message = "声望=${c.reputation} 较高，俱乐部可能过强",
                        suggestion = "建议声望 ≤ ${config.reputationHighThreshold}"
                    ))
                }
            }
        }
        return warnings
    }

    private fun checkSingleAttributeTooHigh(attr: PlayerAttributesEntity, warnings: MutableList<BalanceWarning>) {

                    severity = BalanceSeverity.LOW,
                    field = name,
                    message = "单属性 $name=$value 较高",
                    suggestion = "建议单属性 ≤ ${config.singleAttrHighThreshold}"
                ))
            }
        }
    }
```

```kotlin
// exporter/DataExporter.kt
class DataExporter(
    private val historyDb: HistoryDatabase,
    private val saveDb: SaveDatabase?,
    private val config: EditorConfig,
    private val ioDispatcher: CoroutineDispatcher = Dispatchers.IO
) {

    /**
     * 导出全量数据包
     * @param targetDir 目标目录
     * @param format 导出格式（CSV / JSON）
     * @param includeSave 是否包含存档数据
     * @param callback 进度回调
     * @return ExportReport
     */
    suspend fun exportPack(
        targetDir: File,
        format: ExportFormat = ExportFormat.CSV,
        includeSave: Boolean = false,
        callback: ExportProgressCallback? = null
    ): ExportReport = withContext(ioDispatcher) {
        val report = ExportReport()
        targetDir.mkdirs()

        // 1. 导出 history.db 全表
        val historyTables = config.exportHistoryTables
        var index = 0
        for (table in historyTables) {
            index++
            callback?.onProgress(table, index, historyTables.size)
            try {
                when (format) {
                    ExportFormat.CSV -> CsvExporter(historyDb).exportTable(table, File(targetDir, "$table.csv"))
                    ExportFormat.JSON -> JsonExporter(historyDb).exportTable(table, File(targetDir, "$table.json"))
                }
                report.successCount++
            } catch (e: Exception) {
                report.errors.add(ExportError(table = table, message = e.message ?: "未知错误"))
                report.failedCount++
            }
        }

        // 2. 可选：导出 save.db
        if (includeSave && saveDb != null) {
            val saveTables = config.exportSaveTables
            for (table in saveTables) {
                index++
                callback?.onProgress("save_$table", index, historyTables.size + saveTables.size)
                try {
                    when (format) {
                        ExportFormat.CSV -> CsvExporter(historyDb).exportSaveTable(saveDb, table, File(targetDir, "save_$table.csv"))
                        ExportFormat.JSON -> JsonExporter(historyDb).exportSaveTable(saveDb, table, File(targetDir, "save_$table.json"))
                    }
                    report.successCount++
                } catch (e: Exception) {
                    report.errors.add(ExportError(table = "save_$table", message = e.message ?: "未知错误"))
                    report.failedCount++
                }
            }
        }

        // 3. 生成 manifest.json
        val manifest = ManifestGenerator(historyDb, config).generate(includeSave)
        val manifestFile = File(targetDir, "manifest.json")
        manifestFile.writeText(Json.encodeToString(DataPackManifest.serializer(), manifest))

        // 4. 计算 checksum
        val checksum = ChecksumUtil.calculateDirectoryChecksum(targetDir)
        val updatedManifest = manifest.copy(checksum = "sha256:$checksum")
        manifestFile.writeText(Json.encodeToString(DataPackManifest.serializer(), updatedManifest))

        report.durationMs = System.currentTimeMillis() - report.startTime
        callback?.onComplete(report)
        report
    }
}
```

```kotlin
                "transfers" -> exportTransfers(writer)

            while (cursor.moveToNext()) {

                    appendEscaped(cursor.getString(2)).append(',')     // real_name
                    appendNullable(cursor.getString(3)).append(',')    // display_name
                    appendNullable(cursor.getString(4)).append(',')    // birth_date
                    appendEscaped(cursor.getString(5)).append(',')     // nationality
                    appendNullable(cursor.getString(6)).append(',')    // second_nationality
                    appendNullable(cursor.getInt(7)).append(',')       // height
                    appendNullable(cursor.getInt(8)).append(',')       // weight
                    appendNullable(cursor.getString(9)).append(',')    // preferred_foot
                    append(cursor.getString(10)).append(',')           // primary_position
                    appendNullable(cursor.getString(11)).append(',')   // secondary_positions
```

```kotlin
// history/EditHistoryManager.kt
class EditHistoryManager(
    private val editorDb: EditorDatabase,
    private val config: EditorConfig
) {

    private val undoStacks = mutableMapOf<String, UndoRedoStack>()  // sessionId → stack
    private val redoStacks = mutableMapOf<String, UndoRedoStack>()

    /**
     * 记录编辑命令
     */
    suspend fun recordCommand(sessionId: String, command: EditCommand) {
        val stack = undoStacks.getOrPut(sessionId) { UndoRedoStack(config.maxUndoSteps) }
        stack.push(command)
        // 清空 redo 栈（新操作后旧的 redo 失效）
        redoStacks.remove(sessionId)
    }

    /**
     * 撤销
     */
    suspend fun undo(sessionId: String, historyDb: HistoryDatabase, saveDb: SaveDatabase?): EditReport {
        val stack = undoStacks[sessionId] ?: return EditReport().fail("无可撤销操作")
        val command = stack.pop() ?: return EditReport().fail("无可撤销操作")

        val report = EditReport()
        try {
            command.undo(historyDb, saveDb)
            redoStacks.getOrPut(sessionId) { UndoRedoStack(config.maxUndoSteps) }.push(command)
            // 更新日志状态
            editorDb.editorDao().markUndone(command.logId)
            report.updatedCount = 1
        } catch (e: Exception) {
            report.fail("撤销失败：${e.message}")
            // 撤销失败，把命令推回栈
            stack.push(command)
        }
        return report
    }

    /**
     * 重做
     */
    suspend fun redo(sessionId: String, historyDb: HistoryDatabase, saveDb: SaveDatabase?): EditReport {
        val stack = redoStacks[sessionId] ?: return EditReport().fail("无可重做操作")
        val command = stack.pop() ?: return EditReport().fail("无可重做操作")

        val report = EditReport()
        try {
            command.redo(historyDb, saveDb)
            undoStacks.getOrPut(sessionId) { UndoRedoStack(config.maxUndoSteps) }.push(command)
            editorDb.editorDao().markRedone(command.logId)
            report.updatedCount = 1
        } catch (e: Exception) {
            report.fail("重做失败：${e.message}")
            stack.push(command)
        }
        return report
    }

    /**
     * 持久化编辑日志（避免崩溃丢失）
     */
    suspend fun persistLog(sessionId: String) {
        val stack = undoStacks[sessionId] ?: return
        for (command in stack.all()) {
            if (!command.persisted) {
                editorDb.editorDao().insertLog(command.toLogEntity(sessionId))
                command.persisted = true
            }
        }
    }

    /**
     * 查询会话编辑历史
     */
    suspend fun getHistory(sessionId: String): List<EditLogEntity> {
        return editorDb.editorDao().getBySession(sessionId)
    }

    /**
     * 获取可撤销步数
     */
    fun getUndoableCount(sessionId: String): Int = undoStacks[sessionId]?.size() ?: 0

    /**
     * 获取可重做步数
     */
    fun getRedoableCount(sessionId: String): Int = redoStacks[sessionId]?.size() ?: 0
}

// history/UndoRedoStack.kt
class UndoRedoStack(private val maxSteps: Int) {
    private val stack = ArrayDeque<EditCommand>()

    fun push(command: EditCommand) {
        stack.addLast(command)
        while (stack.size > maxSteps) stack.removeFirst()
    }

    fun pop(): EditCommand? = if (stack.isEmpty()) null else stack.removeLast()

    fun size(): Int = stack.size

    fun all(): List<EditCommand> = stack.toList()
}
```

### 1. 编辑流程（读取 → 修改 → 校验 → 保存）

```
FUNCTION editFlow(entityType, entityId, modifications):
    // 1. 开启会话（若未开启）
    IF sessionId IS NULL:
        sessionId = editorManager.startSession(currentMode)

    // 2. 读取原始数据，构建草稿态
    editable = editor.loadEditable(entityId)
    // editable.original = 原始 Entity
    // editable.draft = 原始 Entity 的副本（待修改）

    // 3. 应用用户修改到草稿
    FOR (field, value) IN modifications:
        editable = editor.updateField(editable, field, value)
        // 修改记录到内存 diff 列表

    // 4. 用户点击「保存」
    // 4.1 权限校验
    IF NOT permissionManager.canEdit(entityType):
        RETURN Report.fail("权限不足")

    // 4.2 数据校验（复用 T01 DataValidator）
    validation = EditorValidator.validate(editable, entityType)
    IF NOT validation.isValid:
        RETURN Report.fail("校验失败", validation.errors)

    // 4.3 平衡性检查（仅警告，不阻塞）
    warnings = BalanceChecker.check(editable, entityType)
    IF warnings.hasHigh AND config.confirmOnHighWarning:
        IF NOT userConfirm(warnings):
            RETURN Report.cancelled("用户取消")

    // 4.4 编辑前备份（仅 history.db 离线模式）
    IF currentMode == OFFLINE AND NOT sessionBackedUp:
        backupPath = BackupManager.backupHistoryDb()
        sessionBackedUp = true

    // 4.5 写入数据库（事务）
    db.beginTransaction()
    TRY:
        command = EditCommandFactory.create(editable, entityType, currentMode)
        command.execute(historyDb, saveDb)
        historyManager.recordCommand(sessionId, command)
        historyManager.persistLog(sessionId)
        db.setTransactionSuccessful()
    CATCH e:
        db.rollback()
        RETURN Report.fail("写入失败：" + e.message)
    FINALLY:
        db.endTransaction()

    // 4.6 失效 cache.db
    CacheRebuilder.markDirty(entityType)

    RETURN Report.success(warnings)
```

### 2. 撤销/重做算法

```
FUNCTION undo(sessionId):
    stack = undoStacks[sessionId]
    IF stack IS NULL OR stack.isEmpty:
        RETURN Report.fail("无可撤销操作")

    command = stack.pop()

    // 1. 临时打开写入权限（若 history.db 模式）
    IF command.targetDb == "history":
        HistoryDbAccessor.openForWrite()

    // 2. 执行逆操作
    db.beginTransaction()
    TRY:
        IF command.operationType == "INSERT":
            // 新增的撤销 = 删除
            db.delete(command.targetTable, command.targetId)
        ELSE IF command.operationType == "DELETE":
            // 删除的撤销 = 恢复
            db.insert(command.originalEntity)
        ELSE: // UPDATE
            // 修改的撤销 = 写回旧值
            db.update(command.targetTable, command.originalEntity)

        // 3. 推入 redo 栈
        redoStacks[sessionId].push(command)

        // 4. 更新日志状态
        editorDb.markUndone(command.logId)

        db.setTransactionSuccessful()
    CATCH e:
        // 撤销失败，把命令推回栈
        stack.push(command)
        RETURN Report.fail("撤销失败：" + e.message)
    FINALLY:
        db.endTransaction()
        IF command.targetDb == "history":
            HistoryDbAccessor.closeAndRestoreReadOnly()

    RETURN Report.success()


FUNCTION redo(sessionId):
    stack = redoStacks[sessionId]
    IF stack IS NULL OR stack.isEmpty:
        RETURN Report.fail("无可重做操作")

    command = stack.pop()

    db.beginTransaction()
    TRY:
        // 重做 = 重新执行原命令
        command.execute(historyDb, saveDb)
        undoStacks[sessionId].push(command)
        editorDb.markRedone(command.logId)
        db.setTransactionSuccessful()
    CATCH e:
        stack.push(command)
        RETURN Report.fail("重做失败：" + e.message)
    FINALLY:
        db.endTransaction()

    RETURN Report.success()
```

```
FUNCTION exportToCsv(targetDir):
    // 1. 创建目标目录
    targetDir.mkdirs()

    // 2. 逐表导出（流式，避免内存爆炸）
    FOR table IN config.exportHistoryTables:
        outputFile = File(targetDir, table + ".csv")
        writer = outputFile.bufferedWriter()

        // 2.1 写表头（与 T01 CSV 表头一致）
        writer.write(getCsvHeader(table))
        writer.newLine()

        // 2.2 流式读取 + 写入
        cursor = historyDb.query("SELECT * FROM " + table)
        WHILE cursor.moveToNext():
            row = buildCsvRow(cursor, table)
            writer.write(row)
            writer.newLine()
        cursor.close()
        writer.close()

    // 3. 生成 manifest.json
    manifest = ManifestGenerator.generate()
    manifest.packId = config.exportPackIdPrefix + timestamp
    manifest.packType = "local_edited"
    manifest.schemaVersion = 2
    manifest.dataVersion = config.exportDataVersion
    manifest.containsRealNames = dataPackManifest.containsRealNames

    // 4. 计算 checksum
    manifest.checksum = "sha256:" + calculateDirectoryChecksum(targetDir)

    // 5. 写 manifest.json
    File(targetDir, "manifest.json").writeText(jsonEncode(manifest))

    RETURN ExportReport(successCount, failedCount)


FUNCTION buildCsvRow(cursor, table):
    // 按 T01 字段顺序拼接，处理引号转义
    sb = StringBuilder()
    FOR (i, field) IN getTableFields(table).withIndex():
        IF i > 0: sb.append(",")
        value = cursor.getValue(field)
        IF value IS NULL:
            CONTINUE  // 空字符串
        IF value.contains(",") OR value.contains("\"") OR value.contains("\n"):
            sb.append("\"").append(value.replace("\"", "\"\"")).append("\"")
        ELSE:
            sb.append(value)
    RETURN sb.toString()
```

```
FUNCTION balanceCheck(editable, targetTable):
    warnings = []

    IF targetTable == "player":
        player = editable.draft
        FOR (seasonId, attr) IN player.attributesDraft:
            // 1. CA 过高警告（V0.2 §五 CA 价值曲线）
            IF attr.ca > config.caHighThreshold:  // 默认 180
                warnings.add(BalanceWarning(HIGH, "ca",
                    "CA=" + attr.ca + " 超过高水平阈值 " + config.caHighThreshold,
                    "建议 CA ≤ " + config.caHighThreshold))

            // 2. PA 过高警告
            IF attr.pa > config.paHighThreshold:  // 默认 190
                warnings.add(BalanceWarning(HIGH, "pa",
                    "PA=" + attr.pa + " 超过高潜力阈值 " + config.paHighThreshold,
                    "建议 PA ≤ " + config.paHighThreshold))

            // 3. PA < CA 警告（违反 V0.2 §五 成长模型）
            IF attr.pa < attr.ca:
                warnings.add(BalanceWarning(MEDIUM, "pa",
                    "PA=" + attr.pa + " < CA=" + attr.ca,
                    "PA 应 ≥ CA"))

            // 4. CA-PA 差距过大（V0.2 §五 潜力兑现率）
            IF attr.pa - attr.ca > config.maxCaPaGap:  // 默认 60
                warnings.add(BalanceWarning(MEDIUM, "pa",
                    "PA-CA=" + (attr.pa - attr.ca) + " 差距过大",
                    "建议 PA-CA ≤ " + config.maxCaPaGap))

            // 5. 单属性过高（V0.2 §五 球星模板量化）
            FOR field IN ["shooting","finishing","passing","dribbling","pace","strength","defending","vision"]:
                value = attr.getField(field)
                IF value > config.singleAttrHighThreshold:  // 默认 90
                    warnings.add(BalanceWarning(LOW, field,
                        field + "=" + value + " 较高",
                        "建议单属性 ≤ " + config.singleAttrHighThreshold))

            // 6. 门将属性非门将位置检查
            IF player.primaryPosition != "GK":
                gkSum = attr.gkDiving + attr.gkReflexes + attr.gkHandling + attr.gkPositioning + attr.gkOneOnOne
                IF gkSum > 50:
                    warnings.add(BalanceWarning(LOW, "gk_attributes",
                        "非门将位置有门将属性",
                        "非门将球员的 gk_* 属性应为 0"))

    ELSE IF targetTable == "club":
        club = editable.draft
        // 7. 声望过高
        IF club.reputation > config.reputationHighThreshold:  // 默认 95
            warnings.add(BalanceWarning(MEDIUM, "reputation",
                "声望=" + club.reputation + " 过高",
                "建议声望 ≤ " + config.reputationHighThreshold))

        // 8. 设施等级全部满级
        IF club.trainingLevel >= 100 AND club.youthLevel >= 100 AND club.financeLevel >= 100:
            warnings.add(BalanceWarning(MEDIUM, "facilities",
                "全部设施满级",
                "建议保留差异化，不要全部 100"))

    RETURN warnings
```

```kotlin
@Composable
fun NumberInputField(
    label: String,
    value: Int?,
    onValueChange: (Int?) -> Unit,
    min: Int? = null,
    max: Int? = null,
    suffix: String? = null,
    readOnly: Boolean = false,
    modifier: Modifier = Modifier
) {
    var text by remember(value) { mutableStateOf(value?.toString() ?: "") }
    var error by remember { mutableStateOf<String?>(null) }

    OutlinedTextField(
        value = text,
        onValueChange = { input ->
            text = input
            val parsed = input.toIntOrNull()
            when {
                input.isBlank() -> {
                    onValueChange(null)
                    error = null
                }
                parsed == null -> {
                    error = "请输入数字"
                }
                min != null && parsed < min -> {
                    error = "最小值 $min"
                }
                max != null && parsed > max -> {
                    error = "最大值 $max"
                }
                else -> {
                    onValueChange(parsed)
                    error = null
                }
            }
        },
        label = { Text(label) },
        isError = error != null,
        supportingText = { error?.let { Text(it, color = MaterialTheme.colors.error) } },
        trailingIcon = { suffix?.let { Text(it) } },
        readOnly = readOnly,
        keyboardOptions = KeyboardOptions(keyboardType = KeyboardType.Number),
        modifier = modifier.fillMaxWidth()
    )
}
```

## 九、性能要求

| 操作 | 目标 | 实现策略 |
|------|------|----------|
| 编辑器主页加载 | ≤500 ms | 预加载模块计数 + LazyColumn 懒加载 |
| 球员编辑页加载 | ≤500 ms | 单球员 + 单赛季属性查询，索引覆盖 |
| 单字段保存 | ≤200 ms | 单行 upsert + 内存草稿更新 |
| 批量保存（100 球员） | ≤2 秒 | 事务批量 upsert + 分批 100/批 |
| 撤销/重做 | ≤50 ms | 内存栈操作 + 单行 upsert |
| 全量导出（10000 球员 + 500 俱乐部） | ≤5 秒 | 流式 Cursor + BufferedWriter + 批量 flush |
| 单表导出 | ≤1 秒 | 流式读取 + 不全量加载到内存 |
| 编辑历史查询 | ≤100 ms | 索引查询 session_id + 分页 50 条 |
| 权限校验 | ≤5 ms | 内存 Map 查询 |
| 平衡性检查 | ≤20 ms | 内存计算，无 DB 查询 |
| 备份 history.db（5-15MB） | ≤2 秒 | 文件复制，异步不阻塞 UI |
| 备份 save.db（≤80MB） | ≤5 秒 | 文件复制，显示进度 |

### 性能优化要点

1. **草稿态内存缓存**：EditableEntity 在 ViewModel 中持有，UI 修改直接更新草稿，不触发 DB 查询。
2. **流式导出**：CsvExporter 使用 Cursor 流式读取 + BufferedWriter，避免 10000+ Entity 全量加载到 List。
3. **撤销/重做内存栈**：UndoRedoStack 内存维护，撤销操作仅 1 次 DB upsert，无复杂查询。
4. **编辑日志异步持久化**：EditHistoryManager.persistLog 异步写入 editor.db，不阻塞保存流程。
5. **cache.db 失效标记**：编辑后仅标记 dirty，不立即重建，下次主游戏访问时按需重建。
6. **批量保存事务**：批量编辑用单大事务包裹，减少 commit 开销。
7. **权限校验内存化**：PermissionManager 启动时加载 tablePermissions 到内存 Map，O(1) 查询。

## 十一、验收标准

依据 T25 任务范围与 V0.1 `03_页面结构与交互说明.md` §10 + V0.2 `02_数据来源治理与版本管理.md` §十：

1. **球员编辑**：可修改姓名/国籍/位置/属性/CA/PA/合同/身价，校验生效（必填+范围+枚举+唯一）。
2. **俱乐部编辑**：可修改基础信息/财政/设施/青训配置，可添加/移除球员到阵容。
3. **联赛编辑**：可修改联赛结构/规则/商业系数，可添加/移除参赛俱乐部。
4. **转会编辑**：可修改转会记录，可标记/取消蝴蝶关键节点，可编辑替代决策池（高级权限）。
5. **剧本编辑**：可修改开局年份/可选联赛/开关/经济参数（高级/开发者权限）。
6. **历史新星编辑**：可修改可发现时间/默认路径/标签/legend_level（高级权限）。
7. **数据校验**：复用 T01 校验逻辑，必填/范围/外键/唯一/枚举 5 类规则全部生效。
8. **平衡性检查**：CA 过高/PA 过高/PA<CA/单属性过高 4 类警告生效，HIGH 级别二次确认。
9. **撤销/重做**：可撤销最多 50 步，撤销后可重做，崩溃后历史可查询（持久化）。
10. **数据导出**：支持 CSV/JSON 格式，字段严格对齐 T01，导出后可被 T01 重新导入。
11. **数据导入**：复用 T01 DataImportManager，导入后清空编辑历史。
12. **manifest 生成**：导出时生成 manifest.json，含 pack_id/schema_version/checksum。
13. **权限分级**：普通/高级/开发者三档，高级需确认码，开发者彩蛋式解锁。
14. **编辑模式**：离线/实时/存档三模式，离线编辑 history.db，实时编辑当前存档，存档编辑指定 save.db。
15. **自动备份**：离线/存档模式开启会话前自动备份，保留 5 个，可从备份恢复。
16. **实时编辑安全**：比赛日阻止编辑，建议 checkpoint，自动 checkpoint 间隔可配。
17. **编辑历史持久化**：editor_edit_log 表记录操作，崩溃后可查询历史。
18. **cache.db 失效**：编辑后标记 cache 脏，主游戏访问时按需重建。
19. **性能达标**：编辑器加载 ≤500ms，单字段保存 ≤200ms，全量导出 ≤5 秒，撤销/重做 ≤50ms。
20. **配置化**：所有参数走 editor_config.json，修改不重编译生效。
21. **UI 完整**：EditorHomeScreen + 7 大模块编辑页 + 编辑历史页 + 导出页 + 导入页，状态覆盖 loading/empty/error/normal/locked/warning。
22. **不影响主流程**：编辑器是二级页面，主游戏流程不依赖编辑器，编辑器异常不崩溃主游戏。

## 十四、风险与应对

### 1. 实时编辑破坏存档

- **表现**：玩家在游戏运行中修改 save_player_state，导致存档损坏（如引用不存在的球员）
- **应对**：
  - 实时编辑前建议创建 checkpoint（一键按钮）
  - 严格校验（复用 T01 DataValidator），不合法数据拒绝写入
  - 比赛模拟进行中阻止编辑（`block_edit_during_simulation`）
  - 自动 checkpoint 间隔可配（默认 10 分钟）
- **降级**：若存档损坏，可从 checkpoint 恢复；最坏情况从自动备份恢复

### 2. 撤销/重做数据丢失

- **表现**：崩溃后内存栈丢失，无法撤销
- **应对**：
  - EditLogEntity 持久化到 editor.db，崩溃后可查询历史
  - 每次保存后立即 persistLog（同步写入）
  - UndoRedoStack 限制 50 步，避免内存爆炸
- **降级**：崩溃后内存栈丢失，但可从备份恢复到崩溃前状态

### 3. history.db 只读约束失效

- **表现**：编辑时写入被 PRAGMA query_only 拒绝
- **应对**：
  - HistoryDbAccessor 复用 T01 HistoryDbWriter，显式 `PRAGMA query_only = OFF`
  - 编辑完成后 closeAndRestoreReadOnly 恢复 `PRAGMA query_only = ON`
  - 单测强制校验：编辑前 query_only=OFF，结束后 query_only=ON
- **降级**：若 query_only 无法关闭，提示用户「无法编辑只读库」

### 4. 导出格式不兼容 T01

- **表现**：T25 导出的 CSV 被 T01 导入时报错（字段错位/缺失）
- **应对**：
  - CsvExporter 字段名严格对齐 T01 CSV 表头（`players.csv` / `player_attributes.csv` 等）
  - 单测：导出 → 导入循环验证，0 错误
  - manifest.json 结构与 T01 完全一致
- **降级**：提供字段映射配置，允许手动调整导出字段名

### 5. 编辑导致跨表引用完整性破坏

- **表现**：删除球员后 squad_membership 引用孤儿
- **应对**：
  - EditorValidator.checkForeignKeys 检查引用关系
  - 删除前警告「该球员有阵容关系，删除会导致孤儿记录」
  - 跨表校验（复用 T01 CrossTableValidator）
- **降级**：提供「级联删除」选项（V2），V1 仅警告

### 6. 大数据量导出性能不达标

- **表现**：10000 球员全量导出 >5 秒
- **应对**：
  - CsvExporter 流式 Cursor + BufferedWriter，避免全量加载到内存
  - 批量 flush（每 1000 行）
  - 关闭 journal_mode / synchronous（导出期间）
- **降级**：分表导出，每表独立文件

### 7. 权限确认码泄露

- **表现**：高级/开发者确认码被公开传播
- **应对**：
  - 确认码配置在 editor_config.json，玩家可自定义
  - 开发者模式彩蛋式解锁（连续点击版本号 7 次）
  - 高级/开发者模式仅影响本地编辑，不影响联机（V1 无联机）
- **降级**：无实质风险，确认码仅是体验设计

### 8. 编辑器影响主游戏性能

- **表现**：编辑器开启时主游戏卡顿
- **应对**：
  - 编辑器是二级页面，不与主游戏并发运行
  - 编辑操作在 IO 线程，不阻塞主线程
  - cache.db 失效标记异步处理
- **降级**：编辑器运行时禁用主游戏自动推进

### 9. 编辑历史体积膨胀

- **表现**：editor_edit_log 表无限增长
- **应对**：
  - UndoRedoStack 限制 50 步（配置项 max_undo_steps）
  - editor_edit_log 定期清理（保留最近 1000 条）
  - 编辑会话结束时归档旧日志
- **降级**：editor.db 体积报警时提示用户清理

### 10. 实时编辑与 AI 模拟冲突

- **表现**：玩家编辑时 AI 俱乐部正在转会，导致数据竞争
- **应对**：
  - 实时编辑期间暂停 AI 模拟（`block_edit_during_simulation`）
  - 编辑保存时加锁（单写者）
  - 编辑结束后恢复 AI 模拟
- **降级**：编辑期间 AI 模拟结果回滚
