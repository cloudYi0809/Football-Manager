# T09 成长月结 - 详细实现方案

> 任务编号：T09
> 周期：W35-W37（3 周）
> 优先级：P0（B 轨成长系统运行时执行框架，Gate 2 验收硬指标"成长曲线合理"的承担者）
> 依据：V0.1 `07_球员成长_训练_伤病算法.md` §二/§九 + V0.2 `08_球员成长_训练_伤病细化算法.md` 全文 + V0.2 `09_长程性能_存档_版本迁移.md` §十/§十一
> 上游依赖：T0Y（成长算法核心公式）、T00（三库结构）、T02（出场时间统计）、T08（伤病状态）、T23（士气数据）、T16（青训球员与导师）
> 下游依赖：T04（成长曲线展示）、T19（赛季归档快照）、T07（每月 1 日调用本任务）
> 铁律：复用 T0Y 10 因子公式 + 7 档年龄表 + 8 因子兑现率 / 异常保护防"小妖全部满潜" / 活跃范围分流保证 P95 ≤3 秒 / 成长快照持久化
> 定位：T0Y 是"算法库"，T09 是"运行时执行框架"，决定月度成长是否被正确触发、聚合、持久化、可见

## 一、业务目标与范围

### 1. 业务目标

- **算法落地**：把 T0Y 的纯函数式成长算法封装为按月执行的运行时服务，由 T07 每月 1 日调用一次。
- **多源聚合**：把训练（T07 每日积累）、比赛（T02 出场时间）、伤病（T08）、士气（T23）、青训（T16）、俱乐部设施（T00 save_club_state）等多表数据聚合为 T0Y 计算所需的单一输入对象。
- **活跃范围分流**：FULL / ACTIVE / LIGHT / MINIMAL 四档处理，保证 100 球员俱乐部月结 ≤1s，全档月结总耗时 ≤2s。
- **成长可见**：每月写一条 `growth_snapshot` 快照，供 T04 阵容页成长曲线绘制与 T19 赛季归档。
- **事件驱动**：突破性成长 / 成长停滞 / 潜力兑现 / 早衰 / 训练态度恶化 / 导师正向影响 6 类事件，让玩家感知球员命运转折。
- **异常保护**：单月成长上限、连续停滞保护、已达潜力不再涨、衰退期强制下滑，避免"小妖全部满潜"与"老将永葆青春"。

### 2. V1 做清单

- 月结主流程：遍历 → 收集输入 → 调用 T0Y 计算 → 应用 CA/PA/属性变化 → 写快照 → 触发事件。
- 4 张持久化表：`growth_snapshot` / `monthly_training_record` / `monthly_playing_time` / `growth_event`。
- 4 档活跃范围处理：FULL（一线队+预备队）/ ACTIVE（U21+U18）/ LIGHT（外租）/ MINIMAL（未加载联赛）。
- T0Y 算法适配层：`GrowthCalculatorAdapter` 把多源聚合输入转换为 T0Y `MonthlyGrowthCalculator.calculate()` 所需参数。
- 潜力兑现率计算：复用 T0Y `PotentialRealizationCalculator` 8 因子，按月更新 `realization_score`。
- 6 类成长事件触发与持久化。
- 成长快照批量写入（事务化），支持 T04 按球员查询、T19 按赛季归档。
- 月结报告 Compose 页面（可选，展示本月成长汇总 Top N）。
- 性能日志：单球员耗时、全队耗时、总耗时写入 `perf_log`。

### 3. V1 不做清单（推迟）

- 训练专项细化 UI（V1 仅复用 T0Y `TrainingSpecializationHandler`，UI 由 T0M6-1 完成）。
- 导师关系图谱可视化（V1 仅持久化导师影响结果，UI 归 M6 沉浸系统）。
- 成长预测/模拟器（V1 仅展示历史曲线，不预测未来）。
- 跨存档成长数据继承（V1 单存档独立）。
- 球员个性化成长动画/演出。
- 国家人才池动态评分（V1 用静态配置值）。
- 训练 camp / 海外拉练加成（归 M6）。

### 4. 与 T0Y 的边界（关键）

| 维度 | T0Y（算法库） | T09（运行时框架） |
|------|----------------|---------------------|
| 定位 | 纯函数式计算 | 有状态月结服务 |
| 公式 | 10 因子月度成长 + 7 档年龄表 + 8 因子兑现率 | 不重写公式，仅调用 |
| 输入 | 接收已聚合的参数对象 | 负责从多表聚合输入 |
| 输出 | 返回 `GrowthResult`（无副作用） | 把 `GrowthResult` 落库 + 触发事件 |
| 触发 | 被动调用 | 由 T07 每月 1 日主动触发 |
| 持久化 | 无 | 4 张表读写 |
| 活跃范围 | 不感知 | FULL/ACTIVE/LIGHT/MINIMAL 分流 |

**铁律**：T09 不得重写 T0Y 的任何公式，10 因子权重、7 档年龄系数、8 因子兑现率权重全部从 `growth_config.json` 读取后透传给 T0Y。

## 三、数据模型（Room Entity + DAO）

### 1. 三库分离说明

| 表 | 所在库 | 读写权限 | 生命周期 | 说明 |
|----|--------|----------|----------|------|
| `growth_snapshot` | save.db | 读写 | 每月追加，赛季归档时聚合到 history.db | 月度快照，T04 成长曲线主数据源 |
| `monthly_training_record` | save.db | 读写 | 每月 1 条/球员，归档时删除明细 | T07 每日训练积累的月度汇总 |
| `monthly_playing_time` | save.db | 读写 | 每月 1 条/球员，归档时聚合到赛季统计 | T02 比赛出场分钟数月度汇总 |
| `growth_event` | save.db | 读写 | 永久保留（玩家可见） | 6 类成长事件流 |
| `player_attributes`（已有） | history.db | 只读 + 赛季末追加 | 每赛季 1 条 | T19 归档时把当赛季末快照写入此处 |

**关键约束**：
- 月度快照只落 save.db，不直接写 history.db（history.db 玩家不可写）。
- 赛季归档时由 T19 调用 `GrowthSnapshotQuery.getSeasonEndSnapshot()` 取出赛季末快照，写入 history.db `player_attributes` 表作为该赛季属性记录。
- T04 成长曲线查询：当前赛季内查 save.db `growth_snapshot`，历史赛季查 history.db `player_attributes`。

### 2. GrowthSnapshotEntity（成长快照）

```kotlin
// save/entity/GrowthSnapshotEntity.kt
@Entity(
    tableName = "growth_snapshot",
    indices = [
        Index(value = ["save_id", "player_id", "snapshot_date"], unique = true),
        Index(value = ["save_id", "club_id", "snapshot_date"]),
        Index(value = ["save_id", "player_id", "season_id"])
    ]
)
data class GrowthSnapshotEntity(
    @PrimaryKey(autoGenerate = true) val snapshotId: Int,
    @ColumnInfo(name = "save_id") val saveId: Int,
    @ColumnInfo(name = "player_id") val playerId: Int,
    @ColumnInfo(name = "club_id") val clubId: Int,
    @ColumnInfo(name = "season_id") val seasonId: Int,
    @ColumnInfo(name = "snapshot_date") val snapshotDate: String,        // ISO yyyy-MM-dd（月结日）
    @ColumnInfo(name = "age") val age: Int,
    @ColumnInfo(name = "ca_before") val caBefore: Int,
    @ColumnInfo(name = "ca_after") val caAfter: Int,
    @ColumnInfo(name = "ca_delta") val caDelta: Int,                     // 可为负
    @ColumnInfo(name = "pa_before") val paBefore: Int,
    @ColumnInfo(name = "pa_after") val paAfter: Int,
    @ColumnInfo(name = "pa_delta") val paDelta: Int,
    @ColumnInfo(name = "realization_score") val realizationScore: Double, // 0-1
    @ColumnInfo(name = "range_tier") val rangeTier: String,              // FULL/ACTIVE/LIGHT/MINIMAL
    @ColumnInfo(name = "growth_phase") val growthPhase: String,          // 7 档年龄阶段
    // 10 因子分解（用于调试与 T04 详细展示）
    @ColumnInfo(name = "factor_training_quality") val factorTrainingQuality: Double,
    @ColumnInfo(name = "factor_playing_time") val factorPlayingTime: Double,
    @ColumnInfo(name = "factor_mentor") val factorMentor: Double,
    @ColumnInfo(name = "factor_club_facility") val factorClubFacility: Double,
    @ColumnInfo(name = "factor_talent") val factorTalent: Double,
    @ColumnInfo(name = "factor_age") val factorAge: Double,
    @ColumnInfo(name = "factor_injury") val factorInjury: Double,
    @ColumnInfo(name = "factor_morale") val factorMorale: Double,
    @ColumnInfo(name = "factor_random") val factorRandom: Double,
    @ColumnInfo(name = "factor_national_pool") val factorNationalPool: Double,
    // 属性快照（JSON 字符串，避免 30+ 列宽表）
    @ColumnInfo(name = "attributes_json") val attributesJson: String,
    @ColumnInfo(name = "attribute_changes_json") val attributeChangesJson: String, // 本月变化
    @ColumnInfo(name = "notes") val notes: String?                        // 成长日志
)
```

**字段说明**：
- `ca_delta` / `pa_delta` 可为负（衰退期）。
- `attributes_json` 存当月末全部属性快照（如 `{"pace":85,"acceleration":87,...}`），用于 T04 成长曲线多维度展示。
- `attribute_changes_json` 存本月变化（如 `{"pace":+1,"stamina":+2}`），用于月结报告。
- 10 因子分解字段单独存储，便于 T04 详细页展示"本月成长来源"饼图。
- `unique = ["save_id", "player_id", "snapshot_date"]` 防止同月重复写入。

### 6. 枚举定义

```kotlin
// model/enums/GrowthRangeTier.kt
enum class GrowthRangeTier {
    FULL,       // 一线队 + 预备队：完整 10 因子计算

}

// model/enums/GrowthEventType.kt
```

### 1. MonthlyGrowthExecutor（月结执行器，入口）

```kotlin
// api/MonthlyGrowthExecutor.kt
class MonthlyGrowthExecutor(
    private val saveDb: SaveDatabase,
    private val historyDb: HistoryDatabase,
    private val inputCollector: GrowthInputCollector,
    private val rangeProcessor: ActiveRangeProcessor,
    private val snapshotWriter: GrowthSnapshotWriter,
    private val eventTrigger: GrowthEventTrigger,
    private val reportGenerator: MonthlyGrowthReportGenerator,
    private val config: MonthlyGrowthConfig,
    private val perfLogger: PerfLogger
) {

    /**
     * 月结入口（由 T07 MonthlyTaskExecutor 每月 1 日调用）
     *
     * @param saveId 存档 ID
     * @param clubId 俱乐部 ID（单俱乐部月结，T07 遍历活跃范围俱乐部逐个调用）
     * @param executionDate 月结日期（月初 1 日）
     * @return 月结汇总结果
     */
    suspend fun executeMonthlyGrowth(
        saveId: Int,
        clubId: Int,
        executionDate: LocalDate
    ): MonthlyGrowthResult {
        val startTime = System.currentTimeMillis()
        val month = executionDate.format(DateTimeFormatter.ofPattern("yyyy-MM"))
        val seasonId = saveDb.saveWorldStateDao().getCurrentSeasonId(saveId)

        return try {
            // 1. 拉取俱乐部全部球员（含一线队/预备队/U21/U18/外租）
            val allPlayers = saveDb.savePlayerStateDao().getByClubIncludeLoan(saveId, clubId)
            val club = saveDb.saveClubStateDao().getByClub(saveId, clubId)!!

            // 2. 按活跃范围分流
            val tieredPlayers = rangeProcessor.classifyByRange(allPlayers, clubId, saveId)

            // 3. 批量收集输入（一次查多表，避免 N+1）
            val inputs = inputCollector.collectBatch(saveId, clubId, tieredPlayers, executionDate, seasonId)

            // 4. 分档执行
            val results = mutableListOf<Pair<GrowthInput, GrowthResult>>()
            for (input in inputs) {
                val result = rangeProcessor.processByRange(input)
                if (result != null) {
                    results.add(input to result)
                }
            }

            // 5. 事务化应用成长 + 写快照
            val snapshots = saveDb.runInTransaction {
                val applied = results.map { (input, result) ->
                    rangeProcessor.applyGrowth(input, result)
                }
                val snaps = snapshotWriter.writeBatch(applied, saveId, seasonId, executionDate)
                snaps
            }

            // 6. 触发事件
            val events = eventTrigger.detectAndTriggerBatch(results, snapshots, saveId, executionDate)

            // 7. 生成报告
            val topGrowers = reportGenerator.topGrowers(results, snapshots, limit = 5)
            val topDecliners = reportGenerator.topDecliners(results, snapshots, limit = 5)

            val duration = System.currentTimeMillis() - startTime
            perfLogger.log("monthly_growth_club", duration, saveId, mapOf(
                "clubId" to clubId.toString(),
                "players" to allPlayers.size.toString(),
                "snapshots" to snapshots.size.toString()
            ))

            // 8. 性能告警
            if (duration > config.perfWarningClubMs) {
                perfLogger.warn("月结单俱乐部耗时 ${duration}ms 超阈值 ${config.perfWarningClubMs}ms")
            }

            MonthlyGrowthResult(
                saveId = saveId,
                clubId = clubId,
                month = month,
                executionDate = executionDate,
                totalPlayers = allPlayers.size,
                processedPlayers = results.size,
                skippedPlayers = allPlayers.size - results.size,
                rangeBreakdown = tieredPlayers.mapValues { it.value.size },
                snapshotsWritten = snapshots.size,
                eventsTriggered = events.size,
                topGrowers = topGrowers,
                topDecliners = topDecliners,
                eventList = events,
                durationMs = duration,
                success = true
            )

        } catch (e: Exception) {
            val duration = System.currentTimeMillis() - startTime
            perfLogger.error("月结失败 saveId=$saveId clubId=$clubId", e)
            MonthlyGrowthResult(
                saveId = saveId, clubId = clubId, month = month,
                executionDate = executionDate, totalPlayers = 0, processedPlayers = 0,
                skippedPlayers = 0, rangeBreakdown = emptyMap(),
                snapshotsWritten = 0, eventsTriggered = 0,
                topGrowers = emptyList(), topDecliners = emptyList(),
                eventList = emptyList(), durationMs = duration,
                success = false, errorMessage = e.message
            )
        }
    }
}
```

### 2. GrowthInputCollector（多源输入聚合器）

```kotlin
// input/GrowthInputCollector.kt
class GrowthInputCollector(
    private val saveDb: SaveDatabase,
    private val historyDb: HistoryDatabase,
    private val trainingAggregator: MonthlyTrainingAggregator,
    private val playingTimeAggregator: MonthlyPlayingTimeAggregator,
    private val clubContextLoader: ClubContextLoader,
    private val config: MonthlyGrowthConfig
) {

    /**
     * 批量收集输入（避免 N+1，一次查多表，内存中关联）
     */
    suspend fun collectBatch(
        saveId: Int,
        clubId: Int,
        tieredPlayers: Map<GrowthRangeTier, List<SavePlayerStateEntity>>,
        executionDate: LocalDate,
        seasonId: Int
    ): List<GrowthInput> {
        val month = executionDate.format(DateTimeFormatter.ofPattern("yyyy-MM"))
        val allPlayers = tieredPlayers.values.flatten()
        val playerIds = allPlayers.map { it.playerId }

        // 1. 一次性批量查询（每张表只查 1 次）
        val playerBases = historyDb.playerDao().getByIds(playerIds).associateBy { it.playerId }
        val attributesMap = historyDb.playerAttributeDao()
            .getLatestByPlayerIds(playerIds, seasonId).associateBy { it.playerId }
        val trainingMap = saveDb.monthlyTrainingRecordDao()
            .getByClubMonth(saveId, clubId, month).associateBy { it.playerId }
        val playingTimeMap = saveDb.monthlyPlayingTimeDao()
            .getByClubMonth(saveId, clubId, month).associateBy { it.playerId }
        val injuries = saveDb.saveInjuryDao().getActiveByClub(saveId, clubId).associateBy { it.playerId }
        val injuryHistoryMap = saveDb.saveInjuryDao()
            .getHistoryByClubSince(saveId, clubId, executionDate.minusMonths(6)).groupBy { it.playerId }
        val moraleMap = saveDb.playerMoraleDao().getByClub(saveId, clubId).associateBy { it.playerId }
        val mentorAssignments = saveDb.mentorAssignmentDao().getByClub(saveId, clubId).associateBy { it.playerId }
        val mentorPlayerIds = mentorAssignments.values.mapNotNull { it.mentorPlayerId }.distinct()
        val mentorPlayers = if (mentorPlayerIds.isNotEmpty())
            saveDb.savePlayerStateDao().getByIds(saveId, mentorPlayerIds).associateBy { it.playerId }
        else emptyMap()

        // 2. 俱乐部上下文（只查 1 次）
        val club = saveDb.saveClubStateDao().getByClub(saveId, clubId)!!
        val clubCtx = clubContextLoader.load(saveId, club, executionDate)
        val youthAcademy = saveDb.youthAcademyStateDao().getByClub(saveId, clubId)

        // 3. 组装输入
        return allPlayers.mapNotNull { player ->
            val base = playerBases[player.playerId] ?: return@mapNotNull null
            val attrs = attributesMap[player.playerId] ?: return@mapNotNull null
            val age = calculateAge(base.birthDate, executionDate)
            val phase = classifyPhase(age)
            val tier = tieredPlayers.entries.find { it.value.contains(player) }?.key
                ?: GrowthRangeTier.MINIMAL

            val training = trainingMap[player.playerId] ?: trainingAggregator.buildDefault(
                saveId, player, clubId, month, executionDate
            )
            val playingTime = playingTimeMap[player.playerId] ?: playingTimeAggregator.buildDefault(
                saveId, player, clubId, month
            )
            val mentorAsg = mentorAssignments[player.playerId]
            val mentor = mentorAsg?.mentorPlayerId?.let { mentorPlayers[it] }
            val mentorEffect = mentorAsg?.let {
                if (mentor != null) calculateMentorEffectPlaceholder(it, mentor, player)
                else 0.0
            } ?: 0.0

            GrowthInput(
                player = player,
                playerBase = base,
                attributes = attrs,
                age = age,
                growthPhase = phase,
                rangeTier = tier,
                monthlyTraining = training,
                monthlyPlayingTime = playingTime,
                activeInjury = injuries[player.playerId],
                injuryHistory = injuryHistoryMap[player.playerId] ?: emptyList(),
                morale = moraleMap[player.playerId],
                mentor = mentor,
                mentorEffect = mentorEffect,
                club = club,
                clubTrainingQuality = clubCtx.trainingQuality,
                clubCoachLevel = clubCtx.coachLevel,
                clubLeagueIntensity = clubCtx.leagueIntensity,
                tacticalFitScore = clubCtx.tacticalFitScore, // FULL 范围才精确
                youthAcademyStyle = youthAcademy?.academyStyle,
                nationTalentPoolBonus = clubCtx.nationTalentPoolBonus,
                seasonId = seasonId,
                executionDate = executionDate
            )
        }
    }

    /** 7 档年龄分类（严格对齐 T0Y） */
    private fun classifyPhase(age: Int): GrowthPhase = when (age) {
        in 14..15 -> GrowthPhase.EXPLOSIVE
        in 16..17 -> GrowthPhase.RAPID
        in 18..20 -> GrowthPhase.STEADY
        in 21..23 -> GrowthPhase.PRE_PRIME
        in 24..27 -> GrowthPhase.PRIME
        in 28..31 -> GrowthPhase.PRE_DECLINE
        else -> GrowthPhase.DECLINE
    }
}
```

```kotlin
// adapter/GrowthCalculatorAdapter.kt
class GrowthCalculatorAdapter(
    private val monthlyCalculator: MonthlyGrowthCalculator,        // T0Y
    private val potentialCalculator: PotentialRealizationCalculator, // T0Y
    private val specializationHandler: TrainingSpecializationHandler, // T0Y
    private val mentorHandler: MentorInfluenceHandler,             // T0Y
    private val injuryImpactCalculator: InjuryImpactCalculator,    // T0Y
    private val config: MonthlyGrowthConfig
) {

    /**
     * 把 GrowthInput 适配为 T0Y MonthlyGrowthCalculator 所需参数并调用
     * 不重写公式，仅做参数映射
     */
    suspend fun calculate(input: GrowthInput): GrowthResult {
        // 1. 构造 T0Y 所需的 SavePlayerStateWithAttributes
        val playerWithAttrs = mapToT0YPlayer(input)

        // 2. 构造 MonthlyPlayerStats（T0Y 出场时间统计）
        val monthlyStats = mapToMonthlyStats(input)

        // 3. 调用 T0Y 10 因子公式（核心，不重写）
        val growthResult = monthlyCalculator.calculate(
            player = playerWithAttrs,
            monthlyStats = monthlyStats,
            clubTrainingQuality = input.clubTrainingQuality,
            clubCoachLevel = input.clubCoachLevel,
            leagueIntensity = input.clubLeagueIntensity,
            tacticalFitScore = input.tacticalFitScore
        )

        // 4. 叠加导师加成（不修改 T0Y 公式，仅作为外部加成）
        val withMentor = if (input.mentorEffect > 0) {
            growthResult.copy(
                caDelta = growthResult.caDelta + (input.mentorEffect * config.caGrowthMultiplier).toInt(),
                notes = growthResult.notes + "导师加成 +${(input.mentorEffect * 100).toInt()}%"
            )
        } else growthResult

        // 5. 训练专项（若球员有个人训练 focus）
        val withSpecialization = if (input.monthlyTraining?.trainingFocus != null &&
            input.monthlyTraining.trainingFocus != "BALANCED") {
            val program = TrainingProgram(
                focus = TrainingFocus.valueOf(input.monthlyTraining.trainingFocus),
                intensity = input.monthlyTraining.intensityAvg
            )
            val specChanges = specializationHandler.applySpecialization(
                playerWithAttrs, program, input.clubTrainingQuality
            )
            withMentor.copy(
                attributeChanges = withMentor.attributeChanges + specChanges
            )
        } else withMentor

        return withSpecialization
    }

    /**
     * 调用 T0Y 潜力兑现率（8 因子，不重写）
     */
    suspend fun calculateRealization(input: GrowthInput): Double {
        val playerWithAttrs = mapToT0YPlayer(input)
        val monthlyStats = mapToMonthlyStats(input)
        return potentialCalculator.calculate(
            player = playerWithAttrs,
            stats = monthlyStats,
            trainingQuality = input.clubTrainingQuality
        )
    }

    private fun mapToT0YPlayer(input: GrowthInput): SavePlayerStateWithAttributes {
        // 把 GrowthInput 映射为 T0Y 期望的复合对象
        // 包括：currentCa, potentialPa, birthDate, primaryPosition, morale,
        //       injuryStatus, recentInjuryCount, hasMajorInjuryHistory,
        //       injuryProneness, professionalAttitude, coachingLevel,
        //       leagueIntensity, tacticalFitScore, leadership 等
        return SavePlayerStateWithAttributes(
            playerId = input.player.playerId,
            currentCa = input.player.currentCa,
            potentialPa = input.player.currentPa,
            birthDate = input.playerBase.birthDate,
            primaryPosition = input.playerBase.primaryPosition,
            morale = input.morale?.morale ?: input.player.morale,
            injuryStatus = input.player.injuryStatus,
            recentInjuryCount = input.injuryHistory.size,
            hasMajorInjuryHistory = input.injuryHistory.any { it.severity >= 3 },
            injuryProneness = input.attributes.injuryProneness,
            professionalAttitude = input.attributes.professionalism,
            coachingLevel = input.clubCoachLevel,
            leagueIntensity = input.clubLeagueIntensity,
            tacticalFitScore = input.tacticalFitScore,
            leadership = input.attributes.leadership,
            initialPotentialPa = input.attributes.pa
        )
    }

    private fun mapToMonthlyStats(input: GrowthInput): MonthlyPlayerStats {
        val pt = input.monthlyPlayingTime
        return MonthlyPlayerStats(
            minutesPlayed = pt?.minutesPlayed ?: 0,
            maxPossibleMinutes = pt?.maxPossibleMinutes ?: 0,
            matchesPlayed = pt?.matchesPlayed ?: 0,
            avgRating = pt?.avgRating ?: 0.0
        )
    }
}
```

```kotlin
    /**
     * 月度更新潜力兑现：
     * 1. 调用 T0Y 8 因子计算 realization_score
     * 2. 根据兑现率调整 current_pa（兑现率低 → 降 PA）
     * 3. 重伤后额外降 PA（T0Y applyMajorInjury 已处理，此处仅月度复查）
     */
    suspend fun updateMonthly(input: GrowthInput): PotentialUpdate {
        val realization = adapter.calculateRealization(input)

        // T0Y adjustPotentialPa：兑现率低则 PA 下调
        val playerWithAttrs = adapter.mapToT0YPlayer(input)
        val adjustedPa = t0yPotentialCalc.adjustPotentialPa(playerWithAttrs, realization)

        val paBefore = input.player.currentPa
        val paAfter = adjustedPa.coerceAtLeast(input.player.currentCa)
        val paDelta = paAfter - paBefore

        // 重伤后 PA 持续影响（V0.2 §十，T0Y 已定义，此处月度复查）
        val majorInjuryPaPenalty = calculateMajorInjuryPaPenalty(input)

        return PotentialUpdate(
            realizationScore = realization,
            paBefore = paBefore,
            paAfter = paAfter - majorInjuryPaPenalty,
            paDelta = paDelta - majorInjuryPaPenalty,
            majorInjuryPaPenalty = majorInjuryPaPenalty
        )
    }

    /**
     * V0.2 §十 重大伤病持续影响
     * ACL/跟腱等重伤恢复后 6 个月内每月再扣 PA（递减）
     */
    private fun calculateMajorInjuryPaPenalty(input: GrowthInput): Int {
        val recentMajor = input.injuryHistory.filter {
            it.severity >= 3 &&
            it.injuryType in listOf("ACL", "ACHILLES", "MENISCUS", "FRACTURE")
        }
        if (recentMajor.isEmpty()) return 0

        // 距离重伤月数越近，惩罚越大（第 1 月 -1，第 2 月 -1，第 3 月 0...）
        val penalty = recentMajor.sumOf { injury ->
            val monthsSince = ChronoUnit.MONTHS.between(
                LocalDate.parse(injury.startDate),
                input.executionDate
            )
            when {
                monthsSince <= 1 -> config.majorInjuryPaPenaltyMonth1  // -1
                monthsSince <= 3 -> config.majorInjuryPaPenaltyMonth3  // -1
                monthsSince <= 6 -> config.majorInjuryPaPenaltyMonth6  // 0
                else -> 0
            }
        }
        return penalty
    }
}
```

```kotlin
class GrowthEventDetector(
    private val saveDb: SaveDatabase,
    private val config: MonthlyGrowthConfig
) {

    /** 1. 突破性成长：月度 CA 增长 > 阈值（默认 +3） */
    fun detectBreakthrough(input: GrowthInput, result: GrowthResult): GrowthEventEntity? {
        if (result.caDelta <= config.breakthroughCaThreshold) return null
        return buildEvent(
            input, result, GrowthEventType.BREAKTHROUGH_GROWTH,

            title = "突破性成长",
            description = "${input.playerBase.realName} 本月 CA +${result.caDelta}，状态爆表！",
            metricValue = result.caDelta.toDouble(),
            threshold = config.breakthroughCaThreshold.toDouble()
        )
    }

    /** 2. 成长停滞：连续 N 月 CA 增长 < 阈值（默认 3 月 < 1） */
    suspend fun detectStagnation(
        input: GrowthInput, result: GrowthResult, saveId: Int, dateStr: String
    ): GrowthEventEntity? {
        if (input.age >= config.declineAgeStart) return null // 衰退期不算停滞
        val startDate = LocalDate.parse(dateStr).minusMonths(config.stagnationMonths.toLong()).toString()
        val stagnationCount = saveDb.growthSnapshotDao().countStagnationMonths(
            saveId, input.player.playerId, startDate, config.stagnationCaThreshold
        )
        if (stagnationCount < config.stagnationMonths) return null
        if (result.caDelta >= config.stagnationCaThreshold) return null
        return buildEvent(
            input, result, GrowthEventType.GROWTH_STAGNATION,
            severity = "WARN",
            title = "成长停滞",
            description = "${input.playerBase.realName} 已连续 ${config.stagnationMonths} 个月成长缓慢，需要比赛时间！",
            metricValue = stagnationCount.toDouble(),
            threshold = config.stagnationMonths.toDouble()
        )
    }

    /** 3. 潜力兑现：CA 达到 PA 的 90% */
    fun detectPotentialFulfilled(input: GrowthInput, result: GrowthResult): GrowthEventEntity? {
        val ratio = input.player.currentCa.toDouble() / input.player.currentPa.coerceAtLeast(1)
        if (ratio < config.potentialFulfilledRatio) return null
        // 防重复：检查近 30 天是否已触发
        return buildEvent(
            input, result, GrowthEventType.POTENTIAL_FULFILLED,
            severity = "INFO",
            title = "潜力兑现",
            description = "${input.playerBase.realName} 已兑现 ${(ratio * 100).toInt()}% 潜力！",
            metricValue = ratio,
            threshold = config.potentialFulfilledRatio
        )
    }

    /** 4. 早衰：28 岁前开始衰退（CA 月度下降） */
    fun detectEarlyDecline(input: GrowthInput, result: GrowthResult): GrowthEventEntity? {
        if (input.age >= config.earlyDeclineAgeThreshold) return null
        if (result.caDelta >= 0) return null
        return buildEvent(
            input, result, GrowthEventType.EARLY_DECLINE,
            severity = "CRITICAL",
            title = "早衰警告",
            description = "${input.playerBase.realName} 仅 ${input.age} 岁就出现能力下降，建议检查伤病历史！",
            metricValue = input.age.toDouble(),
            threshold = config.earlyDeclineAgeThreshold.toDouble()
        )
    }

    /** 5. 训练态度恶化：职业态度月降 > 5（由导师负面影响或士气低导致） */
    fun detectAttitudeDeterioration(input: GrowthInput, result: GrowthResult): GrowthEventEntity? {
        // 此处需要与上月职业态度对比，简化为：导师负向且职业态度 < 40
        val attitude = input.attributes.professionalism
        if (attitude >= config.attitudeLowThreshold) return null
        if (input.mentorEffect >= 0) return null // 仅负向导师触发
        return buildEvent(
            input, result, GrowthEventType.ATTITUDE_DETERIORATION,
            severity = "WARN",
            title = "训练态度恶化",
            description = "${input.playerBase.realName} 职业态度仅 ${attitude}，训练表现下滑！",
            metricValue = attitude.toDouble(),
            threshold = config.attitudeLowThreshold.toDouble()
        )
    }

    /** 6. 导师正向影响：导师加成 > 0.08 */
    fun detectMentorPositive(input: GrowthInput, result: GrowthResult): GrowthEventEntity? {
        if (input.mentorEffect < config.mentorPositiveThreshold) return null
        return buildEvent(
            input, result, GrowthEventType.MENTOR_POSITIVE,
            severity = "INFO",
            title = "导师正向影响",
            description = "${input.mentor?.let { "受导师指导" } ?: "导师"} 让 ${input.playerBase.realName} 收获成长！",
            metricValue = input.mentorEffect,
            threshold = config.mentorPositiveThreshold,
            relatedPlayerId = input.mentor?.playerId
        )
    }

        title: String,

        metricValue: Double,
        threshold: Double,

            saveId = input.player.saveId,
            playerId = input.player.playerId,
            clubId = input.club.clubId,
            triggerDate = input.executionDate.toString(),

            severity = severity,
            title = title,
            description = description,
            caAtTrigger = result.caAfter,
            paAtTrigger = input.player.currentPa,
            metricValue = metricValue,
            threshold = threshold,
            relatedPlayerId = relatedPlayerId,
            isRead = false
        )
    }
```

```kotlin
// apply/GrowthApplier.kt
class GrowthApplier(
    private val saveDb: SaveDatabase,
    private val anomalyGuard: AnomalyGuard
) {

    /**
     * 应用成长（事务化）
     * 1. 异常保护（防小妖满潜 / 防老将不衰退）
     * 2. 更新 CA/PA
     * 3. 更新属性
     */
    suspend fun apply(
        input: GrowthInput,
        result: GrowthResult,
        potentialUpdate: PotentialUpdate
    ) {
        val guarded = anomalyGuard.protect(input, result)

        saveDb.runInTransaction {
            // CA
            saveDb.savePlayerStateDao().updateCa(input.player.id, guarded.caAfter)
            // PA
            saveDb.savePlayerStateDao().updatePa(input.player.id, potentialUpdate.paAfter)
            // 属性（4 类：技术/精神/身体/门将，写回 save.db 属性镜像表）
            guarded.attributeChanges.forEach { (attr, delta) ->
                if (delta != 0) {
                    saveDb.savePlayerAttributeDao().applyDelta(input.player.saveId, input.player.playerId, attr, delta)
                }
            }
        }
    }
}

// apply/AnomalyGuard.kt
class AnomalyGuard(private val config: MonthlyGrowthConfig) {

    /**
     * V0.2 §十五 成长异常保护（复用 T0Y §十一 思路，T09 运行时再次校验）
     * 防止：
     * 1. 小妖成长过快（单月超上限）
     * 2. 老将不衰退（衰退期 CA 不增）
     * 3. 已达潜力继续成长
     * 4. 衰退期身体属性不下降
     */
    fun protect(input: GrowthInput, result: GrowthResult): GrowthResult {
        var caDelta = result.caDelta
        var attributeChanges = result.attributeChanges

        // 1. 单月成长上限（按年龄）
        val maxMonthly = when {
            input.age <= 19 -> config.maxMonthlyGrowthYoung       // 3
            input.age <= 23 -> config.maxMonthlyGrowthMid          // 4
            input.age <= 27 -> config.maxMonthlyGrowthPrime        // 2
            else -> 0                                               // 衰退期不增
        }
        if (caDelta > maxMonthly) {
            caDelta = maxMonthly
        }

        // 2. 衰退期强制不增长（28+ 不再 +CA）
        if (input.age >= config.declineAgeStart && caDelta > 0) {
            caDelta = 0
        }

        // 3. 已达潜力不再增长
        if (input.player.currentCa >= input.player.currentPa && caDelta > 0) {
            caDelta = 0
        }

        // 4. 早衰保护：32+ 强制身体属性下降
        if (input.age >= config.forceDeclineAgeStart) {
            attributeChanges = attributeChanges.toMutableMap().apply {
                this["pace"] = (this["pace"] ?: 0) - config.forceDeclinePaceDelta
                this["acceleration"] = (this["acceleration"] ?: 0) - config.forceDeclineAccDelta
            }
        }

        return result.copy(caDelta = caDelta, caAfter = result.caBefore + caDelta, attributeChanges = attributeChanges)
    }
}
```

```
        inputs = inputCollector.collectBatch(saveId, clubId, tieredPlayers, executionDate, seasonId)

        # 4. 分档执行（每档调用对应 Handler）
        results = []
        for input in inputs:
            result = activeRangeProcessor.processByRange(input)
            if result != null:
                results.append((input, result))

        # 5. 事务化写快照
        snapshots = saveDb.runInTransaction {
            applied = results.map { (input, result) ->
                activeRangeProcessor.applyGrowth(input, result)
            }
            snapshotWriter.writeBatch(applied, saveId, seasonId, executionDate)
        }

        # 6. 触发事件
        events = eventTrigger.detectAndTriggerBatch(results, snapshots, saveId, executionDate)

        # 7. 性能日志
        duration = now() - startTime
        perfLogger.log("monthly_growth_club", duration, saveId, {
            clubId, players: allPlayers.size, snapshots: snapshots.size
        })
        if duration > config.perfWarningClubMs:
            perfLogger.warn("月结单俱乐部超时")

        return MonthlyGrowthResult(success=true, ...)

    catch e:
        perfLogger.error("月结失败", e)
        return MonthlyGrowthResult(success=false, errorMessage=e.message)
```

### 3. 活跃范围分流

```
function classifyByRange(players, clubId, saveId):
    result = {FULL: [], ACTIVE: [], LIGHT: [], MINIMAL: []}
    for p in players:
        if p.careerStatus != "active": continue

        tier = when:
            p.loanClubId != null                              -> LIGHT
            p.squadRole in ["starter", "backup", "prospect"]   -> FULL
            p.squadRole in ["u21", "u18", "youth"]             -> ACTIVE
            else                                               -> MINIMAL

        result[tier].append(p)
    return result

function processByRange(input):
    switch input.rangeTier:
        case FULL:
            # 完整 10 因子（调用 T0Y）
            result = adapter.calculate(input)
            potentialUpdate = potentialCalc.updateMonthly(input)
            applier.apply(input, result, potentialUpdate)
            return result

        case ACTIVE:
            # 简化 10 因子（tacticalFit 用默认 0.7）
            simplifiedInput = input.copy(tacticalFitScore = 0.7)
            result = adapter.calculate(simplifiedInput)
            potentialUpdate = potentialCalc.updateMonthly(simplifiedInput)
            applier.apply(simplifiedInput, result, potentialUpdate)
            return result

        case LIGHT:
            # 月度汇总：年龄系数 × 出场比例 × 微调系数
            ageBase = ageTable.getGrowthFactor(input.age, input.player.currentPa)
            playingFactor = mapMinutesRatioToFactor(input.monthlyPlayingTime?.minutesRatio)
            caDelta = int(ageBase * playingFactor * config.lightGrowthMultiplier)
            caAfter = clamp(caBefore + caDelta, 1, input.player.currentPa)
            saveDb.savePlayerStateDao.updateCa(input.player.id, caAfter)
            return GrowthResult(caDelta, attributeChanges={})

        case MINIMAL:
            # 仅退役检查
            retireAge = ageTable.getRetireAge(input.player)
            if input.age < retireAge: return null
            saveDb.savePlayerStateDao.updateCareerStatus(input.player.id, "retired")
            return GrowthResult(caDelta=0, notes=["退役"])
```

### 4. 成长事件判定

```
function detectAndTriggerBatch(results, snapshots, saveId, executionDate):
    events = []
    dateStr = format(executionDate, "yyyy-MM-dd")

    for (input, result) in results:
        # 1. 突破性成长：CA 增长 > 阈值
        if result.caDelta > config.breakthroughCaThreshold:
            events.append(buildEvent(BREAKTHROUGH_GROWTH, "INFO",
                "突破性成长", "本月 CA +${result.caDelta}"))

        # 2. 成长停滞：连续 3 月 CA 增长 < 阈值
        if input.age < config.declineAgeStart:
            startDate = dateStr - 3 months
            stagnationCount = snapshotDao.countStagnationMonths(
                saveId, input.player.playerId, startDate, config.stagnationCaThreshold)
            if stagnationCount >= 3 and result.caDelta < config.stagnationCaThreshold:
                events.append(buildEvent(GROWTH_STAGNATION, "WARN",
                    "成长停滞", "连续 ${stagnationCount} 月成长缓慢"))

        # 3. 潜力兑现：CA / PA >= 0.9
        ratio = input.player.currentCa / input.player.currentPa
        if ratio >= config.potentialFulfilledRatio:
            events.append(buildEvent(POTENTIAL_FULFILLED, "INFO",
                "潜力兑现", "已兑现 ${(ratio*100).toInt()}%"))

        # 4. 早衰：28 岁前 CA 下降
        if input.age < config.earlyDeclineAgeThreshold and result.caDelta < 0:
            events.append(buildEvent(EARLY_DECLINE, "CRITICAL",
                "早衰警告", "${input.age} 岁出现能力下降"))

        # 5. 训练态度恶化：职业态度 < 阈值 + 负向导师
        if input.attributes.professionalism < config.attitudeLowThreshold and input.mentorEffect < 0:
            events.append(buildEvent(ATTITUDE_DETERIORATION, "WARN",
                "训练态度恶化", "职业态度 ${input.attributes.professionalism}"))

        # 6. 导师正向影响：mentorEffect > 阈值
        if input.mentorEffect > config.mentorPositiveThreshold:
            events.append(buildEvent(MENTOR_POSITIVE, "INFO",
                "导师正向影响", "导师加成 +${(input.mentorEffect*100).toInt()}%"))

    # 防重复：同月同球员同类事件只保留 1 条
    dedupedEvents = dedupByPlayerTypeMonth(events, dateStr)

    # 批量入库
    if dedupedEvents.isNotEmpty():
        growthEventDao.insertAll(dedupedEvents)

    return dedupedEvents
```

```kotlin
class GrowthSnapshotQuery(private val saveDb: SaveDatabase, private val historyDb: HistoryDatabase) {

    /**
     * 查询球员成长曲线点（T04 成长曲线图数据源）
     * - 当前赛季：从 save.db growth_snapshot 读月度快照
     * - 历史赛季：从 history.db player_attributes 读赛季末快照

    suspend fun getCurvePoints(saveId: Int, playerId: Int, seasonId: Int): List<GrowthCurvePoint> {
        val currentSeasonId = saveDb.saveWorldStateDao().getCurrentSeasonId(saveId)
        return if (seasonId == currentSeasonId) {

                .map { GrowthCurvePoint(date = it.snapshotDate, ca = it.caAfter, pa = it.paAfter) }

            historyDb.playerAttributeDao().getByPlayerSeason(playerId, seasonId)
                .map { GrowthCurvePoint(date = "${it.seasonId}", ca = it.ca, pa = it.pa) }
        }
    }
}
```

### 2. 性能优化策略

#### 2.1 批量查询避免 N+1

```
错误做法（N+1）：

    training = trainingDao.get(saveId, player.id, month)  # 100 次查询

正确做法（1 次）：
trainingMap = trainingDao.getByClubMonth(saveId, clubId, month).associateBy { it.playerId }
```

#### 2.2 事务化批量写入

```kotlin
saveDb.runInTransaction {
    snapshots.forEach { snapshotDao.insert(it) }  // 一次事务提交
}
```

#### 2.3 活跃范围分流（核心策略）

| 范围 | 球员数 | 单球员耗时 | 小计 | 占比 |
|------|--------|------------|------|------|
| FULL | 30 | 10ms | 300ms | 39% |
| ACTIVE | 40 | 5ms | 200ms | 26% |
| LIGHT | 10 | 2ms | 20ms | 3% |
| MINIMAL | 0 | 1ms | 0ms | 0% |
| 快照写入 | - | - | 200ms | 26% |
| 事件触发 | - | - | 50ms | 6% |
| **总计** | **80** | - | **770ms** | **100%** |

#### 2.4 异步并行（可选优化）

```kotlin
// FULL 与 ACTIVE 范围可并行（无共享数据）
```

- 当赛季：保留全部月度快照（约 10 个月 × 100 球员 = 1000 条，≤500KB）
- 上赛季：仅保留赛季末快照（T19 归档后删除明细，100 条）

## 十五、风险与应对

### 1. T0Y 算法集成偏差

- **表现**：T09 适配层把参数映射错，导致 T0Y 计算结果与单测不一致。
- **应对**：
  - W36 D1 编写"参数映射一致性测试"：固定输入跑 T0Y 单测与 T09 适配层，对比 `GrowthResult` 完全一致。
  - `GrowthCalculatorAdapter` 不修改任何 T0Y 公式，仅做参数对象转换。
  - 10 因子权重配置读取后做 checksum 校验，与 `growth_config.json` 比对。
- **红线**：T09 不得重写任何 T0Y 公式。

### 2. 性能不达标（单俱乐部 > 1s）

- **表现**：100 球员月结超过 1s，拖慢 T07 每日推进。
- **应对**：
  - 收紧活跃范围：把更多球员归到 LIGHT（仅 CA 微调）。
  - 批量查询优化：检查是否有 N+1 查询（用 `EXPLAIN QUERY PLAN` 验证索引命中）。
  - 异步并行：FULL 与 ACTIVE 范围用 `scope.async` 并行。
  - 快照写入：把 100 条 INSERT 合并为 1 个事务（已是事务化，检查是否真的批量）。
- **红线**：不削减 FULL 范围（一线队必须完整计算）。

### 3. 小妖全部满潜（异常保护失效）

- **表现**：20 年长程后高 PA 球员普遍达到 PA 上限。
- **应对**：
  - `AnomalyGuard` 三重保护：单月上限 + 衰退期不增 + 已达潜力不涨。
  - 潜力兑现率：低职业态度球员 PA 下调（T0Y `adjustPotentialPa`）。
  - 重伤 PA 惩罚：ACL 等重伤每月扣 PA（递减 6 个月）。
  - Gate 3 长程测试专项验证：高 PA 球员兑现率分布合理（平均 70-80%，不全部 100%）。
- **红线**：单月 CA 增长 ≤3，28+ 球员 CA 不再增长。

### 4. 快照体积膨胀（20 年存档 > 80MB）

- **表现**：每月 100 球员快照 × 240 个月 = 24000 条，attributes_json 字段过大。
- **应对**：
  - 快照保留策略：当赛季保留全部，上赛季仅赛季末（`applyRetentionPolicy`）。
  - T19 归档：赛季末快照写入 history.db `player_attributes` 后，删除 save.db 当赛季明细。
  - attributes_json 仅存关键字段（CA/PA/10 因子），不存全属性（全属性查 history.db）。
- **红线**：20 年存档 ≤80MB（Gate 3 验收）。

### 5. T08/T23/T16 集成时序冲突

- **表现**：T09 排在 W35-W37，但 T16（W56-W58）/T23（W69-W70）排在后面，月结时这些表不存在。
- **应对**：
  - `GrowthInputCollector` 对未存在的表做容错（查不到则用默认值）。
  - mentor_assignment / player_morale / youth_academy_state 表不存在时，mentorEffect=0、morale=50、youthStyle=null。
  - 接口预留：`GrowthInput` 字段全部 nullable，下游任务接入后填值。
  - W37 集成测试用桩数据验证：mentorEffect=0 时月结正常，不崩溃。

### 6. 月结失败导致 T07 整体回滚

- **表现**：月结抛异常，T07 `advanceOneDay` 回滚到上一日，工资/董事会等已执行的月度任务丢失。
- **应对**：
  - T09 月结内部 try-catch，失败返回 `success=false`，不抛异常给 T07。
  - T07 接到 `success=false` 时不回滚，仅记录待办"上月成长异常"。
  - 下次月结时跳过已写快照的球员（idempotent，靠 `REPLACE` 策略）。
- **红线**：月结失败不影响其他月度任务执行。

### 7. 同月多次月结（玩家读旧存档）

- **表现**：玩家在月初月结后读上月存档，再次推进到月初，触发重复月结。
- **应对**：
  - `growth_snapshot` 唯一索引 `(save_id, player_id, snapshot_date)` 防重复。
  - `insertAll` 用 `OnConflictStrategy.REPLACE`：同月重复月结覆盖旧快照。
  - `save_player_state` 的 CA/PA 也是覆盖写，不会叠加。
- **验证**：W37 测试"同月连续 2 次月结"，结果与 1 次一致。
