## 一、业务目标与范围

### 1. 业务目标

为《绿茵王朝：2002》提供完整可靠的存档运行时管理系统，覆盖存档全生命周期：创建、加载、保存、切换、删除、版本迁移、完整性校验、自动备份。存档是足球经理游戏最重要的资产，必须保证不损坏、可迁移、体积可控、崩溃可恢复。

T03 在 T00 三库结构（表结构定义 + 基础 DatabaseManager）之上，构建完整的运行时存档管理层，包含统一入口、异步保存、checkpoint 回退、版本迁移、完整性检测、自动备份、cache 重建、UI 交互等能力。

### 2. V1 做的范围

| 类别 | 功能内容 | 目标 |
|------|----------|------|
| 创建新存档 | 剧本选择 + 模式开关 + 加载范围 + 俱乐部选择 + 初始化 save.db + 初始化 cache.db + 记录元信息 | 可创建完整可玩存档 |
| 加载存档 | 存档列表 + 选择 + 完整性校验 + WAL 模式打开 + cache 重建 + 状态恢复 | 可加载并继续游戏 |
| 保存存档 | 自动保存（每日推进后）+ 手动保存 + checkpoint + 异步保存 + 失败回滚 | 保存不丢数据不卡 UI |
| 切换存档 | 关闭当前 + 选择新存档 + 加载新存档 | 多存档可切换 |
| 删除存档 | 二次确认 + 删除 save.db + 删除元信息 + 可选删 cache | 可清理存档 |
| 存档元信息 | ID/名称/创建时间/最后保存时间/游戏内日期/俱乐部/剧本/范围/开关/版本/大小 | 元信息完整可查 |
| 多存档管理 | 列表排序 + 名称搜索 + 数量上限（≤20） | 多存档可管理 |
| 版本迁移 | 版本号管理 + schema 升级 + 兼容性检查 + 旧存档升级 | 升级不破坏存档 |
| 存档完整性 | checkpoint + 校验和 + 损坏检测 + 损坏修复（回退 checkpoint）+ 自动备份 | 损坏可检测可修复 |
| 存档性能 | 自动保存 ≤2 秒 / 加载 ≤3 秒 / 体积 ≤80MB / VACUUM ≤10 秒 / 事件压缩 | 长程不卡顿不膨胀 |
| 存档安全 | 保存失败回滚 + 异常崩溃恢复 + 存储空间检查 + 并发保护 | 安全可靠 |

### 3. V1 不做（推迟）

- **存档云同步**：V1 仅本地存储，不联网同步
- **存档压缩加密**：V1 不对存档文件压缩加密
- **多存档并发加载**：V1 同时只能加载一个存档（DatabaseManager 单例 + currentSaveId 单值）
- **存档导出/分享**：V1 不支持导出存档文件分享给其他设备
- **增量保存**：V1 全量保存（每次保存写完整状态），增量保存在 V2 考虑
- **存档合并**：V1 不支持两个存档合并
- **外部存储备份**：V1 仅备份到应用私有目录，不备份到 SD 卡/云盘

### 1. 存档元信息（SaveMeta）

存档元信息采用**双存储**策略：
- **权威存储**：`SaveManifestEntity`（T00 已定义），存在于 `save_xxx.db` 内，是元数据的权威来源。

## 四、核心类设计

### 1. SaveManager（存档管理器，统一入口）

```kotlin
// api/SaveManager.kt
class SaveManager(
    private val context: Context,
    private val databaseManager: DatabaseManager,   // T00 已实现
    private val config: SaveConfig,
    private val ioDispatcher: CoroutineDispatcher = Dispatchers.IO
) {
    // 当前加载的存档 ID（null 表示未加载）
    @Volatile
    private var currentSaveId: String? = null

    // 当前存档状态
    private val _state = MutableStateFlow(SaveState.UNLOADED)
    val state: StateFlow<SaveState> = _state.asStateFlow()

    // 并发保护：每个存档一把锁，防止同时写入
    private val saveLocks = mutableMapOf<String, Mutex>()
    private val globalLock = Mutex()

    // 子管理器（懒加载）
    private val saveCreator by lazy { SaveCreator(context, databaseManager, config) }
    private val saveLoader by lazy { SaveLoader(context, databaseManager, config) }
    private val saveWriter by lazy { SaveWriter(context, databaseManager, config) }
    private val saveDeleter by lazy { SaveDeleter(context, databaseManager, config) }
    private val checkpointManager by lazy { CheckpointManager(context, databaseManager, config) }
    private val saveMigrator by lazy { SaveMigrator(databaseManager) }
    private val integrityChecker by lazy { SaveIntegrityChecker(databaseManager) }
    private val backupManager by lazy { SaveBackupManager(context, config) }
    private val cacheRebuilder by lazy { CacheRebuilder(databaseManager) }

    /**
     * 创建新存档
     */
    suspend fun createSave(request: CreateSaveRequest): SaveResult<String> = withContext(ioDispatcher) {
        _state.value = SaveState.CREATING
        try {
            // 校验存档数量上限
            val saveCount = listAllSaves().size
            if (saveCount >= config.maxSaveCount) {
                return@withContext SaveResult.Failed("存档数量已达上限 ${config.maxSaveCount}，请先删除旧存档")
            }

            // 校验存储空间
            val availableBytes = getAvailableStorage()
            if (availableBytes < config.minRequiredStorageBytes) {
                return@withContext SaveResult.Failed("存储空间不足，至少需要 ${config.minRequiredStorageMb}MB")
            }

            val saveId = saveCreator.create(request)
            currentSaveId = saveId
            _state.value = SaveState.LOADED
            SaveResult.Success(saveId)
        } catch (e: Exception) {
            _state.value = SaveState.UNLOADED
            SaveResult.Failed("创建存档失败：${e.message}", e)
        }
    }

    /**
     * 加载存档
     */
    suspend fun loadSave(saveId: String): SaveResult<LoadedSave> = withContext(ioDispatcher) {
        globalLock.withLock {
            // 如果当前已加载其他存档，先关闭
            if (currentSaveId != null && currentSaveId != saveId) {
                closeCurrentSave()
            }
        }

        _state.value = SaveState.LOADING
        try {
            val loaded = saveLoader.load(saveId)
            currentSaveId = saveId
            _state.value = SaveState.LOADED
            SaveResult.Success(loaded)
        } catch (e: Exception) {
            _state.value = SaveState.UNLOADED
            SaveResult.Failed("加载存档失败：${e.message}", e)
        }
    }

    /**
     * 保存当前存档（异步，不阻塞 UI）
     */
    suspend fun saveCurrentSave(trigger: SaveTrigger = SaveTrigger.MANUAL): SaveResult<Unit> {
        val saveId = currentSaveId ?: return SaveResult.Failed("无已加载存档")
        val lock = getSaveLock(saveId)

        return withContext(ioDispatcher) {
            lock.withLock {
                _state.value = SaveState.SAVING
                try {
                    saveWriter.save(saveId, trigger)
                    _state.value = SaveState.LOADED
                    SaveResult.Success(Unit)
                } catch (e: Exception) {
                    _state.value = SaveState.LOADED
                    SaveResult.Failed("保存失败：${e.message}", e)
                }
            }
        }
    }

    /**
     * 切换存档
     */
    suspend fun switchSave(targetSaveId: String): SaveResult<LoadedSave> = withContext(ioDispatcher) {
        globalLock.withLock {
            _state.value = SaveState.SWITCHING
            // 1. 保存当前存档（如有）
            if (currentSaveId != null) {
                saveWriter.save(currentSaveId!!, SaveTrigger.SWITCH)
            }
            // 2. 关闭当前存档
            closeCurrentSaveInternal()
            // 3. 加载新存档
            val loaded = saveLoader.load(targetSaveId)
            currentSaveId = targetSaveId
            _state.value = SaveState.LOADED
            SaveResult.Success(loaded)
        }
    }

    /**
     * 删除存档
     */
    suspend fun deleteSave(saveId: String, confirmed: Boolean): SaveResult<Unit> = withContext(ioDispatcher) {
        if (!confirmed) return@withContext SaveResult.Failed("需要二次确认")

        // 如果删除的是当前存档，先关闭
        if (currentSaveId == saveId) {
            closeCurrentSave()
        }
        saveDeleter.delete(saveId)
        SaveResult.Success(Unit)
    }

    /**
     * 列出所有存档（按最后保存时间排序）
     */
    suspend fun listAllSaves(sortBy: SaveSortBy = SaveSortBy.LAST_PLAYED): List<SaveMeta> = withContext(ioDispatcher) {
        val saveDir = File(context.filesDir, "saves")
        val metaFiles = saveDir.listFiles { f -> f.name.endsWith(".meta.json") } ?: emptyArray()

        val metas = metaFiles.mapNotNull { file ->
            try {
                SaveMetaReader.readFromFile(file)
            } catch (e: Exception) {
                // JSON 损坏，尝试从 DB 恢复
                val saveId = file.name.removePrefix("save_").removeSuffix(".meta.json")
                recoverMetaFromDb(saveId)
            }
        }

        when (sortBy) {
            SaveSortBy.LAST_PLAYED -> metas.sortedByDescending { it.lastPlayedAt }
            SaveSortBy.CREATED -> metas.sortedByDescending { it.createdAt }
            SaveSortBy.CLUB_NAME -> metas.sortedBy { it.managerClubName }
            SaveSortBy.SEASON -> metas.sortedByDescending { it.seasonsPlayed }
        }
    }

    /**
     * 搜索存档（按名称）
     */
    suspend fun searchSaves(query: String): List<SaveMeta> = withContext(ioDispatcher) {
        val all = listAllSaves()
        if (query.isBlank()) all
        else all.filter { it.saveName.contains(query, ignoreCase = true) }
    }

    /**
     * 关闭当前存档
     */
    suspend fun closeCurrentSave() = withContext(ioDispatcher) {
        globalLock.withLock { closeCurrentSaveInternal() }
    }

    private fun closeCurrentSaveInternal() {
        currentSaveId?.let {
            databaseManager.closeSaveDb()
        }
        currentSaveId = null
        _state.value = SaveState.UNLOADED
    }

    private fun getSaveLock(saveId: String): Mutex {
        return synchronized(saveLocks) {
            saveLocks.getOrPut(saveId) { Mutex() }
        }
    }

    private fun getAvailableStorage(): Long {
        val stat = StatFs(context.filesDir.path)
        return stat.availableBlocksLong * stat.blockSizeLong
    }

    private fun recoverMetaFromDb(saveId: String): SaveMeta? {
        return try {
            val db = SaveDatabase.create(context, saveId)
            val manifest = db.saveManifestDao().get() ?: return null
            val worldState = db.saveWorldStateDao().get() ?: return null
            val meta = mapManifestToMeta(manifest, worldState)
            SaveMetaWriter.writeToFile(context, meta)
            db.close()
            meta
        } catch (e: Exception) {
            null
        }
    }

    private fun mapManifestToMeta(manifest: SaveManifestEntity, worldState: SaveWorldStateEntity): SaveMeta {
        // 从 DB Entity 映射到 SaveMeta
        return SaveMeta(
            saveId = manifest.saveId,
            saveName = worldState.saveName,
            gameVersion = manifest.gameVersion,
            schemaVersion = manifest.schemaVersion,
            dataPackId = manifest.dataPackId,
            dataPackVersion = manifest.dataPackVersion,
            createdAt = manifest.createdAt,
            lastPlayedAt = manifest.lastPlayedAt,
            lastCheckpointId = manifest.lastCheckpointId,
            currentDate = manifest.currentDate,
            currentSeasonId = worldState.currentSeasonId,
            managerClubId = worldState.managerClubId,
            managerClubName = "",
            scenarioId = worldState.scenarioId ?: "",
            scenarioName = "",
            mode = worldState.mode ?: "career",
            loadRange = LoadRange(emptyList(), 0, emptyList(), emptyList()),
            switches = GameSwitches(),
            saveSizeBytes = 0,
            seasonsPlayed = 0
        )
    }
```

```kotlin
        scenario: Scenario

        val historyDb = databaseManager.getHistoryDb()
        val seasonId = 1   // 2002/03

        // 球员状态：只拷贝加载范围内的球员
        val loadRange = request.loadRange
        val activeClubIds = getActiveClubIds(historyDb, loadRange)

        // 分批拷贝球员状态（避免内存溢出）
        val batchSize = 500
        var offset = 0
        do {
            val players = historyDb.playerDao().getPlayersByClubsBatch(activeClubIds, seasonId, offset, batchSize)
            if (players.isEmpty()) break

            val states = players.map { pa ->
                val membership = historyDb.squadMembershipDao().getPlayerClubInSeason(pa.playerId, seasonId)
                SavePlayerStateEntity(
                    id = 0,
                    saveId = 1,
                    playerId = pa.playerId,
                    currentClubId = membership?.clubId,
                    loanClubId = if (membership?.isLoan == 1) membership.loanFromClubId else null,

                    currentPa = pa.pa,

                    morale = 60,
                    injuryStatus = "healthy",
                    injuryUntil = null,
                    contractUntil = membership?.contractUntil,
                    wage = membership?.wage ?: 0,
                    marketValue = membership?.marketValue ?: 0,
                    careerStatus = "active",
                    squadRole = membership?.squadRole
                )
            }
            saveDb.savePlayerStateDao().insertAll(states)
            offset += batchSize
        } while (players.size == batchSize)

        // 俱乐部状态
        activeClubIds.forEach { clubId ->
            val club = historyDb.clubDao().getById(clubId) ?: return@forEach
            saveDb.saveClubStateDao().insert(SaveClubStateEntity(
                id = 0,
                saveId = 1,

                reputation = club.reputation,
                boardSatisfaction = 60,
                fanSatisfaction = 60,
                dressingRoomMorale = 60
            ))
        }
    }

        return loadRange.leagueIds.flatMap { leagueId ->
            historyDb.clubDao().getClubIdsByLeague(leagueId)
        }
    }
```

```kotlin
// loader/SaveLoader.kt
class SaveLoader(
    private val context: Context,
    private val databaseManager: DatabaseManager,
    private val config: SaveConfig
) {

    /**
     * 加载存档完整流程
     * V0.2 §五 + §八
     */
    suspend fun load(saveId: String): LoadedSave = withContext(Dispatchers.IO) {
        // 1. 读取元信息
        val meta = SaveMetaReader.read(context, saveId)
            ?: throw SaveNotFoundException("存档不存在：$saveId")

        // 2. 校验存档文件存在
        val dbFile = File(getSaveDir(context), "save_$saveId.db")
        if (!dbFile.exists()) {
            throw SaveFileNotFoundException("存档数据库文件不存在：${dbFile.absolutePath}")
        }

        // 3. 打开 save.db（WAL 模式）
        val saveDb = SaveDatabase.create(context, saveId)

        // 4. 完整性检测
        val integrityReport = SaveIntegrityChecker(databaseManager).check(saveDb)
        if (integrityReport.isSevere()) {
            saveDb.close()
            throw SaveCorruptedException("存档损坏：${integrityReport.summary}")
        }

        // 5. 轻微错误自动修复
        if (integrityReport.hasMinorIssues()) {
            SaveRepairService(saveDb).repairMinor(integrityReport)
        }

        // 6. schema_version 迁移
        val manifest = saveDb.saveManifestDao().get()!!
        if (manifest.schemaVersion < SchemaVersion.CURRENT) {
            val migrationResult = SaveMigrator(databaseManager).migrate(saveDb, manifest.schemaVersion, SchemaVersion.CURRENT)
            when (migrationResult) {
                is MigrationResult.Failed -> {
                    saveDb.close()
                    throw SaveMigrationException("版本迁移失败：${migrationResult.reason}")
                }
                is MigrationResult.Incompatible -> {
                    saveDb.close()
                    throw SaveIncompatibleException("存档版本不兼容：${migrationResult.from} → ${migrationResult.to}")
                }
                else -> { /* 成功，继续 */ }
            }
        }

        // 7. 重建 cache.db（如需要）
        if (config.rebuildCacheOnLoad) {
            CacheRebuilder(databaseManager).rebuild(saveDb)
        }

        // 8. 更新 last_played_at
        val now = LocalDateTime.now().format(DateTimeFormatter.ISO_LOCAL_DATE_TIME)
        saveDb.saveManifestDao().updateLastPlayed(saveId, now)
        SaveMetaWriter.updateLastPlayed(context, saveId, now)

        // 9. 恢复游戏状态
        val gameState = GameStateRestorer().restore(saveDb)

        LoadedSave(saveId, saveDb, meta, gameState)
    }
}
```

```kotlin
// writer/SaveWriter.kt
class SaveWriter(
    private val context: Context,
    private val databaseManager: DatabaseManager,
    private val config: SaveConfig
) {

    /**
     * 异步保存存档
     * 流程：checkpoint → 写入 → 更新元信息 → 记录性能
     */
    suspend fun save(saveId: String, trigger: SaveTrigger): Unit = withContext(Dispatchers.IO) {
        val startTime = System.currentTimeMillis()
        val saveDb = databaseManager.getSaveDb()

        // 1. 保存前创建 checkpoint（用于回退）
        if (config.createCheckpointBeforeSave && trigger != SaveTrigger.AUTO_DAILY) {
            CheckpointManager(context, databaseManager, config).createCheckpoint(
                saveId, CheckpointType.PRE_SAVE, "保存前 checkpoint（$trigger）"
            )
        }

        // 2. 执行保存（写入数据库）
        try {
            saveDb.runInTransaction {
                // 更新 save_manifest 的 currentDate 和 lastPlayedAt
                val manifest = saveDb.saveManifestDao().get()
                val now = LocalDateTime.now().format(DateTimeFormatter.ISO_LOCAL_DATE_TIME)
                saveDb.saveManifestDao().update(
                    saveId = saveId,
                    currentDate = manifest.currentDate,
                    lastPlayedAt = now
                )
            }

            // 3. 更新元信息 JSON（双写）
            updateMetaJson(saveId)

            // 4. checkpoint 清理（保存后清理过期 PRE_SAVE checkpoint）
            if (trigger != SaveTrigger.AUTO_DAILY) {
                CheckpointManager(context, databaseManager, config).cleanByType(CheckpointType.PRE_SAVE)
            }

        } catch (e: Exception) {
            // 5. 保存失败回滚
            RollbackHandler(context, databaseManager, config).rollback(saveId)
            throw SaveWriteException("保存失败，已回滚：${e.message}", e)
        }

        // 6. 记录性能日志
        val durationMs = System.currentTimeMillis() - startTime
        recordPerfLog(saveDb, saveId, trigger, durationMs)

        // 7. 性能预警
        if (durationMs > config.autoSaveTargetMs) {
            android.util.Log.w("SaveWriter", "保存耗时 ${durationMs}ms 超过目标 ${config.autoSaveTargetMs}ms")
        }
    }

    private fun updateMetaJson(saveId: String) {
        val saveDb = databaseManager.getSaveDb()
        val manifest = saveDb.saveManifestDao().get()
        val worldState = saveDb.saveWorldStateDao().get()
        if (manifest != null && worldState != null) {
            val dbFile = File(getSaveDir(context), "save_$saveId.db")
            SaveMetaWriter.updateSaveMeta(context, saveId) { meta ->
                meta.copy(
                    lastPlayedAt = manifest.lastPlayedAt,
                    currentDate = manifest.currentDate,
                    saveSizeBytes = dbFile.length()
                )
            }
        }
    }

    private fun recordPerfLog(saveDb: SaveDatabase, saveId: String, trigger: SaveTrigger, durationMs: Long) {
        try {
            val dbFile = File(getSaveDir(context), "save_$saveId.db")
            saveDb.perfLogDao().insert(PerfLogEntity(
                id = 0,
                saveId = saveId,
                logDate = LocalDateTime.now().format(DateTimeFormatter.ISO_LOCAL_DATE_TIME),
                actionType = "save_${trigger.name.lowercase()}",
                durationMs = durationMs.toInt(),
                memoryMb = null,
                dbSizeMb = dbFile.length() / (1024.0 * 1024.0),
                extraJson = null
            ))
        } catch (e: Exception) {
            // 性能日志失败不影响保存
        }
    }
}

// writer/RollbackHandler.kt
class RollbackHandler(
    private val context: Context,
    private val databaseManager: DatabaseManager,
    private val config: SaveConfig
) {

    /**
     * 保存失败回滚
     * 从最近的 PRE_SAVE checkpoint 恢复
     */
    fun rollback(saveId: String) {
        val checkpointManager = CheckpointManager(context, databaseManager, config)
        val saveDb = databaseManager.getSaveDb()

        // 查找最近的 PRE_SAVE checkpoint
        val latestCheckpoint = saveDb.checkpointDao().getLatestByType(CheckpointType.PRE_SAVE.code)
            ?: saveDb.checkpointDao().getLatest()  // 退而求其次用最近的任意类型 checkpoint

        if (latestCheckpoint != null) {
            checkpointManager.restoreFromCheckpoint(latestCheckpoint.checkpointId)
        }
        // 如果没有 checkpoint 可恢复，无法回滚（首次保存场景）
    }
}
```

```kotlin
// checkpoint/CheckpointManager.kt
class CheckpointManager(
    private val context: Context,
    private val databaseManager: DatabaseManager,
    private val config: SaveConfig
) {

    /**
     * 创建 checkpoint
     * V0.2 §六：
     * - light: 每 30 天，日期+关键状态+队伍表
     * - season: 每赛季，完整赛季结束状态
     * - migration: 迁移前，完整备份
     * - user: 用户手动，完整备份
     * - pre_save: 保存前快照，用于回滚
     */
    suspend fun createCheckpoint(saveId: String, type: CheckpointType, description: String = ""): String =
        withContext(Dispatchers.IO) {
            val checkpointId = "cp_${UUID.randomUUID().toString().take(8)}"
            val now = LocalDateTime.now().format(DateTimeFormatter.ISO_LOCAL_DATE_TIME)
            val saveDb = databaseManager.getSaveDb()
            val manifest = saveDb.saveManifestDao().get()!!

            // 1. 创建 checkpoint 目录
            val checkpointDir = File(getCheckpointDir(context), "$saveId/$checkpointId")
            checkpointDir.mkdirs()

            // 2. 复制当前 save.db 到 checkpoint 目录
            val currentDbFile = File(getSaveDir(context), "save_$saveId.db")
            val backupDbFile = File(checkpointDir, "save_$saveId.db")
            currentDbFile.copyTo(backupDbFile, overwrite = true)

            // 3. 计算 checksum
            val checksum = ChecksumUtil.calculateMD5(backupDbFile)

            // 4. 写入 checkpoint 元信息文件
            val checkpointMeta = CheckpointMeta(
                checkpointId = checkpointId,
                saveId = saveId,
                type = type,
                checkpointDate = manifest.currentDate,
                createdAt = now,
                filePath = backupDbFile.absolutePath,
                checksum = checksum,
                fileSizeBytes = backupDbFile.length(),
                gameVersion = manifest.gameVersion
            )
            File(checkpointDir, "checkpoint.meta.json").writeText(Json.encodeToString(checkpointMeta))

            // 5. 写入 checkpoint 表记录
            saveDb.checkpointDao().insert(CheckpointEntity(
                checkpointId = checkpointId,
                saveId = saveId,
                checkpointType = type.code,
                checkpointDate = manifest.currentDate,
                filePath = backupDbFile.absolutePath,
                checksum = checksum
            ))

            // 6. 更新 save_manifest 的 last_checkpoint_id
            saveDb.saveManifestDao().updateLastCheckpoint(saveId, checkpointId)

            // 7. 清理过期 checkpoint
            cleanByType(type)

            checkpointId
        }

    /**
     * 从 checkpoint 恢复
     */
    suspend fun restoreFromCheckpoint(checkpointId: String): Boolean = withContext(Dispatchers.IO) {
        val saveDb = databaseManager.getSaveDb()
        val cp = saveDb.checkpointDao().getById(checkpointId) ?: return@withContext false
        val backupFile = File(cp.filePath)
        if (!backupFile.exists()) return@withContext false

        // 校验 checksum
        if (ChecksumUtil.calculateMD5(backupFile) != cp.checksum) {
            android.util.Log.e("CheckpointManager", "checkpoint checksum 校验失败")
            return@withContext false
        }

        // 关闭当前 DB 连接，覆盖文件，重新打开
        val saveId = cp.saveId
        databaseManager.closeSaveDb()
        val currentDbFile = File(getSaveDir(context), "save_$saveId.db")
        backupFile.copyTo(currentDbFile, overwrite = true)
        // 重新打开由 SaveLoader 处理

        true
    }

    /**
     * 清理过期 checkpoint（按类型保留数量）
     */
    suspend fun cleanByType(type: CheckpointType) = withContext(Dispatchers.IO) {
        val saveDb = databaseManager.getSaveDb()
        val maxCount = type.maxRetained
        val oldCheckpoints = saveDb.checkpointDao().getOldByType(type.code, maxCount)
        oldCheckpoints.forEach { cp ->
            // 删除 checkpoint 文件
            File(cp.filePath).delete()
            val checkpointDir = File(cp.filePath).parentFile
            checkpointDir?.let { dir ->
                if (dir.exists() && dir.listFiles()?.isEmpty() == true) dir.delete()
            }
            // 删除 checkpoint 记录
            saveDb.checkpointDao().delete(cp)
        }
    }

    companion object {
        fun getCheckpointDir(context: Context): File {
            val dir = File(context.filesDir, "checkpoints")
            if (!dir.exists()) dir.mkdirs()
            return dir
        }

        fun getSaveDir(context: Context): File {
            val dir = File(context.filesDir, "saves")
            if (!dir.exists()) dir.mkdirs()
            return dir
        }
    }
}

object ChecksumUtil {
    fun calculateMD5(file: File): String {
        val digest = MessageDigest.getInstance("MD5")
        file.inputStream().use { input ->
            val buffer = ByteArray(8192)
            var read: Int
            while (input.read(buffer).also { read = it } > 0) {
                digest.update(buffer, 0, read)
            }
        }
        return digest.digest().joinToString("") { "%02x".format(it) }
    }

    fun calculateSHA256(file: File): String {
        val digest = MessageDigest.getInstance("SHA-256")
        file.inputStream().use { input ->
            val buffer = ByteArray(8192)
            var read: Int
            while (input.read(buffer).also { read = it } > 0) {
                digest.update(buffer, 0, read)
            }
        }
        return digest.digest().joinToString("") { "%02x".format(it) }
    }
```

```kotlin
// migration/SaveMigrator.kt
class SaveMigrator(private val databaseManager: DatabaseManager) {

    /**
     * 执行版本迁移
     * V0.2 §五 流程：
     * 1. 检测 schema_version
     * 2. 创建完整备份（MIGRATION checkpoint）
     * 3. 校验剩余存储空间
     * 4. 执行迁移脚本
     * 5. 校验关键表完整性
     * 6. 失败回滚
     * 7. 成功后更新版本号
     */
    suspend fun migrate(saveDb: SaveDatabase, fromVersion: Int, toVersion: Int): MigrationResult =
        withContext(Dispatchers.IO) {
            // 1. 创建迁移备份 checkpoint
            val backupPath = CheckpointManager(
                databaseManager.getSaveDb().openHelper.context,
                databaseManager,
                SaveConfig()
            ).createCheckpoint(saveDb.saveManifestDao().get().saveId, CheckpointType.MIGRATION, "迁移 $fromVersion→$toVersion 备份")

            // 2. 校验存储空间
            val availableBytes = getAvailableStorage(databaseManager.getSaveDb().openHelper.context)
            if (availableBytes < 100 * 1024 * 1024) {
                return@withContext MigrationResult.Failed("存储空间不足，至少需要 100MB")
            }

            // 3. 逐版本执行迁移
            try {
                var currentVersion = fromVersion
                while (currentVersion < toVersion) {
                    val migrationStep = MigrationRegistry.findMigration(currentVersion, currentVersion + 1)
                        ?: return@withContext MigrationResult.Incompatible(currentVersion, currentVersion + 1)

                    when (migrationStep.type) {
                        MigrationType.INCOMPATIBLE -> {
                            return@withContext MigrationResult.Incompatible(currentVersion, currentVersion + 1)
                        }
                        MigrationType.AUTO, MigrationType.MANUAL -> {
                            saveDb.openHelper.writableDatabase.run {
                                beginTransaction()
                                try {
                                    migrationStep.executor.migrate(this)
                                    setTransactionSuccessful()
                                } finally {
                                    endTransaction()
                                }
                            }
                            currentVersion++
                        }
                    }
                }

                // 4. 校验关键表完整性
                val integrity = SaveIntegrityChecker(databaseManager).check(saveDb)
                if (integrity.isSevere()) {
                    // 5. 失败回滚
                    CheckpointManager(
                        databaseManager.getSaveDb().openHelper.context,
                        databaseManager,
                        SaveConfig()
                    ).restoreFromCheckpoint(backupPath)
                    return@withContext MigrationResult.Failed("迁移后完整性校验失败，已回滚：${integrity.summary}")
                }

                // 6. 更新 schema_version
                saveDb.saveManifestDao().updateSchemaVersion(toVersion)

                MigrationResult.Success(fromVersion, toVersion)

            } catch (e: Exception) {
                // 回滚
                CheckpointManager(
                    databaseManager.getSaveDb().openHelper.context,
                    databaseManager,
                    SaveConfig()
                ).restoreFromCheckpoint(backupPath)
                MigrationResult.Failed("迁移异常，已回滚：${e.message}")
            }
        }

    /**
     * 兼容性检查（不执行迁移）
     */
    fun checkCompatibility(fromVersion: Int, toVersion: Int): MigrationType {
        var currentVersion = fromVersion
        while (currentVersion < toVersion) {
            val step = MigrationRegistry.findMigration(currentVersion, currentVersion + 1)
                ?: return MigrationType.INCOMPATIBLE
            if (step.type == MigrationType.INCOMPATIBLE) return MigrationType.INCOMPATIBLE
            currentVersion++
        }
        return MigrationType.AUTO
    }

    private fun getAvailableStorage(context: Context): Long {
        val stat = StatFs(context.filesDir.path)
        return stat.availableBlocksLong * stat.blockSizeLong
    }
}
```

```kotlin
// integrity/SaveIntegrityChecker.kt
class SaveIntegrityChecker(private val databaseManager: DatabaseManager) {

    /**
     * V0.2 §八 完整性检测 7 项：
     * 1. manifest 是否存在
     * 2. 关键表是否存在
     * 3. save_world_state 是否只有一条当前状态
     * 4. 当前执教俱乐部是否存在
     * 5. 当前日期是否合法
     * 6. 球员状态外键是否完整
     * 7. 比赛表是否有孤儿记录
     */
    fun check(saveDb: SaveDatabase): IntegrityReport {
        val issues = mutableListOf<IntegrityIssue>()

        // 1. manifest
        val manifest = saveDb.saveManifestDao().get()
        if (manifest == null) {
            issues.add(IntegrityIssue(Severity.SEVERE, "save_manifest 不存在"))
            return IntegrityReport(issues)
        }

        // 2. 关键表存在性
        val requiredTables = listOf(
            "save_manifest", "save_world_state", "save_player_state",
            "save_club_state", "save_injury", "save_transfer_offer"
        )
        val existingTables = saveDb.openHelper.readableDatabase.query("SELECT name FROM sqlite_master WHERE type='table'")
            .use { cursor ->
                generateSequence { if (cursor.moveToNext()) cursor.getString(0) else null }.toSet()
            }
        requiredTables.forEach { table ->
            if (table !in existingTables) {
                issues.add(IntegrityIssue(Severity.SEVERE, "关键表缺失：$table"))
            }
        }

        // 3. save_world_state 唯一性
        val worldStateCount = saveDb.saveWorldStateDao().count()
        when {
            worldStateCount == 0 -> issues.add(IntegrityIssue(Severity.SEVERE, "save_world_state 为空"))
            worldStateCount > 1 -> issues.add(IntegrityIssue(Severity.MEDIUM, "save_world_state 有 $worldStateCount 条，应为 1"))
        }

        // 4. 当前执教俱乐部存在
        val worldState = saveDb.saveWorldStateDao().get()
        if (worldState != null) {
            val managerClub = saveDb.saveClubStateDao().getByClub(worldState.save_id, worldState.managerClubId)
            if (managerClub == null) {
                issues.add(IntegrityIssue(Severity.SEVERE, "当前执教俱乐部状态不存在：${worldState.managerClubId}"))
            }
        }

        // 5. 当前日期合法
        try {
            LocalDate.parse(manifest.currentDate)
        } catch (e: Exception) {
            issues.add(IntegrityIssue(Severity.SEVERE, "当前日期非法：${manifest.currentDate}"))
        }

        // 6. 球员状态外键完整性
        val orphanPlayers = saveDb.openHelper.readableDatabase.query(
            """SELECT COUNT(*) FROM save_player_state sps
               WHERE sps.player_id NOT IN (SELECT player_id FROM player)""".trimIndent()
        ).use { if (it.moveToFirst()) it.getInt(0) else 0 }
        if (orphanPlayers > 0) {
            issues.add(IntegrityIssue(Severity.MEDIUM, "球员状态有 $orphanPlayers 条孤儿记录（player_id 不存在于 history.player）"))
        }

        // 7. 孤儿比赛记录检查（简化：检查 save 内部一致性）
        // 比赛表在 history.db 中，存档本身不存比赛，此处检查 save 内部状态一致性

        return IntegrityReport(issues)
    }
}

data class IntegrityReport(val issues: List<IntegrityIssue>) {
    fun isSevere(): Boolean = issues.any { it.severity == Severity.SEVERE }
    fun hasMinorIssues(): Boolean = issues.any { it.severity == Severity.MINOR }
    val summary: String get() = issues.joinToString("; ") { "${it.severity}: ${it.description}" }
}
```

```kotlin
class BackupCleaner(private val context: Context, private val config: SaveConfig) {

    fun cleanOldBackups(saveId: String) {
        val saveBackupDir = File(SaveBackupManager.getBackupDir(context), saveId)
        if (!saveBackupDir.exists()) return

        val backups = saveBackupDir.listFiles { f -> f.isDirectory } ?: return
        val sortedBackups = backups.sortedByDescending { it.lastModified() }

        // 保留最近 maxBackups 个
        if (sortedBackups.size > config.maxBackupsPerSave) {
            sortedBackups.drop(config.maxBackupsPerSave).forEach { dir ->
                dir.deleteRecursively()
            }
        }

        // 清理超过 maxBackupAgeDays 天的备份
        val maxAgeMs = config.maxBackupAgeDays * 24 * 60 * 60 * 1000L
        val now = System.currentTimeMillis()
        backups.forEach { dir ->
            if (now - dir.lastModified() > maxAgeMs) {
                dir.deleteRecursively()
            }
        }
    }
}
```

```kotlin
     * 1. 删除 save.db 文件（含 WAL/SHM）
     * 2. 删除 meta.json
     * 3. 删除 checkpoints
     * 4. 可选删除 backups
     */
    suspend fun delete(saveId: String, deleteBackups: Boolean = false) = withContext(Dispatchers.IO) {
        val saveDir = File(context.filesDir, "saves")

        // 1. 删除 save.db 及其 WAL/SHM
        File(saveDir, "save_$saveId.db").delete()
        File(saveDir, "save_$saveId.db-wal").delete()
        File(saveDir, "save_$saveId.db-shm").delete()

        // 2. 删除 meta.json
        File(saveDir, "save_$saveId.meta.json").delete()

        // 3. 删除 checkpoints
        val checkpointDir = File(CheckpointManager.getCheckpointDir(context), saveId)
        checkpointDir.deleteRecursively()

        // 4. 可选删除 backups
        if (deleteBackups) {
            val backupDir = File(SaveBackupManager.getBackupDir(context), saveId)
            backupDir.deleteRecursively()
        }
    }
}
```

```kotlin
object SaveMetaReader {

    /**
     * 从 JSON 文件读取元信息（快速，不打开 DB）
     */
    fun read(context: Context, saveId: String): SaveMeta? {
        val metaFile = File(CheckpointManager.getSaveDir(context), "save_$saveId.meta.json")
        return readFromFile(metaFile)
    }

    fun readFromFile(file: File): SaveMeta? {
        if (!file.exists()) return null
        return try {
            Json.decodeFromString(file.readText())
        } catch (e: Exception) { null }
    }
}

// meta/SaveMetaWriter.kt
object SaveMetaWriter {

    /**
     * 双写：DB manifest + JSON 文件
     */
    suspend fun writeAll(context: Context, saveDb: SaveDatabase, meta: SaveMeta) = withContext(Dispatchers.IO) {
        // 1. 写入 DB manifest（权威）
        saveDb.saveManifestDao().insert(SaveManifestEntity(
            saveId = meta.saveId,
            gameVersion = meta.gameVersion,
            schemaVersion = meta.schemaVersion,
            dataPackId = meta.dataPackId,
            dataPackVersion = meta.dataPackVersion,
            currentDate = meta.currentDate,
            createdAt = meta.createdAt,
            lastPlayedAt = meta.lastPlayedAt,
            lastCheckpointId = meta.lastCheckpointId
        ))

        // 2. 写入 JSON 文件（快速索引）
        writeToFile(context, meta)
    }

    fun writeToFile(context: Context, meta: SaveMeta) {
        val metaFile = File(CheckpointManager.getSaveDir(context), "save_${meta.saveId}.meta.json")
        metaFile.writeText(Json.encodeToString(meta))
    }

    fun updateLastPlayed(context: Context, saveId: String, lastPlayedAt: String) {
        updateSaveMeta(context, saveId) { it.copy(lastPlayedAt = lastPlayedAt) }
    }

    fun updateSaveMeta(context: Context, saveId: String, updater: (SaveMeta) -> SaveMeta) {
        val meta = SaveMetaReader.read(context, saveId) ?: return
        val updated = updater(meta)
        writeToFile(context, updated)
    }
```

### 2. 加载存档流程

```
FUNCTION loadSave(saveId):
    // 1. 读取元信息（不打开 DB）
    meta = SaveMetaReader.read(context, saveId)
    IF meta == null:
        THROW SaveNotFoundException

    // 2. 校验文件存在
    IF NOT fileExists("saves/save_$saveId.db"):
        THROW SaveFileNotFoundException

    // 3. 打开 save.db（WAL 模式）
    saveDb = SaveDatabase.create(context, saveId)

    // 4. 完整性检测（7 项）
    report = SaveIntegrityChecker.check(saveDb)
    IF report.isSevere():
        saveDb.close()
        THROW SaveCorruptedException(report.summary)

    // 5. 轻微修复
    IF report.hasMinorIssues():
        SaveRepairService.repairMinor(report)

    // 6. 版本迁移
    manifest = saveDb.saveManifestDao().get()
    IF manifest.schemaVersion < CURRENT_SCHEMA_VERSION:
        result = SaveMigrator.migrate(saveDb, manifest.schemaVersion, CURRENT)
        IF result is Failed OR Incompatible:
            saveDb.close()
            THROW SaveMigrationException

    // 7. 重建 cache.db（如需要）
    IF config.rebuildCacheOnLoad:
        CacheRebuilder.rebuild(saveDb)

    // 8. 更新 last_played_at
    saveDb.saveManifestDao().updateLastPlayed(saveId, now)
    SaveMetaWriter.updateLastPlayed(context, saveId, now)

    // 9. 恢复游戏状态
    gameState = GameStateRestorer.restore(saveDb)

    currentSaveId = saveId
    RETURN LoadedSave(saveId, saveDb, meta, gameState)
```

### 3. 保存存档流程（含 checkpoint）

```
FUNCTION saveCurrentSave(trigger):
    saveId = currentSaveId
    IF saveId == null:
        RETURN Failed("无已加载存档")

    lock = getSaveLock(saveId)
    ACQUIRE lock:
        state = SAVING
        startTime = currentTime()

        // 1. 保存前 checkpoint（非每日自动保存时）
        IF config.createCheckpointBeforeSave AND trigger != AUTO_DAILY:
            CheckpointManager.createCheckpoint(saveId, PRE_SAVE, "保存前快照")

        // 2. 执行保存
        TRY:
            saveDb.runInTransaction {
                manifest = saveDb.saveManifestDao().get()
                saveDb.saveManifestDao().update(
                    saveId, currentDate=manifest.currentDate, lastPlayedAt=now
                )
            }
            // 3. 更新元信息 JSON
            SaveMetaWriter.updateSaveMeta(context, saveId) { meta ->
                meta.copy(lastPlayedAt=now, saveSizeBytes=dbFile.length())
            }
            // 4. 清理过期 PRE_SAVE checkpoint
            IF trigger != AUTO_DAILY:
                CheckpointManager.cleanByType(PRE_SAVE)

        CATCH e:
            // 5. 保存失败回滚
            RollbackHandler.rollback(saveId)
            THROW SaveWriteException

        // 6. 记录性能日志
        durationMs = currentTime() - startTime
        saveDb.perfLogDao().insert(PerfLog(action="save_$trigger", durationMs=durationMs, ...))

        // 7. 性能预警
        IF durationMs > config.autoSaveTargetMs:
            LOG.warn("保存耗时 ${durationMs}ms 超过目标")

        state = LOADED
        RETURN Success
```

### 4. 版本迁移流程

```
FUNCTION migrate(saveDb, fromVersion, toVersion):
    // 1. 创建迁移备份 checkpoint
    backupCheckpointId = CheckpointManager.createCheckpoint(saveId, MIGRATION, "迁移备份")

    // 2. 校验存储空间
    IF getAvailableStorage() < 100MB:
        RETURN Failed("存储空间不足")

    // 3. 逐版本执行迁移
    currentVersion = fromVersion
    WHILE currentVersion < toVersion:
        step = MigrationRegistry.findMigration(currentVersion, currentVersion + 1)
        IF step == null:
            RETURN Incompatible(currentVersion, currentVersion + 1)

        SWITCH step.type:
            CASE INCOMPATIBLE:
                RETURN Incompatible(currentVersion, currentVersion + 1)
            CASE AUTO, MANUAL:
                saveDb.beginTransaction()
                TRY:
                    step.executor.migrate(saveDb)
                    saveDb.setTransactionSuccessful()
                FINALLY:
                    saveDb.endTransaction()
                currentVersion++

    // 4. 校验关键表完整性
    integrity = SaveIntegrityChecker.check(saveDb)
    IF integrity.isSevere():
        // 5. 失败回滚
        CheckpointManager.restoreFromCheckpoint(backupCheckpointId)
        RETURN Failed("迁移后完整性校验失败，已回滚")

    // 6. 更新 schema_version
    saveDb.saveManifestDao().updateSchemaVersion(toVersion)

    RETURN Success(fromVersion, toVersion)
```

### 5. 损坏检测与修复流程

```
FUNCTION checkAndRepair(saveDb):
    // 1. 完整性检测
    report = SaveIntegrityChecker.check(saveDb)

    IF NOT report.hasAnyIssue():
        RETURN "存档完好"

    // 2. 按严重程度分级修复
    IF report.hasMinorIssues():
        SaveRepairService.repairMinor(report)
        RETURN "轻微问题已自动修复"

    IF report.hasMediumIssues():
        // 中等：从最近 checkpoint 恢复
        result = SaveRepairService.repairMedium(context, databaseManager, config)
        IF result is Success:
            // 恢复后重建缓存
            CacheRebuilder.rebuild(saveDb)
            RETURN "已从 checkpoint 恢复"
        ELSE:
            // checkpoint 恢复失败，尝试备份
            backups = SaveBackupManager.listBackups(saveId)
            IF backups.notEmpty():
                SaveBackupManager.restoreFromBackup(saveId, backups.first().backupId)
                RETURN "已从备份恢复"

    IF report.isSevere():
        result = SaveRepairService.repairSevere()
        RETURN "存档严重损坏，请从用户备份恢复"
```

```
FUNCTION executeVacuum(saveDb):
    sizeBefore = getDbSize(saveDb)
    startTime = currentTime()

    // VACUUM 必须在事务外执行
    saveDb.openHelper.writableDatabase.execSQL("VACUUM")

    durationMs = currentTime() - startTime
    sizeAfter = getDbSize(saveDb)
    savedMb = sizeBefore - sizeAfter

    // 记录性能日志
    saveDb.perfLogDao().insert(PerfLog(
        action="vacuum",
        durationMs=durationMs,
        dbSizeMb=sizeAfter,
        extraJson="{\"saved_mb\": $savedMb}"
    ))

    RETURN VacuumResult(sizeBefore, sizeAfter, savedMb, durationMs)
```

## 九、性能要求

| 操作 | 目标 | 实现策略 |
|------|------|----------|
| 自动保存（每日推进后） | ≤2 秒 | 异步协程 + WAL 模式 + 事务批量写入 + 非每日跳过 PRE_SAVE checkpoint |
| 加载存档 | ≤3 秒 | meta.json 快速列表 + WAL 模式打开 + 完整性检测优化 + cache 异步重建 |
| 手动保存 | ≤2 秒 | 同自动保存 + PRE_SAVE checkpoint |
| VACUUM 压缩 | ≤10 秒 | 事务外执行 + 赛季归档后触发 + 体积达 85% 自动触发 |
| 存档列表 | ≤500 毫秒 | meta.json 文件读取（不打开 DB）+ 内存排序 |
| 存档搜索 | ≤200 毫秒 | 内存过滤 + 名称包含匹配 |
| 创建新存档 | ≤5 秒 | 分批拷贝（500/批）+ 事务批量写入 + cache 异步重建 |
| 版本迁移（V1→V2） | ≤5 秒 | 事务执行 + checkpoint 备份在先 |

### 存档体积控制

| 存档阶段 | 目标体积 | 控制手段 |
|----------|----------|----------|
| 新开档 | ≤15MB | 仅拷贝加载范围内球员/俱乐部状态 |
| 1 赛季后 | ≤25MB | T19 赛季归档压缩事件 + VACUUM |
| 5 赛季后 | ≤45MB | 归档 + 清理日志 + VACUUM |
| 10 赛季后 | ≤60MB | 归档 + 加严清理 + VACUUM |
| 20 赛季后 | ≤80MB | V0.2 红线，归档 + 事件压缩 + VACUUM |

### 性能优化要点

1. **WAL 模式**：save.db 使用 `JournalMode.WAL`，提升并发写入性能，读不阻塞写。
2. **异步保存**：SaveWriter 使用 `Dispatchers.IO` 协程，不阻塞 UI 线程；UI 通过 StateFlow 观察保存状态。
3. **批量写入**：SaveDbInitializer 分批拷贝球员状态（500/批），避免内存溢出。
4. **meta.json 快速索引**：存档列表读取 JSON 文件而非打开数据库，避免 20 个存档逐个打开 DB 的开销。
5. **并发保护**：Mutex per save，防止同时写入同一存档；globalLock 保护切换存档操作。
6. **VACUUM 时机**：赛季归档后（T19 触发）+ 体积达 85% 自动触发 + 手动触发。
7. **事件压缩**：T19 赛季归档压缩比赛事件（仅保留比分+进球+红黄牌+Top5 评分），控制体积增长。
8. **checkpoint 清理**：按类型保留数量（LIGHT 4 / SEASON 5 / MIGRATION 2 / USER 10 / PRE_SAVE 2），避免无限增长。
9. **cache 重建优化**：分批构建搜索索引（500/批），避免全量加载到内存。

## 十一、验收标准

依据 T03 任务范围与 V0.2 `09_长程性能_存档_版本迁移.md`：

1. **创建存档**：可选择剧本/模式/范围/俱乐部，创建完整可玩存档，初始化数据正确。
2. **加载存档**：3 秒内加载完成，游戏状态（日期/俱乐部/赛季）正确恢复。
3. **保存存档**：自动保存 ≤2 秒，手动保存 ≤2 秒，异步不阻塞 UI，保存失败可回滚。
4. **切换存档**：保存当前 → 关闭 → 加载新存档，状态正确切换。
5. **删除存档**：二次确认 + 完整清理（save.db + meta.json + checkpoints + 可选 backups）。
6. **存档元信息**：ID/名称/时间/日期/俱乐部/剧本/范围/开关/版本/大小全部记录且 DB 与 JSON 一致。
7. **多存档管理**：列表排序（4 种）+ 名称搜索 + 数量上限（≤20）。
8. **版本迁移**：V1→V2 迁移成功 + 迁移前备份 + 失败回滚 + 不兼容提示。
9. **完整性检测**：7 项检测全部生效 + 严重程度分级（MINOR/MEDIUM/SEVERE）。
10. **损坏修复**：轻微自动修复 + 中等 checkpoint 恢复 + 严重备份恢复。
11. **checkpoint 机制**：5 种类型（LIGHT/SEASON/MIGRATION/USER/PRE_SAVE）+ 创建/恢复/清理。

13. **cache 重建**：加载时重建 + 搜索索引构建 + 分批处理。
14. **存档体积**：20 年 ≤80MB + VACUUM 压缩 + 体积预警（85%）。
15. **并发保护**：Mutex per save + globalLock + 同时只能加载一个存档。

## 十四、风险与应对

### 1. 存档损坏无法恢复

- **表现**：save_xxx.db 文件损坏，checkpoint 和备份全部失效，玩家进度永久丢失
- **应对**：
  - 三层防护：PRE_SAVE checkpoint（每次保存前）+ 自动备份（每周/赛季）+ 手动备份
  - SaveIntegrityChecker 启动时执行 7 项检测，发现问题立即提示
  - SaveBackupManager 保留最近 N 份备份（默认 5 份），覆盖策略 LRU
  - WAL 模式确保写入原子性，崩溃时主库不损坏
  - 关键操作前（迁移/VACUUM/删除）强制创建 PRE_CHECKPOINT
- **降级**：若所有恢复手段失效，提供「导出存档摘要」功能（球员名单/赛季成绩/转会记录以 JSON 导出），玩家可基于摘要手动重建存档

### 2. 保存性能不达标（>2 秒）

- **表现**：单次 SaveWriter.save 耗时超过 2 秒，UI 卡顿或异步保存堆积
- **应对**：
  - Coroutine + Dispatchers.IO 异步保存，UI 不阻塞
  - WAL 模式 + `synchronous=NORMAL`，写性能提升 3-5 倍
  - 增量保存：仅写入变更的表（通过 dirty 标记追踪）
  - 批量事务：单事务内完成所有写入，减少 fsync 次数
  - 性能监控：SavePerformanceMonitor 记录每次保存耗时，超阈值告警
- **降级**：极端情况（存档体积 >80MB）可关闭部分 checkpoint（仅保留 LIGHT），降低 IO 压力；或拆分为「快速保存」（核心表）+「完整保存」（全部表）两档

### 3. 版本迁移失败

- **表现**：用户从 V1 升级到 V2 后，存档打开失败或数据丢失
- **应对**：
  - 迁移前强制创建 MIGRATION 类型 checkpoint（全量备份）
  - MigrationStep 每步执行后校验完整性，失败立即回滚
  - MigrationRegistry 显式注册迁移脚本，不支持跳版本（V1→V3 须先 V1→V2 再 V2→V3）
  - checkCompatibility 返回 AUTO/MANUAL/INCOMPATIBLE 三态，INCOMPATIBLE 拒绝迁移并提示用户
  - 迁移日志完整记录，便于事后分析
- **降级**：迁移失败时自动回滚到迁移前备份；若备份也损坏，提供「只读模式」打开旧存档，玩家可查看历史数据但无法继续游戏

### 4. 存档体积膨胀超过 80MB

- **表现**：长期游戏后 save_xxx.db 超过 80MB，保存/加载变慢，存储空间告急
- **应对**：
  - 赛季归档：T19 将历史赛季数据迁移到 history.db（只读归档），save.db 仅保留当前赛季
  - 定期 VACUUM：清理碎片，压缩体积（默认每月一次）
  - WAL checkpoint：定期将 WAL 合并回主库，避免 WAL 文件膨胀
  - SaveMetaReader 监控体积，超阈值（60MB）提示用户归档
  - 历史新闻/事件日志按保留策略清理（保留最近 3 赛季）
- **降级**：强制触发赛季归档（即使未到赛季末）；或关闭部分历史日志记录（如关闭训练日志详情）

### 5. 并发写入冲突

- **表现**：多个 Coroutine 同时尝试保存同一存档，导致数据错乱或死锁
- **应对**：
  - SaveWriteLock：Mutex per save（每个 save_id 独立锁）+ globalLock（全局写锁）
  - saveWithLock 强制串行化同一存档的所有写操作
  - 保存进行中拒绝新的保存请求，返回 `SAVE_IN_PROGRESS` 状态
  - UI 层显示「保存中...」状态，禁止玩家触发会修改数据的操作
  - 单测覆盖并发场景（10 个 Coroutine 同时保存）
- **降级**：若 Mutex 出现死锁（超时 5 秒），强制释放锁并触发一次完整性检测，必要时回滚到最近 checkpoint

### 6. 存储空间不足

- **表现**：设备存储不足，保存失败或文件截断
- **应对**：
  - SaveWriter 保存前检查可用空间，低于阈值（200MB）拒绝保存并提示用户
  - 自动清理旧备份（BackupCleaner）和旧 checkpoint（CheckpointCleaner）
  - 自动清理 cache.db（CacheRebuilder 可重建，优先清理）
  - SaveMetaReader 显示各存档体积，帮助用户识别大存档
- **降级**：极端情况可降级为「最小保存」（仅玩家核心进度：日期/阵容/积分/财务），牺牲部分历史日志；或提示用户手动删除旧存档

### 7. checkpoint 文件丢失或损坏

- **表现**：checkpoint 文件被外部清理工具删除，或文件系统错误导致损坏
- **应对**：
  - checkpoint 存储在 save 内部目录（`/data/data/<pkg>/files/saves/<save_id>/checkpoints/`），不暴露在外部存储
  - 每个 checkpoint 附带 `checkpoint.meta.json`（含 checksum），加载时校验
  - 关键 checkpoint（SEASON/MIGRATION）额外复制一份到 `backups/` 目录
  - SaveIntegrityChecker 启动时检测 checkpoint 完整性
- **降级**：若 LIGHT checkpoint 丢失，从最近 SEASON checkpoint 恢复（损失最近一周进度）；若所有 checkpoint 丢失，尝试从自动备份恢复；若都失效，触发风险 1 的降级方案

### 8. cache.db 重建失败

- **表现**：CacheRebuilder 从 save.db 重建 cache 时失败，导致 UI 数据缺失
- **应对**：
  - cache.db 标记为「可重建」，任何损坏都可从 save.db 重新生成
  - CacheRebuilder 采用事务批量写入，失败时整体回滚
  - 重建过程显示进度条，避免玩家误以为卡死
  - 重建完成后执行完整性校验（行数对比）
  - 单测覆盖：故意损坏 cache.db，验证可自动重建
- **降级**：若重建仍失败，提供「安全模式」启动（跳过 cache，直接查 save.db，性能下降但功能可用），并提示用户重新创建存档可能解决问题
